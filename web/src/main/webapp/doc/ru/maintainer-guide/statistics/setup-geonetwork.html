Настройка геосети¶

Примечание

GeoNetwork package should be build using the Maven es profile in order to be able to configure the search/content statistics to use ElasticSearch/Kibana:

$ cd GN_SOURCES/
$ mvn clean install -Penv-prod,es

Если вы используете среду разработки, вы можете использовать следующую команду из папки исходников Geonetwork:

$ cd GN_SOURCES/web
$ mvn jetty:run -Penv-dev,es

Чтобы настроить статистику поиска/контента в GeoNetwork, выполните следующие действия:

  • Прокомментируйте следующий фильтр в TOMCAT_DIR/webapps/geonetwork/WEB-INF/web.xml

<!--<filter-mapping>
  <filter-name>XFrameOptionsFilter</filter-name>
  <url-pattern>/*</url-pattern>
</filter-mapping>-->

, иначе при доступе к дашборду в консоли js отображается следующая ошибка:

Refused to display 'http://localhost:8080/geonetwork/dashboards/app/kibana#/dashboard/5b407790-4fa1-11e7-a577-3197d1592a1d?embed=true&_g=()' in a frame because an ancestor violates the following Content Security Policy directive: "frame-ancestors 'none'".

  • Запуск GeoNetwork.

  • Войдите в систему как пользователь «администратор» и загрузите шаблоны (http://localhost:8080/geonetwork/srv/eng/admin.console#/metadata).

  • Создайте несколько записей метаданных (при желании вы можете загрузить образцы).

  • Включите статистику поиска на странице настроек (http://localhost:8080/geonetwork/srv/eng/admin.console#/settings).

  • Выполните поиск в GeoNetwork.

  • Перейдите в «Статистика и статус» в http://localhost:8080/geonetwork/srv/eng/admin.console#/dashboard и выберите «Статистика поиска».

В первый раз вам будет предложено выбрать индекс Kibana по умолчанию. Выберите .dashboards:

и установите по умолчанию, чтобы заставить его работать.

  • Перезагрузите страницу и проверьте панели мониторинга.

Статистика/проверка контента¶

Статистические данные о контенте и статусе проверки также доступны из раздела «Статистика и статус» в http://localhost:8080/geonetwork/srv/eng/admin.console#/dashboard, но требуют ручной индексации контента в ElasticSearch. Это будет улучшено в будущих версиях.

  • Чтобы проиндексировать содержимое и статус проверки в ElasticSearch, перейдите по адресу http://localhost:8080/geonetwork/srv/eng/admin.console#/tools и выберите следующий параметр:

  • Доступ к «Статистика содержания» и «Проверка» в разделе «Статистика и статус» (http://localhost:8080/geonetwork/srv/eng/admin.console#/dashboard)
