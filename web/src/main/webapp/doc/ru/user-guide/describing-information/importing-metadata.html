Импорт новой записи¶

Редактор может импортировать метаданные в файл каталога в различных форматах: XML, MEF или ZIP (см. Формат обмена метаданными (MEF)).

Before you begin¶

The user should have an editor profile to access metadata.

  1. Go to the contribute page and select Import new records.

Using the import new records page, you can:

  • выберите «Загрузить файл со своего компьютера» и выберите один файл XML или MEF для загрузки.

  • выберите «Копировать/Вставить» и скопируйте XML-документ в текстовую область.

  • выберите «Импортировать набор файлов из папки на сервере» и укажите путь к папке на сервере

Чтобы импортировать несколько файлов одновременно, используйте формат MEF или параметры импорта с сервера.

  1. After you have defined the type of import, configure the other import settings:

  • Тип файла: при загрузке или загрузке файла с сервера укажите тип загружаемого файла. Это может быть XML для импорта документа XML или MEF (эквивалент ZIP) для импорта формата MEF.

  • Тип записи:

  • Use Metadata when loading a normal metadata record

  • Use Template when the loaded metadata record will be used as a template.

  • Обработка идентификатора записи определяет, как обрабатывать потенциальные конфликты между UUID загруженной записи и UUID записей метаданных, уже присутствующих в каталоге. Доступны 3 стратегии:

  • Нет: UUID загруженной записи остается неизменным. Если запись метаданных с таким же UUID уже присутствует в каталоге, возвращается сообщение об ошибке.

  • Перезаписать метаданные с тем же UUID: любая существующая запись метаданных в каталоге, имеющая тот же UUID, что и загруженная запись, будет обновлена.

  • Создать UUID для вставленных метаданных: новый UUID влияет на загруженную запись.

  • Применить преобразование XSLT позволяет преобразовать запись, загруженную с использованием таблицы стилей XSLT. Предоставляется список предопределенных преобразований. Выбранное преобразование должно быть совместимо со стандартом загруженной записи (см. Добавление преобразования XSLT для импорта).

  • Validate инициирует проверку записи перед ее загрузкой. В случае ошибки запись отклоняется и сообщается об ошибке.

  • Назначить текущему каталогу назначает текущий каталог источником записи, если в файле MEF указан другой источник.

  • Assign to Group определяет группу загруженной записи.

  • Назначить категории определяет локальную категорию для назначения загруженной записи.

3. Click import to trigger the import. After processing, a summary is provided with the following details:

  • общее количество импортированных метаданных

  • сообщения об ошибках

  • если импортируется только одна запись, предоставляется ссылка на эту запись.
