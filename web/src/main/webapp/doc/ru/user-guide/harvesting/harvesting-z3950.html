Z3950 Сбор урожая¶

Z3950 — это протокол удаленного поиска и сбора, который обычно используется для разрешения поиска и сбора метаданных. Хотя этот протокол часто используется для библиотечных каталогов, с помощью Z3950 также можно выполнять поиск в важных каталогах геопространственных метаданных (например, в коллекциях метаданных правительственных учреждений Австралии, которые участвуют в Австралийском каталоге пространственных данных — ASDD). Этот сборщик позволяет пользователю указать запрос Z3950 и получить записи метаданных с одного или нескольких серверов Z3950.

Добавление комбайна Z3950¶

Доступные варианты:

  • Сайт

    • Название - краткое описание этого комбайна Z3950. Он будет отображаться на главной странице сбора урожая под этим именем.

    • Сервер(ы) Z3950 — это серверы Z3950, которые будут найдены. Вы можете выбрать один или несколько из этих серверов.

    • Запрос Z3950 — укажите запрос Z3950, который будет использоваться при поиске на выбранных серверах Z3950. В настоящее время известно, что это поле поддерживает формат запроса префикса (также известный как нотация запроса префикса), который описан по этому URL-адресу: http://www.indexdata.com/yaz/doc/tools.html#PQF. См. ниже дополнительную информацию и несколько простых примеров.

    • Значок — значок, назначаемый собранным метаданным. Значок будет использоваться при отображении результатов поиска.

  • Опции — параметры расписания.

  • Собранный контент

    • Применить этот XSLT к собранным записям — выберите здесь XSLT, который преобразует собранные записи в другой формат.

    • Проверить — если этот флажок установлен, записи, которые не проходят/не могут быть проверены, будут отклонены.

  • Привилегии

  • Категории

Примечание

этот харвестер автоматически создает новую категорию, названную в честь каждого из серверов Z3950, которые возвращают записи. Записи, возвращаемые сервером, назначаются категории, названной в честь этого сервера.

Подробнее о запросах PQF Z3950¶

PQF — довольно загадочный язык запросов. Он основан на идее атрибутов и наборов атрибутов. Наиболее распространенным набором атрибутов, используемым для геопространственных метаданных на серверах Z3950, является набор атрибутов GEO (который является расширением наборов атрибутов BIB-1 и GILS — см. http://www.fgdc.gov/standards/projects/GeoProfile). Таким образом, все запросы PQF к серверам геопространственных метаданных Z3950 должны начинаться с @attrset geo.

Наиболее полезные типы атрибутов в наборе атрибутов GEO следующие:

@номер атрибута

Значение

Описание

1

Использовать

В каком поле искать

2

Связь

Как сравнить указанный срок

4

Состав

Какой тип термина? например. дата, число, фраза

5

Усечение

Как обрезать, например. верно

Обратите внимание, что это не полный набор сопоставлений между набором атрибутов Z3950 GEO и именами полей индекса GeoNetwork lucene для ISO19139. Ознакомьтесь с INSTALL_DIR/web/geonetwork/xml/search/z3950Server.xsl и INSTALL_DIR/web/geonetwork/xml/schemas/iso19139/index-fields.xsl для получения более подробной информации и приложения A атрибута GEO, установленного для Z3950, по адресу http:/ /www.fgdc.gov/standards/projects/GeoProfile/annex_a.html для получения более подробной информации.

Общие значения атрибута отношения (&quot;@attr=2``):

@атрибут 2=

3

4

5

Больше чем

6

7

9

Таким образом, простой запрос для получения всех записей метаданных, содержащих слово «the» в любом поле, будет выглядеть так:

@attrset geo @attr 1=1016

  • @attr 1=1016 означает, что мы выполняем поиск по любому полю в записи метаданных.

Более сложный поиск в ограничивающей рамке можно сформулировать так:

@attrset geo @attr 1=2060 @attr 4=201 @attr 2=7 &quot;-36.8262 142.6465 -44.3848 151.2598`

  • @attr 1=2060 означает, что мы выполняем поиск по ограничивающей рамке

  • @attr 4=201 означает, что запрос содержит строки координат

  • @attr 2=7 означает, что мы ищем записи, ограничивающая рамка которых перекрывает поле запроса, указанное в конце запроса.

Примечания¶

  • Серверы Z3950 должны быть настроены для GeoNetwork в INSTALL_DIR/web/geonetwork/WEB-INF/classes/JZKitConfig.xml.tem

  • каждый раз, когда харвестер запускается, он удаляет ранее собранные записи и создает новые.
