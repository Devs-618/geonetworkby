Сбор данных GeoPortal REST¶

Этот сборщик подключается к удаленному серверу GeoPortal версии 9.3.x или 10.x и извлекает записи метаданных, которые соответствуют параметрам запроса, указанным с помощью REST API GeoPortal.

Добавление сборщика GeoPortal REST¶

На рисунке выше показаны доступные варианты:

  • Сайт — параметры удаленного сайта.

    • Имя — это краткое описание удаленного сайта. Оно будет отображаться на главной странице сбора данных как имя этого экземпляра сборщика данных GeoPortal REST.

    • Базовый URL — базовый URL-адрес собираемого сервера GeoPortal. например. http://вашхост.com/геопортал. Харвестер добавит дополнительный путь, необходимый для доступа к службам REST на сервере GeoPortal.

    • Значок — значок, назначаемый собранным метаданным. Значок будет использоваться при отображении собранных записей метаданных в результатах поиска.

  • Критерии поиска. С помощью кнопки «Добавить» вы можете добавить несколько критериев поиска. Вы можете запросить любое поле на сервере GeoPortal, используя синтаксис запроса Lucene, описанный по адресу http://webhelp.esri.com/geoportal_extension/9.3.1/index.htm#srch_lucene.htm.

  • Опции — параметры расписания.

  • Собранный контент — параметры, применяемые к собранному контенту.

    • Применить этот XSLT к собранным записям — выберите здесь XSLT, который преобразует собранные записи в другой формат. См. раздел примечаний ниже для типичного использования.

    • Проверить — если этот флажок установлен, метаданные будут проверены после извлечения. Если проверка не проходит, метаданные будут пропущены.

  • Права — назначьте права доступа к собранным метаданным.

Примечания¶

  • этот харвестер использует две службы REST из GeoPortal API:

  • rest/find/document с параметром searchText для возврата RSS-листинга записей метаданных, соответствующих критериям поиска (максимум 100000)

  • rest/document с параметром id из каждого результата, возвращенного в листинге RSS

  • этот комбайн был протестирован с GeoPortal 9.3.x и 10.x. Его можно использовать вместо комбайна CSW, если есть проблемы с обработкой стандартов OGC и т. д.

  • обычно метаданные ISO19115, созданные программным обеспечением Geoportal, не будут иметь префикса «gmd» для пространства имен «http://www.isotc211.org/2005/gmd». GeoNetwork XSLT не будет иметь проблем с пониманием этих метаданных, но не сможет отображать заголовки и списки кодов в средстве просмотра/редакторе. Чтобы решить эту проблему, выберите XSLT «Add-gmd-prefix» для Применить этот XSLT к собранным записям в наборе параметров Собранное содержимое, описанных ранее.
