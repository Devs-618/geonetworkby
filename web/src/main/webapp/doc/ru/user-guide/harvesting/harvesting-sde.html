Сбор узла ARCSDE¶

Это протокол сбора метаданных, хранящихся в установке ArcSDE.

Добавление харвестера ArcSDE¶

Сборщик идентифицирует формат метаданных ESRI: ESRI ISO, ESRI FGDC, чтобы применить необходимые xslts для преобразования метаданных в ISO19139. Варианты конфигурации:

  • Идентификация

    • Имя — это краткое описание узла. Он будет отображаться на главной странице сбора урожая.

    • Группа — администратор этой группы и администратор каталога могут управлять этим узлом.

    • Пользователь Harvester — пользователь, которому принадлежат собранные метаданные.

  • Расписание — настройка расписания для запуска харвестера.

  • Конфигурация для протокола ArcSDE

    • Сервер — IP-адрес или имя сервера ArcSde.

    • Порт — сервисный порт ArcSde (обычно 5151) или порт базы данных ArcSde, в зависимости от выбранного типа подключения, см. ниже раздел Тип подключения.

    • Имя базы данных — имя экземпляра ArcSDE (обычно esri_sde).

    • ArcSde version — версия ArcSde для сбора урожая. Модель данных, используемая ArcSde, различается в зависимости от версии ArcSde.

    • Тип соединения.

      • Служба ArcSde — использует службу ArcSde для получения метаданных.

      Примечание

      Additional installation steps are required to use the ArcSDE harvester because it needs proprietary ESRI Java api jars to be installed.

      ArcSDE Java API libraries need to be installed by the user in GeoNetwork (folder INSTALL_DIR_GEONETWORK/WEB-INF/lib), as these are proprietary libraries not distributed with GeoNetwork.

      Потребуются следующие банки:

      • jpe_sdk.jar

      • jsde_sdk.jar

      dummy-api-XXX.jar необходимо удалить из INSTALL_DIR/web/geonetwork/WEB-INF/lib

      • Прямое подключение к базе данных — использует подключение к базе данных (JDBC) для извлечения метаданных. С

      Примечание

      Для прямого подключения к базе данных необходимо скопировать драйверы JDBC в INSTALL_DIR_GEONETWORK/WEB-INF/lib.

      Драйверы Postgres JDBC распространяются вместе с GeoNetwork, но не для Oracle или SqlServer.

    • Тип базы данных - Тип базы данных ArcSde: Oracle, Postgres, SqlServer. Доступно только в том случае, если тип подключения настроен на Прямое подключение к базе данных.

    • Имя пользователя — имя пользователя для подключения к серверу ArcSDE.

    • Пароль - пароль пользователя ArcSDE.

  • Дополнительные параметры протокола arcsde

    • Проверять записи перед импортом — определяет критерии отклонения метаданных, недопустимых в соответствии с правилами XSD и schematron.

      • Принимать все метаданные без проверки.

      • Принимать метаданные, которые являются допустимыми XSD.

      • Принимать метаданные, которые являются допустимыми XSD и schematron.

  • Права — назначьте права доступа к собранным метаданным.
