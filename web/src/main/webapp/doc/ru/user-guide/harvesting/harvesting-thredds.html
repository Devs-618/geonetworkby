РЕЗЬБЫ Сбор урожая¶

Каталоги THREDDS описывают перечень наборов данных. Они организованы в иерархическом порядке, перечисляя описательную информацию и методы доступа для каждого набора данных. Обычно они каталогизируют наборы данных netCDF, но не ограничиваются этими типами файлов. Этот тип сбора просматривает каталог THREDDS, собирая метаданные для наборов данных и сервисов, описанных в нем или в связанных наборах данных netCDF. Этот тип сбора может извлекать фрагменты метаданных из каталога THREDDS, позволяя пользователю связывать или копировать эти фрагменты в шаблон для создания записей метаданных.

Добавление сборщика каталогов THREDDS¶

Доступные варианты:

  • Сайт

    • Имя - это краткое описание каталога THREDDS. Оно будет отображаться на главной странице сбора данных как имя этого экземпляра сборщика THREDDS.

    • URL-адрес каталога — удаленный URL-адрес каталога THREDDS, из которого будут собираться метаданные. Это должна быть xml-версия каталога (т. е. заканчивающаяся на .xml). Сборщик просканирует все наборы данных и службы, определенные в этом каталоге, создавая для них метаданные, как указано в параметрах, описанных ниже.

    • Язык метаданных — используйте этот параметр, чтобы указать язык собираемых метаданных.

    • Категория темы ISO — используйте этот параметр, чтобы указать категорию темы ISO метаданных службы.

    • Создать метаданные ISO19119 для всех служб в каталоге — выберите этот параметр, чтобы создать метаданные iso19119 для служб, определенных в каталоге THREDDS (например, OpenDAP, OGC WCS, ftp) и для самого каталога THREDDS.

    • Создать метаданные для наборов данных коллекции — выберите этот параметр, чтобы создать метаданные для каждого набора данных коллекции (набор данных THREDDS, содержащий другие наборы данных). Создание метаданных можно настроить с помощью параметров, которые отображаются при выборе этого параметра, как описано ниже.

    • Создать метаданные для атомарных наборов данных — выберите этот параметр, чтобы создать метаданные для каждого атомарного набора данных (набор данных THREDDS, не содержащий других наборов данных — например, для каталогизации набора данных netCDF). Создание метаданных можно настроить с помощью параметров, которые отображаются при выборе этого параметра, как описано ниже.

      • Игнорировать атрибут сбора данных — выберите этот параметр, чтобы собирать метаданные для выбранных наборов данных независимо от атрибута сбора данных для набора данных в каталоге THREDDS. Если этот параметр не выбран, метаданные будут создаваться только для наборов данных, для которых атрибут урожая имеет значение true.

      • Извлечь элементы метаданных DIF и создать метаданные ISO. Выберите этот параметр, чтобы создать метаданные ISO для наборов данных в каталоге THREDDS, содержащих элементы метаданных DIF. Когда выбран этот параметр, отображается список схем, для которых доступна таблица стилей DIFToISO.xsl (см., например, GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/DIFToISO.xsl). Метаданные генерируются путем считывания элементов метаданных DIF в THREDDS в запись метаданных формата DIF и последующего преобразования этой записи DIF в ISO с использованием таблицы стилей DIFToISO.

      • Извлекать метаданные обнаружения наборов данных Unidata с использованием фрагментов — выберите этот параметр, если метаданные в ваших наборах данных THREDDS или netCDF/ncml соответствуют соглашениям об обнаружении наборов данных Unidata (см. http://www.unidata.ucar.edu/software/netcdf-java/formats). /DataDiscoveryAttConvention.html). Вам нужно будет написать свои собственные таблицы стилей, чтобы извлечь эти метаданные в виде фрагментов и определить шаблон для объединения с фрагментами. При выборе этого параметра будут показаны следующие дополнительные параметры:

        • Выберите схему для выходных записей метаданных — выберите схему или профиль метаданных ISO для собранных записей метаданных. Примечание: в списке будут отображаться только те схемы, которые содержат таблицы стилей фрагментов THREDDS (см. следующий вариант расположения этих таблиц стилей).

        • Таблица стилей для создания фрагментов метаданных — выберите таблицу стилей для преобразования метаданных для набора данных (метаданные THREDDS и netCDF ncml, где это применимо) во фрагменты метаданных. Эти таблицы стилей можно найти в каталоге convert/ThreddsToFragments в каталоге схемы, например. для iso19139 это будет GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/ThreddsToFragments.

        • Создать подшаблоны для фрагментов и XLink их в шаблон — выберите этот параметр, чтобы создать подшаблон (=фрагмент метаданных, хранящийся в каталоге GeoNetwork) для каждого сгенерированного фрагмента метаданных.

        • Шаблон для объединения с фрагментами — выберите шаблон, который будет заполнен фрагментами метаданных, сгенерированными для каждого набора данных. Сгенерированные фрагменты метаданных используются для замены элементов, на которые ссылаются, в шаблонах ссылкой на подшаблон, если установлен флажок Создать подшаблоны. Если флажок Создавать подшаблоны не установлен, то фрагменты просто копируются в запись метаданных шаблона.

        • Для атомарных наборов данных предоставляется одна дополнительная опция Собирать только новые или измененные наборы данных. Если этот флажок установлен, будут собираться только те наборы данных, которые были изменены или не существовали на момент последнего запуска сборщика данных.

    • Создать миниатюры — выберите этот параметр, чтобы создать миниатюры для слоев WMS в связанных сервисах WMS.

    • Значок — значок, назначаемый собранным метаданным. Значок будет использоваться при отображении результатов поиска.

  • Опции — параметры расписания.

  • Привилегии

  • Категория службы — выберите категорию для назначения записям службы ISO19119 для служб THREDDS.

  • Категория для наборов данных — выберите категорию для назначения сгенерированных записей метаданных (и любых подшаблонов).

Внизу страницы расположены следующие кнопки:

  • Назад — вернуться на главную страницу сбора урожая. Определение сбора урожая не добавляется.

  • Сохранить — сохраняет это определение харвестера, создавая новый экземпляр харвестера. После завершения операции сохранения отобразится главная страница сбора урожая.

Подробнее о сборе элементов метаданных THREDDS DIF с помощью THREDDS Harvester¶

Каталоги THREDDS могут включать элементы из стандарта метаданных DIF. Библиотека Unidata netcdf-java предоставляет процесс DIFWriter, который может создавать запись метаданных DIF из этих элементов. GeoNetwork имеет таблицу стилей DIFToISO для преобразования этих записей DIF в ISO.

Подробнее о сборе метаданных обнаружения наборов данных Unidata с помощью THREDDS Harvester¶

Варианты, описанные выше для Извлечение метаданных обнаружения набора данных Unidata с использованием фрагментов (см. http://www.unidata.ucar.edu/software/netcdf-java/formats/DataDiscoveryAttConvention.html для получения дополнительной информации об этих соглашениях), вызывают следующие процесс для каждого набора данных коллекции или атомарного набора данных в каталоге THREDDS:

  1. Сборщик объединяет URI каталога, сгенерированный uuid, метаданные THREDDS для набора данных (сгенерированные с помощью веб-службы подмножества каталога) и ncml для наборов данных netCDF в один XML-документ. Пример показан ниже.

  2. Затем этот документ преобразуется с использованием указанной таблицы стилей (см. параметр Stylesheet выше) для получения документа фрагментов метаданных.

  3. Затем вызывается сборщик фрагментов метаданных для создания подшаблонов и/или метаданных для каждого набора данных по запросу.

Пример¶

Элементы метаданных DIF в наборах данных в каталогах THREDDS не так широко используются, как элементы метаданных, соответствующие соглашениям о метаданных обнаружения наборов данных Unidata. В этом примере показано, как собирать элементы метаданных, соответствующие соглашениям об обнаружении данных Unidata (см. http://www.unidata.ucar.edu/software/netcdf-java/formats/DataDiscoveryAttConvention.html).

В качестве примеров того, как собирать фрагменты метаданных из каталога THREDDS, предоставляются две эталонные таблицы стилей. Одна из этих таблиц стилей, thredds-metadata.xsl, предназначена для создания фрагментов метаданных iso19139 из метаданных THREDDS в соответствии с соглашениями об обнаружении наборов данных Unidata. Другая таблица стилей, netcdf-attributes.xsl, предназначена для создания фрагментов iso19139 из наборов данных netCDF в соответствии с соглашениями об обнаружении наборов данных Unidata. Эти таблицы стилей предназначены для использования с шаблоном «HARVESTING TEMPLATE — THREDDS — DATA DISCOVERY» и могут быть найдены в каталоге схемы «convert», например. для ISO19139 это GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/ThreddsToFragments.

Образец шаблона «ШАБЛОН СБОРА — ПОРОГИ — ОБНАРУЖЕНИЕ ДАННЫХ» был предоставлен для использования с таблицами стилей, описанными выше, для схемы метаданных iso19139. Этот шаблон находится в каталоге шаблонов схемы, например. для ISO19139 это GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/thredds-harvester-unidata-data-discovery.xml. * Прежде чем * пытаться запустить этот пример, вы должны убедиться, что этот и другие шаблоны из схемы iso19139 были загружены в GeoNetwork с помощью функции «Добавить шаблоны» в меню «Администрирование».

Теперь мы приведем пример того, как настроить комбайн и собрать метаданные THREDDS из одного из общедоступных каталогов материнских лодок unidata по адресу http://motherlode.ucar.edu:8080/thredds/catalog/satellite/3.9/WEST-CONUS_4km. /каталог.xml. Если бы вы вставили этот URL-адрес в свой браузер, вы бы увидели XML-представление этого каталога THREDDS. Это документ, который считывается и преобразуется в метаданные сборщиком THREDDS.

В GeoNetwork войдите в меню «Администрирование», выберите «Управление сбором урожая», как описано ранее. Добавьте комбайн из каталога THREDDS. Заполните форму управления заготовкой.

Первое, на что следует обратить внимание, это то, что Service URL должен быть http://motherlode.ucar.edu:8080/thredds/catalog/satellite/3.9/WEST-CONUS_4km/catalog.xml. Убедитесь, что вы используете xml-версию каталога. Если вы используете html-версию, вы не сможете собирать какие-либо метаданные.

Теперь, поскольку в этом каталоге THREDDS материнской загрузки unidata есть множество наборов данных на уровне файлов (фактически многие тысячи), мы будем собирать только метаданные коллекций. Для этого вы должны установить флажок Создать метаданные для наборов данных коллекции и игнорировать атомарные наборы данных.

Далее, поскольку метаданные в этом каталоге соответствуют соглашениям об обнаружении данных Unidata, мы выберем Извлечь метаданные обнаружения набора данных Unidata, используя фрагменты.

Далее мы отметим Игнорировать атрибут сбора. Мы делаем это потому, что наборы данных в каталоге THREDDS могут иметь атрибут, указывающий, следует ли собирать набор данных или нет. Поскольку ни один из наборов данных в этом каталоге не имеет атрибута сбора урожая, мы его проигнорируем. Если бы мы не отметили этот флажок, все наборы данных были бы пропущены.

Затем мы выберем схему метаданных, в которой будут записаны собранные метаданные. Здесь мы выберем iso19139, потому что это схема, для которой у нас есть таблицы стилей, которые будут преобразовывать метаданные THREDDS во фрагменты метаданных iso19139, и шаблон, в который эти фрагменты метаданных могут быть скопированы или связаны. После выбора iso19139 появятся варианты, отображающие эти таблицы стилей и шаблоны.

Первый выбор — это таблица стилей, которая создаст фрагменты метаданных iso19139. Поскольку нас интересуют элементы метаданных thredds в каталоге THREDDS, мы выберем метаданные (iso19139) thredds (расположенные в GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/ThreddsToFragments) для преобразования этих элементов в фрагменты метаданных iso19139.

Для целей этой демонстрации мы не будем проверять Создавать подшаблоны для фрагментов (xlinks…). Это означает, что фрагменты метаданных, созданные таблицей стилей, будут скопированы непосредственно в шаблон метаданных. Их нельзя будет повторно использовать (например, разделить между разными записями метаданных). См. предыдущий раздел о фрагментах метаданных, если вы не уверены, что это означает.

Наконец, мы выберем HARVESTING TEMPLATE — THREDDS — UNIDATA DISCOVERY в качестве записи метаданных шаблона, которая будет объединена с фрагментами метаданных для создания выходных записей. Этот шаблон будет загружен в GeoNetwork из GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/thredds-harvester-unidata-data-discovery.xml с помощью функции «Добавить шаблоны» в интерфейсе администрирования. Этот шаблон может быть заполнен метаданными, общими для всех записей, до запуска сборщика. Процесс, с помощью которого шаблон используется для создания записей метаданных, выглядит следующим образом:

  1. Для каждого набора данных в каталоге THREDDS будет скопирован шаблон для создания новой записи метаданных iso19139.

  2. Каждый фрагмент метаданных, собранный из набора данных THREDDS, будет скопирован в новую запись метаданных iso19139 путем сопоставления идентификатора в шаблоне с идентификатором во фрагменте (это совпадение создается разработчиком шаблона и таблицы стилей).

  3. Затем новая запись вставляется в каталог метаданных GeoNetwork, а содержимое записи индексируется в Lucene для поиска.

Затем вы можете заполнить оставшуюся часть формы в зависимости от того, как часто вы хотите обновлять собранные метаданные, какие категории будут присвоены созданной записи метаданных, какой значок будет отображаться с записями метаданных в результатах поиска и что привилегии на созданные записи метаданных будут.

Сохраните экран комбайна. Затем на экране управления сбором урожая установите флажок рядом с только что созданным комбайном, активируйте его, а затем запустите. Через несколько секунд (в зависимости от вашего интернет-соединения и компьютера) вы должны нажать Обновить. Если ваш урожай был успешным, вы должны увидеть панель результатов, похожую на ту, что показана на следующем снимке экрана.

Обратите внимание, что для 48 наборов данных уровня коллекции в этом каталоге THREDDS было создано 48 записей метаданных. Каждая запись метаданных была сформирована путем дублирования шаблона метаданных и последующего копирования в него 13 фрагментов метаданных — таким образом, всего было собрано 624 фрагмента.
