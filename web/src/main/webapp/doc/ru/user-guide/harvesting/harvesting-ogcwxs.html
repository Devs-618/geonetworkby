Сбор услуг OGC¶

Служба OGC реализует операцию GetCapabilities, которую GeoNetwork, выступая в качестве клиента, может использовать для создания метаданных для службы (ISO19119) и ресурсов, предоставляемых службой (ISO19115/19139). Этот харвестер поддерживает следующие сервисы и версии OGC:

  • Картографический веб-сервис (WMS) — версии 1.0.0, 1.1.1, 1.3.0

  • Служба веб-функций (WFS) — версии 1.0.0 и 1.1.0

  • Служба веб-покрытия (WCS) — версия 1.0.0

  • Служба веб-обработки (WPS) — версии 0.4.0 и 1.0.0

  • Службы каталогов для Интернета (CSW) — версия 2.0.2

  • Служба сенсорного наблюдения (SOS) - версия 1.0.0

Добавление OGC Service Harvester¶

Варианты конфигурации:

  • Сайт

    • Имя - Название каталога и будет одним из критериев поиска.

    • Тип — тип службы OGC указывает, должен ли сборщик запрашивать определенный тип службы. Поддерживаемые типы: WMS (1.0.0, 1.1.1, 1.3.0), WFS (1.0.0 и 1.1.0), WCS (1.0.0), WPS (0.4.0 и 1.0.0), CSW (2.0 .2) и SOS (1.0.0).

    • URL-адрес службы. URL-адрес службы — это URL-адрес службы, с которой можно связаться (без таких параметров, как &quot;REQUEST=GetCapabilities&quot;, &quot;VERSION=&quot;, …). Это должен быть допустимый URL-адрес, например http://your.preferred.ogcservice/type_wms.

    • Язык метаданных — обязательное поле, определяющее язык метаданных. Это должен быть язык, используемый администратором веб-службы OGC.

    • Категория темы ISO — используется для заполнения элемента категории темы в метаданных. Рекомендуется выбрать одну из них, поскольку категория темы является обязательной для стандарта ISO19115/19139, если иерархическим уровнем является «наборы данных».

    • Тип импорта — по умолчанию сборщик создает одну запись метаданных службы. Флажки в этой группе определяют другие метаданные, которые будут созданы.

      • Создать метаданные для элементов слоя с использованием информации GetCapabilities: выбор этой опции означает, что сборщик будет циклически перебирать наборы данных, обслуживаемые службой, как описано в документе GetCapabilities.

      • Create metadata for layer elements using MetadataURL attributes: Checkthis option means that the harvester will generate metadata from an XML document referenced in the MetadataUrl attribute of the dataset in the GetCapabilities document. If the document referred to by this attribute is not valid (eg. unknown schema, bad XML format), the GetCapabilities document is used as per the previous option.

      • Создавать эскизы для слоев WMS: при сборе данных из OGC WMS, отметка этой опции означает, что эскизы будут создаваться во время сбора данных.

    • Целевая схема — схема метаданных записей метаданных набора данных, которые будут созданы этим сборщиком данных.

    • Иконка — значок по умолчанию, отображаемый как логотип атрибуции для метаданных, созданных этим сборщиком.

  • Опции — параметры расписания.

  • Привилегии

  • Категория сервиса — метаданные собранного сервиса назначаются категории, выбранной в этом параметре (например, «интерактивные ресурсы»).

  • Категория для наборов данных — метаданные для собранных наборов данных назначаются категории, выбранной в этом параметре (например, «наборы данных»).

Примечания¶

  • каждый раз, когда харвестер запускается, он удаляет ранее собранные записи и создает новые записи. GeoNetwork сгенерирует uuid для всех метаданных (как службы, так и наборов данных). Исключением из этого правила являются метаданные набора данных, созданные с помощью тега MetadataUrl, в документе GetCapabilities, в этом случае вместо этого используется uuid удаленного XML-документа.

  • миниатюры могут быть созданы только при сборе картографической веб-службы OGC (WMS). WMS должна поддерживать проекцию WGS84.

  • выбранная целевая схема должна иметь поддержку XSLT, которые используются сборщиком для преобразования оператора GetCapabilities в записи метаданных из этой схемы. Если сомневаетесь, используйте iso19139.
