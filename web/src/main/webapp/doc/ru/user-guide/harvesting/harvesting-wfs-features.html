Сбор данных WFS GetFeature¶

Метаданные могут присутствовать в таблицах реляционных баз данных, которые обычно используются многими организациями. Размещение OGC Web Feature Service (WFS) над реляционной базой данных позволит извлекать метаданные с помощью стандартных механизмов запросов. Этот тип сбора позволяет пользователю указать запрос GetFeature и сопоставить информацию из объектов с фрагментами метаданных, которые можно связать или скопировать в шаблон для создания записей метаданных.

Добавление сборщика OGC WFS GetFeature¶

Служба веб-объектов OGC (WFS) реализует операцию запроса GetFeature, которая возвращает данные в виде объектов (обычно строк из связанных таблиц в реляционной базе данных). GeoNetwork, выступая в роли клиента, может прочитать ответ GetFeature и применить предоставленную пользователем таблицу стилей XSLT для создания фрагментов метаданных, которые можно связать или скопировать в предоставленный пользователем шаблон для создания записей метаданных.

Доступные варианты:

  • Сайт

  • Имя - это краткое описание комбайна. Оно будет отображаться на главной странице сбора данных как имя этого сборщика данных WFS GetFeature.

  • URL-адрес службы — пустой URL-адрес службы WFS (параметры OGC не требуются)

  • Язык метаданных — язык, который будет использоваться в записях метаданных, создаваемых харвестером.

  • Запрос OGC WFS GetFeature — запрос OGC WFS GetFeature, используемый для извлечения функций из WFS.

  • Схема для выходных записей метаданных — выберите схему или профиль метаданных для собранных записей метаданных. Примечание: в списке будут отображаться только схемы, содержащие таблицы стилей фрагментов WFS (расположение этих таблиц стилей см. в следующей опции).

    • Таблица стилей для создания фрагментов — Предоставляемая пользователем таблица стилей, которая преобразует ответ GetFeature в документ фрагментов метаданных (формат этого документа см. ниже). Таблицы стилей находятся в каталоге WFSToFragments, который находится в каталоге convert выбранной выходной схемы. например. для схемы iso19139 это каталог GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/WFSToFragments.

    • Сохранить большой ответ на диск — установите этот флажок, если вы ожидаете, что ответ WFS GetFeature будет большим (например, более 10 МБ). Если флажок установлен, ответ GetFeature будет сохранен на диск во временном файле. Затем каждая функция будет извлечена из временного файла и использована для создания фрагментов и записей метаданных. Если флажок не установлен, ответ будет храниться в оперативной памяти.

    • Создать подшаблоны. Установите этот флажок, если хотите, чтобы собранные фрагменты метаданных сохранялись как подшаблоны в каталоге метаданных и подключались xlink'ом к шаблону метаданных (см. следующий параметр). Если флажок не установлен, фрагменты будут скопированы в шаблон метаданных.

    • Шаблон для создания метаданных с использованием фрагментов — выберите шаблон метаданных, который будет объединен с собранными фрагментами метаданных для создания записей метаданных. Это стандартная запись шаблона метаданных GeoNetwork.

  • Категория для записей, созданных из связанных фрагментов. Выберите шаблон метаданных, который будет объединен с собранными фрагментами метаданных для создания записей метаданных. Это стандартная запись шаблона метаданных GeoNetwork.

  • Параметры

  • Привилегии

  • Категория для подшаблонов - когда фрагменты сохраняются в GeoNetwork как подшаблоны, они будут отнесены к выбранной здесь категории.

Подробнее о превращении ответа GetFeature во фрагменты метаданных¶

В корне<record> элемент может быть от нуля до многих<record> элементы. При создании метаданных каждый элемент записи приведет к созданию одного документа метаданных, в противном случае<record> Элемент используется для группировки фрагментов метаданных только по мере необходимости (например, фрагменты, сгенерированные для набора данных или объекта).

В течение<record> элемент может быть от нуля до многих<fragment> элементы и от нуля до многих<replacementGroup> элементы. А<replacementGroup> сам элемент может содержать от нуля до многих<fragment> элементы. Заказ<fragment> элементы и<replacementGroup> элементы внутри<record> или<replacementGroup> элемент не важен.

<fragment>элементы содержат отдельные фрагменты xml. Содержание<fragment> может быть любым элементом xml из поддерживаемой схемы GeoNetwork при условии, что этот элемент должен содержать достаточно релевантных метаданных, позволяющих идентифицировать целевую схему (т. е. различать пространства имен).

<replacementGroup>элементы имеют значение только во время генерации метаданных. Они используются для группировки нуля или более фрагментов для вставки или создания ссылок в копии шаблона метаданных, используемого для создания метаданных. Где<replacementGroup> элемент не содержит<fragment> элементы, указанный элемент в копии шаблона будет удален, в противном случае он будет заменен содержимым фрагмента.

Допустимые атрибуты этих элементов и их функции следующие:

Элемент

Атрибут

Описание

Записывать

Уид

Uuid сгенерированной записи метаданных (необязательно — в противном случае он будет назначен харвестером)

Фрагмент

Идентификатор

Идентификатор элемента в шаблоне метаданных для замены/ссылки. Игнорируется, если фрагмент находится в группе замещения.

Наконец, можно привести два примера того, как собирать метаданные из функций сборщика OGC WFS, используя таблицы стилей и шаблоны, поставляемые с GeoNetwork.

Связанный пример GeoServer Boundaries Harvest¶

В этом примере предполагается, что вы установили пакет GeoServer, поставляемый с GeoNetwork. Конечным результатом этого примера будет 251 запись метаданных ISO19139, которые связывают 1506 фрагментов (по 6 на запись), созданных из ответа GetFeature в шейп-файле границ в экземпляре GeoServer, поставляемом с GeoNetwork. Созданные записи содержат метаданные о странах мира.

Порядок действий следующий:

  • В меню Администрирование-&gt;Конфигурация системы включите преобразователь XLink и сохраните конфигурацию в базе данных.

  • Добавьте сборщик ответов OGC WFS GetFeature из меню Администрирование-&gt; Сбор данных.

  • Дайте ему Имя (например, gboundaries) и введите URL-адрес службы wfs из связанного геосервера (например, http://localhost:8080/geoserver/wfs) в поле URL-адрес службы.

  • Мы будем использовать простой запрос GetFeature, чтобы выбрать все страны из шейп-файла границ за WFS. XML для такого запроса (который должен быть введен в текстовую область GetFeature Query):

<wfs:GetFeature service="WFS" version="1.1.0"
        xmlns:wfs="http://www.opengis.net/wfs">

 <wfs:Query typeName="gboundaries"/>

</wfs:GetFeature>

  • Выберите выходную схему — мы выберем iso19139, так как эта схема содержит примеры таблиц стилей и шаблонов, которые нам нужны для этого примера. Обратите внимание, что после выбора этой опции становятся видны следующие опции, и мы предпримем следующие действия:

    • Выберите предоставленную таблицу стилей «geoserver_boundary_fragments», чтобы извлечь фрагменты из ответа GetFeature в Таблица стилей, чтобы использовать их для создания раскрывающегося списка фрагментов. Эту таблицу стилей можно найти в GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/WFSToFragments.

    • Выберите прилагаемый шаблон «Шаблон теста границ стран для фрагментов геосервера WFS» из раскрывающегося списка Шаблон для использования для построения метаданных с использованием фрагментов. Этот шаблон можно найти в GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/geoserver_fragment_tester.xml.

  • Выберите категорию для записей, созданных харвестером, установите флажок Только один запуск, добавьте некоторые привилегии (самый простой — разрешить всем пользователям иметь право просмотра). На этом этапе форма ввода вашего комбайна должна выглядеть так, как показано на следующем снимке экрана.

  • Сохраните форму ввода комбайна.

  • Вы вернетесь в меню операций комбайна, где сможете Активировать комбайн, а затем Запустить его.

Страница результатов показывает, что из ответа WFS GetFeature было получено 1506 фрагментов метаданных. Они были сохранены в базе данных GeoNetwork как подшаблоны и связаны с шаблоном метаданных для формирования 251 новой записи метаданных.

Пример базы данных философов Deegree версии 2.x¶

В этом примере предполагается, что вы загрузили версию 2.x Deegree и загрузили пример базы данных Philosopher. Конечным результатом этого примера будет 7 записей метаданных ISO19139, которые связаны между собой 42 фрагментами (по 6 на запись), созданными из ответа GetFeature из вашей установки deegree. Записи содержат метаданные о 7 известных философах.

Порядок действий следующий:

  • В меню Администрирование-&gt;Конфигурация системы включите преобразователь XLink и сохраните конфигурацию в базе данных.

  • Добавьте сборщик ответов OGC WFS GetFeature из меню Администрирование-&gt; Сбор данных.

  • Дайте ему Имя (например, deegree22-philosopher-test) и введите URL-адрес вашей установки deegree 2.2 в поле URL-адрес службы.

  • Мы будем использовать простой запрос GetFeature, чтобы выбрать всех философов из базы данных WFS. XML для такого запроса (который должен быть введен в текстовую область GetFeature Query):

<wfs:GetFeature version="1.1.0" xmlns:app="http://www.deegree.org/app"
           xmlns:wfs="http://www.opengis.net/wfs">

 <!-- request all Philosopher instances -->
 <wfs:Query typeName="app:Philosopher"/>

</wfs:GetFeature>

  • Выберите выходную схему — мы выберем iso19139, так как эта схема содержит примеры таблиц стилей и шаблонов, которые нам нужны для этого примера. Обратите внимание, что после выбора этой опции становятся видны следующие опции, и мы предпримем следующие действия:

    • Выберите предоставленную таблицу стилей 'deegree2_philosopher_fragments', чтобы извлечь фрагменты из ответа GetFeature в Таблице стилей, чтобы использовать их для создания раскрывающегося списка фрагментов. Эту таблицу стилей можно найти в GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/WFSToFragments.

    • Выберите прилагаемый шаблон «Deegree 22 WFS Fragments Philosopher Database Test Template» из раскрывающегося списка Шаблон, который будет использоваться для построения метаданных с использованием фрагментов. Этот шаблон можно найти в GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/deegree_fragment_tester.xml.

  • Выберите категорию для записей, созданных харвестером, установите флажок Только один запуск, добавьте некоторые привилегии (самый простой — разрешить всем пользователям иметь право просмотра). На этом этапе форма ввода вашего комбайна должна выглядеть так, как показано на следующем снимке экрана.

  • Сохраните форму ввода комбайна.

  • Вы вернетесь в меню операций комбайна, где сможете Активировать комбайн, а затем Запустить его.

После запуска комбайна вы должны увидеть экран результатов, который выглядит примерно так, как показано на следующем снимке экрана.

Страница результатов показывает, что из ответа WFS GetFeature было получено 42 фрагмента метаданных. Они были сохранены в базе данных GeoNetwork как подшаблоны и связаны с шаблоном метаданных для формирования 7 новых записей метаданных.
