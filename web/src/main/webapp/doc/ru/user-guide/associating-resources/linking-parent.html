Группировка наборов данных в серию (т. е. отношение родитель/потомок)¶

В некоторых ситуациях набор данных является частью временной или пространственной коллекции записей, которые могут иметь одну и ту же спецификацию (например, земной покров за разные годы). Чтобы сгруппировать этот набор записей, общее описание коллекции может быть сделано в родительских метаданных, которые затем могут быть присоединены к каждому набору данных серии.

  • Земельный покров Корин

  • Земельный покров Корин 2002 г.

  • Земельный покров Корин 2012 г.

  • …

Отношения родитель/потомок могут быть установлены для записей ISO19139 и Dublin Core. В ISO19139 ссылка на родительскую запись кодируется в дочерней записи следующим образом:

<gmd:parentIdentifier>
   <gco:CharacterString>78f93047-74f8-4419-ac3d-fc62e4b0477b</gco:CharacterString>
</gmd:parentIdentifier>

Отношение родитель/потомок в ISO19139 также может быть закодировано с использованием агрегатов (см. связывание других).

При создании такой связи пользователи смогут перемещаться между записями в режиме поиска и записи.

Нажмите «Ссылка на родителя», чтобы получить доступ к панели, обеспечивающей простой поиск для выбора метаданных целевого родителя.
