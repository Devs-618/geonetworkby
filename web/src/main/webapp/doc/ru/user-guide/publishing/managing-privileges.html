Управление привилегиями¶

To manage privileges to your metadata record and any attached data, you will need to identify User Groups and the privileges you want to assign to users in these groups. eg. Viewing the metadata, downloading the data attached to the record, etc.

For example, you can specify that the metadata and related services are visible to all (Internet users) or just to internal users only (Intranet). Privileges are assigned on a per group basis. Depending on the user profile (Guest, Registered User, Editor, Admin etc.) access to these functions may differ on a per user basis.

Assigning privileges¶

Чтобы назначить привилегии, выполните следующие действия:

  1. Find your metadata record using the search option. Whether you have multiple or single results from the search, on top of the individual record or next to the record you will always see a row of buttons including a Privileges button.

  2. Click the Privileges button. A drop down menu appears. You can assign certain privileges to specific groups using checkboxes.

  3. Click the small box next to the privilege to place or remove a checkmark. The Set All and Clear All buttons allow you to place and remove the checkmarks all at once.

Ниже приведено краткое описание каждой привилегии, чтобы помочь вам определить, какие из них вы должны назначить той или иной группе (группам).

Опубликовать: Пользователи в указанных группах могут просматривать метаданные, например. если он соответствует критериям поиска, введенным таким пользователем.

Загрузить: Пользователи в указанных группах могут загружать данные.

Интерактивная карта: Пользователи в указанных группах могут получить интерактивную карту. Интерактивная карта должна быть создана отдельно с использованием сервера веб-карт, такого как GeoServer, который распространяется вместе с GeoNetwork.

Избранное: при случайном выборе GeoNetwork запись метаданных может появиться в разделе «Избранное» на домашней странице GeoNetwork.

Уведомлять: пользователи в указанной группе получают уведомление, если загружаются данные, прикрепленные к записи метаданных.

Review and make sure that you understand Пользователи, группы и роли in the User and Group Administration section of this manual.

Примечание

Общедоступная запись метаданных — это запись метаданных, которая имеет право просмотра для группы с именем «Все».

Следующие правила применяются к разрешениям на просмотр и редактирование записи метаданных:

Viewing¶

Администратор может просматривать любые метаданные.

Рецензент контента может просматривать метаданные, если:

  • Метаданные являются общедоступными или

  • Метаданные являются частью группы, членом которой является пользователь.

Администратор пользователей или редактор могут просматривать:

  • All metadata that has the view privilege selected for one of the groups they are a member of.

  • All metadata created by them.

Зарегистрированный пользователь может просматривать:

  • All metadata that has the view privilege selected for one of the groups they are a member of.

Any user (logged in or not) can view the public metadata.

Редактирование¶

Администратор может редактировать любые метаданные.

Рецензент может редактировать метаданные, если:

  • Владелец метаданных является членом одной из групп, назначенных рецензенту.

  • They are the metadata owner.

A User Administrator or an Editor can only edit metadata they created.

Setting Privileges¶

Установка привилегий для записи метаданных¶

A button to access the Privileges page for a metadata record displays in the search results or when you are viewing the record for:

  • Все администраторы

  • Все рецензенты, входящие в одну из групп, назначенных владельцу метаданных.

  • Владелец метаданных

Only Administrators and Reviewers can edit privileges for the All and Intranet groups.

Установка привилегий для выбранного набора записей метаданных¶

You can set privileges on a selected set of records in the search results using the «actions on selected set» menu.

Применяются следующие правила:

  • группы - это те, к которым принадлежит пользователь

  • указанные привилегии будут применяться только к записям, на которые у пользователя есть права владения или администрирования — любые другие записи будут пропущены

  • the current records privileges will be reset and replaced by the selected privilege
