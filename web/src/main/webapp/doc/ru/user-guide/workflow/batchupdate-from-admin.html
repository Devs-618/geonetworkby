Обновление набора записей из консоли администратора¶

В некоторых случаях пользователи могут захотеть обновить набор записей. Пользователи с правами администратора могут из консоли администратора открыть страницу инструменты, пакетный процесс.

На этой странице пользователь может:

  • Выберите набор записей, выполнив поиск и выбрав записи для изменения.

  • Выберите процесс из списка

См.также

Можно добавить новый процесс. См. Добавление пакетного процесса.

  • Определить параметры процесса (если есть)

  • Запустите процесс и следите за прогрессом.

Процесс может применяться только к записям, которые может редактировать текущий пользователь. Если нет, то нередактируемая запись игнорируется и процесс продолжается. Отчет содержит следующую информацию:

  • Количество записей для обработки

  • Количество записей, затронутых процессом

  • Количество записей без изменений (для процесса поиска и замены)

  • Количество записей, для которых процесс не найден (процесс зависит от стандарта и может не существовать в зависимости от стандарта).

  • Количество записей, которые текущий пользователь не может редактировать

Перед запуском процесса рекомендуется сделать резервную копию всех записей метаданных для обновления на случай, если с процессом что-то пойдет не так.

Поиск и замена¶

Этот процесс ищет значения в элементах и заменяет их другими значениями. Он поддерживает записи ISO19139 и Dublin Core. Конфигурация следующая:

  • Выберите пакет (ISO19139 или Dublin Core)

  • Выберите элемент из этого пакета для замены (предварительно настроенный элемент относится к контакту, но его можно расширить — см. ниже)

  • Определите значение поиска

  • Определите замену

  • Нажмите «Добавить замену».

Можно настроить и добавить несколько замен. После завершения настройки пользователь может сохранить конфигурацию, нажав «Загрузить конфигурацию». Конфигурация загружается в формате JSON и может быть обновлена и повторно загружена позже путем копирования/вставки в текстовое поле конфигурации замены и нажатием «Загрузить конфигурацию».

Пример конфигурации:

[{
  "package":"iso19139",
  "element":"iso19139\\..*\\.individualName",
  "searchval":"John",
  "replaceval":"Pierre"
  }]

В конфигурации «элемент» определяет целевой элемент в записи метаданных. Он начинается с идентификатора схемы, а затем определяет путь к элементу. Это регулярное выражение и может использовать .* для сопоставления всех элементов в любом месте документа. Чтобы соответствовать более конкретному элементу, пользователь может определить полный путь, например iso19139\\.contact\\.individualName, чтобы он соответствовал только индивидуальному имени контакта в метаданных.

searchval также является регулярным выражением<http://www.regular-expressions.info/tutorial.html> `_. Это может быть простой текст или более сложное выражение. Например, при перехвате групп поиск ``(.*)` и замена на Mr $1 заменит John на Mr John.
