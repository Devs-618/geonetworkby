Обновление набора записей с помощью XSLT¶

Применение изменений¶

Пользователь с правами администратора может использовать API каталога для обновления набора записей с помощью XSLT-преобразования. Шаги следующие:

  • Войдите в систему как администратор

  • Поиск записей для обновления с использованием сервиса q, например. выберите все шаблоны в определенном стандарте: http://localhost:8080/geonetwork/srv/eng/q?_schema=iso19115-3&amp;_isTemplate=y

  • Выберите все записи, соответствующие поиску с http://localhost:8080/geonetwork/srv/eng/metadata.select?selected=add-all

  • Запустите процесс с помощью http://localhost:8080/geonetwork/srv/eng/md.processing.batch?process=my-custom-process.

  • Проверьте отчет о проделанной работе с помощью http://localhost:8080/geonetwork/srv/eng/md.processing.batch.report.

В конце процесса в резюме описывается:

  • Количество записей для обработки

  • Количество записей для обработки

  • Количество не найденных записей (в случае удаления одной записи после выбора)

  • Количество записей с ошибками

  • Количество записей с процессом, не определенным в его стандарте

  • Количество записей, которые текущий пользователь не может редактировать

Процесс также можно применить к одной записи метаданных с помощью службы md.processing. например. http://localhost:8080/geonetwork/srv/eng/md.processing?uuid=46eac9e4-33cb-45b7-a104-7bcc8e654c98&amp;process=keywords-mapper&amp;search=water&amp;replace=Water

Параметры:

  • uuid или id: UUID метаданных или внутренний идентификатор метаданных

  • процесс: идентификатор процесса

  • добавить другие дополнительные параметры в зависимости от процесса (см. ниже).

Добавление пакетного процесса¶

Создание файла обработки¶

Пакетный процесс определяется для каждой схемы. Чтобы проверить список доступных процессов для стандартной проверки <datadirectory> /config/плагины схемы/<schemaId> /процесс папка.

Папка процесса содержит набор XSLT. Имя файла XSLT без расширения используется для запуска процесса. Например, при использовании md.processing.batch?process=my-custom-process процесс XSLT ДОЛЖЕН называться my-custom-process.xsl.

Обработка XML записи¶

Процесс XSLT будет применяться к каждой записи метаданных в выборке. Каждый документ будет иметь в качестве корневого элемента XML-документ метаданных с элементом geonet:info. Элемент geonet:info содержит метаданные о метаданных. Этот элемент ДОЛЖЕН быть удален процессом, чтобы не изменять запись при сохранении в базе данных.

<gmd:MD_Metadata>
    ...
    <geonet:info xmlns:geonet="http://www.fao.org/geonetwork">
        <id>73481</id>
        <uuid>bb151890-2da5-4cfb-8659-7839e7138be7</uuid>
        <schema>iso19139</schema>
        <createDate>2015-12-23T17:05:36</createDate>
        <changeDate>2015-12-23T18:07:40</changeDate>
        <source>2cc603e1-981c-41a2-a183-39429c7dcc49</source>
        <ownerId>1</ownerId>
        <edit>true</edit>
        <owner>true</owner>
        <isPublishedToAll>false</isPublishedToAll>
        <view>true</view>
        <notify>true</notify>
        <download>true</download>
        <dynamic>true</dynamic>
        <featured>true</featured>
        <selected>true</selected>
    </geonet:info>
</gmd:MD_Metadata>

Добавление параметров¶

Процесс XSLT может извлекать параметры, которые могут быть предоставлены в URL-адресе, с помощью xsl:param. Например, при использовании md.processing.batch?process=my-custom-process&amp;myParameter=test.

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform" version="2.0"
                exclude-result-prefixes="#all">

    <xsl:param name="myParameter" select="''"/>

В этом примере xsl:param будет доступен как переменная с использованием $myParameter и будет иметь пустое значение, если не задано параметром URL. Чтобы проверить правильность установки параметра, используйте xsl:message для вывода информации в файл журнала.

<xsl:param name="myParameter" select="''"/>
<xsl:message>myParameter: <xsl:value-of select="$myParameter"/></xsl:message>

Процесс XSLT также имеет доступ к параметрам каталога:

  • guiLang: Текущий язык пользовательского интерфейса

  • baseUrl: базовый URL службы (например, http://localhost:8080/geonetwork)

  • catalogUrl: URL-адрес каталога (например, http://localhost:8080/geonetwork/srv/eng)

  • nodeId: идентификатор узла (по умолчанию srv)

Чтобы использовать один из этих параметров в процессе, используйте xsl:param:

<xsl:param name="guiLang" select="''"/>

Сделать копию - это минимум.¶

Процесс ДОЛЖЕН, по крайней мере, делать:

  • копия всего

  • удалить метаданные geonet:info

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
                xmlns:geonet="http://www.fao.org/geonetwork" version="2.0"
                exclude-result-prefixes="#all">

    <!-- Do a copy of every nodes and attributes recursively -->
    <xsl:template match="@*|node()">
        <xsl:copy>
            <xsl:apply-templates select="@*|node()"/>
        </xsl:copy>
    </xsl:template>

    <!-- Remove geonet:* elements. -->
    <xsl:template match="geonet:*" priority="2"/>
</xsl:stylesheet>

Затем настройте процесс для обновления записи метаданных.

Примеры обработки¶

См. примеры в source file schemas/iso19139/src/main/plugin/iso19139/process.

Помимо того минимума, который ДОЛЖЕН выполнять процесс, процесс может определять дополнительные действия, используя новые шаблоны:

  • Удаление элемента. Например. удаление всего отчета о DQ_TopologicalConsistency:

<xsl:template match="gmd:report[gmd:DQ_TopologicalConsistency]"
              priority="2"/>

Установите приоритет на 2, чтобы ваш шаблон имел приоритет над основным шаблоном, делающим копию всего.

Регистрация процесса как действия редактора¶

Процесс XSLT можно использовать в редакторе для запуска определенных действий. Например, в представлении INSPIRE отображается кнопка для добавления идентификатора ресурса, если он не определен, заканчивающийся идентификатором метаданных.

<action type="batch"
        process="add-resource-id"
        if="count(gmd:MD_Metadata/gmd:identificationInfo/*/
                      gmd:citation/gmd:CI_Citation/
                          gmd:identifier[
                          ends-with(
                              gmd:MD_Identifier/gmd:code/gco:CharacterString,
                              //gmd:MD_Metadata/gmd:fileIdentifier/gco:CharacterString
                          )]) = 0"/>

См. ссылку: «создание пользовательского редактора».

Добавление преобразования XSLT для импорта¶

Добавьте преобразования XSL в папку web/geonetwork/xsl/conversion/import, чтобы предоставить пользователю новые параметры импорта. Файлы можно добавлять в эту папку без перезапуска приложения.
