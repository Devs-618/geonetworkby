Настройка виртуальной CSW¶

Warning: This feature will be deprecated in future version (4.x). Use Portal configuration instead.

This feature adds the capability to create custom CSW entry points that apply extra criteria to the CSW requests, allowing to implement several useful cases like, for example:

  • Определите точку входа INSPIRE CSW, чтобы доставлять только связанные с INSPIRE метаданные, хранящиеся в каталоге.

  • Определите точки входа в CSW, чтобы предоставлять только метаданные, относящиеся к определенной теме (темам): климат, границы и т. д.

The filter parameter value should use the Lucene query parser syntax, which is applied to these CSW operations:

  • GetRecords: фильтр применяется к запросу CSW в качестве дополнительного критерия запроса.

  • GetRecordById: фильтр применяется с метаданными id, запрошенным в качестве дополнительного критерия запроса.

  • GetDomain: фильтр применяется в качестве критерия запроса для получения запрошенных свойств метаданных.

  • GetCapabilities: фильтр применяется в качестве критерия запроса для заполнения списка ключевых слов метаданных в документе GetCapabilities.

Список доступных полей индекса Lucene для использования в параметре filter можно получить из файлов index-fields.xsl в папках схемы, расположенных в WEB-INF/xml/schemas.

Поскольку на операции Harvest и Transaction параметр filter не влияет, во избежание путаницы лучше использовать эту функцию в качестве конечных точек CSW только для чтения.
