Конфигурация системы¶

Большинство параметров конфигурации системы могут быть изменены пользователями-администраторами с помощью веб-интерфейса в Консоль администратора &gt; Настройки.

Важно

Настройка этих параметров критически важна для каталога в рабочем контексте. Неправильное понимание некоторых настроек может привести к тому, что система не будет работать должным образом. Например, загрузки могут быть неправильно обработаны или сбор метаданных с других серверов может не работать.

Поскольку форма настроек длинная, кнопка «сохранить» повторяется между разделами и сохраняет все настройки.

Описание каталога¶

  • Имя каталога Имя узла. Информация, которая помогает идентифицировать каталог пользователю-человеку. Имя отображается на баннере, в CSW GetCapabilities.

  • Идентификатор каталога Универсальный уникальный идентификатор (uuid), отличающий ваш каталог от любого другого каталога. Это уникальный идентификатор вашего каталога, и лучше оставить его как uuid. Он будет использоваться харвестером по протоколу GeoNetwork для идентификации исходного каталога.

  • Организация Организация, которой принадлежит узел. Опять же, это информация, которая помогает пользователю идентифицировать каталог.

  • SVN UUID Subversion repository attached to the node. This identifier is created and/or checked on startup to verify that the database match the SVN repository. The repository is used for metadata versioning.

Каталог¶

  • Версия Версия каталога (только для чтения, версия базы данных)

  • Дополнительная версия Дополнительная версия каталога (только для чтения, версия базы данных)

Сервер каталогов¶

  • Host Имя или IP-адрес узла (без http://). Например, они используются при редактировании метаданных для создания ссылок на ресурсы и при возврате возможностей сервера во время запроса CSW.

  • Если ваш узел общедоступен из Интернета, вы должны использовать доменное имя.

  • Если ваш узел скрыт внутри вашей частной сети и у вас есть брандмауэр или веб-сервер, который перенаправляет входящие запросы на узел, вам необходимо ввести публичный адрес брандмауэра или веб-сервера. Типичной конфигурацией является наличие веб-сервера Apache на общедоступном адресе A, который перенаправляет запросы на сервер Tomcat на частный адрес B. В этом случае вы должны ввести A в параметре host.

  • Порт Номер порта сервера (обычно 80 или 8080). Если вы используете HTTP, установите его на 80.

  • Предпочитаемый протокол Определяет протокол для доступа к каталогу. Протокол HTTP, используемый для доступа к серверу. Выбор http означает, что вся связь с каталогом будет видна всем, кто слушает протокол. Поскольку это включает в себя имена пользователей и пароли, это небезопасно. Выбор https означает, что вся связь с каталогом будет зашифрована, и поэтому слушателю будет намного сложнее ее расшифровать.

  • Безопасный порт Номер защищенного порта.

  • Уровень ведения журнала Определите уровень ведения журнала приложения. После модификации журнал можно проверить в разделе «Статистика и статус» в разделе «Активность».

Интранет-параметры¶

Общей потребностью организации является автоматическое различение анонимных внутренних пользователей, которые получают доступ к узлу внутри организации (интранет), и анонимных внешних пользователей из Интернета. Каталог определяет анонимных пользователей внутри организации как принадлежащих к группе Интранет, а анонимных пользователей извне организации определяют как группу Все. Чтобы автоматически различать пользователей, принадлежащих к группе интрасети, необходимо сообщить каталогу IP-адрес интрасети и сетевую маску.

  • Маска сети Маска сети интрасети (например, 255.255.255.0).

  • Сеть* Адрес интрасети в форме IP (например, 147.109.100.0).

Z39.50¶

  • Включить сервер Z39.50 Включить сервер Z39.50. Перезапустите каталог, чтобы это изменение стало активным.

  • Порт Измените порт прослушивания Z39.50. Перезапустите каталог, чтобы это изменение стало активным.

Прокси сервер¶

The settings page offers to set the configuration of a proxy server. This configuration is used by the application to access the internet to get online resources, for example as part of a harvest process.

  • Использовать прокси Включите прокси-сервер, если каталог находится за прокси-сервером и его необходимо использовать для доступа к удаленным ресурсам.

  • Прокси-хост IP-адрес или имя прокси-сервера.

  • Порт Порт прокси

  • Имя пользователя-прокси Имя пользователя

  • Пароль пользователя прокси Пароль имени пользователя

  • *Игнорировать список хостов Чтобы пропустить определенные хосты, введите определенный IP-адрес или имя хоста, например www.mydomain.com, или диапазон адресов с использованием подстановочных знаков, например 192.168.2.*. Использование | для разделения различных значений хоста.

Параметры прокси-сервера JVM также могут потребоваться для правильной настройки прокси-сервера для всего удаленного доступа.

Обратная связь¶

Электронная почта может быть отправлена каталогом.

  • вы используете систему саморегистрации пользователей

  • вы используете рабочий процесс статуса метаданных (см. жизненный цикл)

  • загружается файл, загруженный с записью метаданных, и выбирается привилегия уведомления

В этом разделе настраивается используемый почтовый сервер.

  • Электронная почта Это адрес электронной почты администратора, используемый для отправки отзывов.

  • Хост SMTP Имя или IP-адрес почтового сервера для отправки электронной почты.

  • Порт SMTP Порт SMTP.

  • Использовать SSL Включить режим SSL

  • Имя пользователя Имя пользователя, если требуется подключение на SMTP-сервере

  • Пароль Имя пользователя пароль, если требуется подключение на SMTP-сервере

Результаты поиска метаданных¶

Параметры конфигурации в этой группе определяют ограничения на взаимодействие пользователя с результатами поиска.

  • Максимум выбранных записей Максимальное количество результатов поиска, которые пользователь может выбрать и обработать с помощью пакетных операций, например. Установите привилегии, категории и т. д. Этот параметр позволяет избежать длительного действия, которое может привести к ошибке нехватки памяти.

Рейтинг метаданных¶

Если включено, каталог будет рассчитывать пользовательские рейтинги для метаданных только из этого узла (не распространяться среди других узлов GeoNetwork). Это относится только к записям, собранным с использованием протокола GeoNetwork.

Служба загрузки¶

Редактор метаданных поддерживает загрузку одного или нескольких файлов, которые можно сохранить вместе с записью метаданных. Когда такая запись отображается в результатах поиска, предоставляется кнопка «Загрузить», которая позволяет пользователю выбрать, какой файл он хочет загрузить. Эта группа опций определяет, как будет происходить эта загрузка.

  • Используйте простую службу загрузки файлов GeoNetwork При нажатии на любой файл, сохраненный с записью метаданных, этот файл будет доставлен непосредственно пользователю через браузер.

  • Использовать службу заявлений об отказе от ответственности и ограничений GeoNetwork При нажатии на любой файл, хранящийся с записью метаданных, пользователю будет предоставлен zip-архив (через браузер), который содержит файл данных, саму запись метаданных и сводку метаданных ограничения ресурсов. в виде html-документа. Кроме того, пользователю необходимо будет предоставить некоторые сведения (имя, организацию, адрес электронной почты и необязательный комментарий) и просмотреть ограничения ресурсов, прежде чем он сможет загрузить zip-архив.

Метаданные XLink¶

Преобразователь XLink заменяет содержимое элементов с атрибутом @xlink:href (за исключением некоторых элементов, таких как srv:operatesOn) содержимым, полученным из содержимого URL-адреса @xlink:href. Преобразователь XLink должен быть включен, если вы хотите собирать фрагменты метаданных или повторно использовать фрагменты метаданных в своих записях метаданных (например, при использовании каталога контактов).

  • Включить разрешение XLink: включает/отключает преобразователь XLink.

  • Включить локальный XLink Локальные XLink используют local://<lang> /<service> URL-адрес для ссылок на связанные разделы вместо URL-адреса HTTP. Локальные XLinks обычно быстрее, чем HTTP XLinks.

См.также

Для повышения производительности каталог будет кэшировать содержимое, которого нет в локальном каталоге. Очистите кеш XLink из Консоли администратора &gt; Инструменты, если фрагменты были обновлены.

Обновление метаданных¶

Для каждой схемы метаданных каталог имеет преобразование XSL (update-fixed-info.xsl), которое можно применить к записи метаданных, принадлежащей этой схеме. Целью этого преобразования является разрешение применения фиксированной схемы, сведений о сайте и каталоге к записи метаданных каждый раз, когда запись метаданных сохраняется в редакторе. Например, это преобразование используется для создания и хранения URL-адресов любых файлов, загруженных и сохраненных с записью метаданных в редакторе.

  • Автоматические исправления: включено по умолчанию. Не рекомендуется использовать редактор метаданных, если автоматическое исправление отключено. См. http://trac.osgeo.org/geonetwork/ticket/368 для более подробной информации.

Статистика поиска¶

Включает/отключает сбор статистики поиска. Статистика поиска хранится в базе данных и может быть запрошена с помощью страницы «Статистика поиска».

Хранение статистики поиска требует очень мало вычислительных ресурсов, поскольку она записывается в базу данных в фоновом потоке. Однако хранение базы данных для очень загруженного сайта должно быть тщательно спланировано.

Оптимизация индекса¶

Параметры конфигурации в этой группе определяют, когда запускается Lucene Index Optimizer. По умолчанию это происходит каждый день в полночь. С недавними обновлениями Lucene, особенно Lucene 3.6.1, оптимизатор становится менее полезным, поэтому эта группа конфигурации, скорее всего, будет удалена в будущих версиях.

Поставщик Инициативы открытого архива (OAI-PMH)¶

Параметры в этой группе управляют тем, как сервер OAI отвечает на запросы сбора данных OAIPMH с удаленных сайтов.

  • Поиск по дате: OAI Harvesters могут запрашивать записи из GeoNetwork в диапазоне дат. GeoNetwork может использовать одно из двух полей даты из метаданных для проверки совпадения с этим диапазоном дат. По умолчанию выбран Временный экстент, который представляет собой временной экстент из записи метаданных. Другая опция, Дата модификации, использует дату модификации записи метаданных в базе данных GeoNetwork. Дата модификации — это последний раз, когда запись метаданных обновлялась или собиралась GeoNetwork.

  • Тайм-аут маркера возобновления: записи метаданных, соответствующие запросу на поиск урожая OAI, обычно возвращаются в сборщик группами фиксированного размера (например, группами по 10 записей). В каждую группу включен токен возобновления, чтобы сборщик мог запросить следующую группу записей. Время ожидания токена возобновления — это время (в секундах), в течение которого сервер GeoNetwork OAI будет ожидать использования токена возобновления. Если время ожидания превышено, сервер GeoNetwork OAI удалит результаты поиска и откажется распознавать токен возобновления. Цель этой функции — обеспечить высвобождение ресурсов на сервере GeoNetwork OAI.

  • Размер кэша: максимальное количество одновременных сборов OAI, которое может поддерживать сервер GeoNetwork OAI.

Перезапустите каталог, чтобы принять во внимание все настройки OAI.

ВДОХНОВЕНИЕ Атомная подача¶

Позволяет определить конфигурацию каналов Atom, на которые ссылаются метаданные, для предоставления услуг, связанных с техническим руководством INSPIRE для служб загрузки. <http://inspire.ec.europa.eu/documents/Network_Services/Technical_Guidance_Download_Services_3.0.pdf> `_:

  • Выберите тип подачи атомов:

    • Удаленный: получение каналов атомов, на которые ссылаются метаданные в онлайн-ресурсах.

    • Локальный (для реализации в будущих версиях): создайте поток атомов, используя содержимое метаданных.

  • Расписание получения каналов: можно запланировать периодическое извлечение каналов атомов.

  • Значение протокола Atom: значение протокола в онлайн-ресурсах метаданных для идентификации ресурсов канала Atom (значение по умолчанию — INSPIRE Atom). GeoNetwork идентифицирует файл Atom из других ресурсов, просматривая значение протокола онлайн-ресурса. Поскольку для этого протокола не существует общепринятого значения, GeoNetwork позволяет администратору установить значение, которое будет использоваться в качестве протокола, идентифицирующего ресурсы Atom:

    <gmd:transferOptions>
    <gmd:MD_DigitalTransferOptions>
      <gmd:onLine>
        <gmd:CI_OnlineResource>
          <gmd:linkage>
            <gmd:URL>http://geodata.nationaalgeoregister.nl/atom/index.xml</gmd:URL>
          </gmd:linkage>
          <gmd:protocol>
            <gco:CharacterString>INSPIRE Atom</gco:CharacterString>
          </gmd:protocol>
        </gmd:CI_OnlineResource>
      </gmd:onLine>
    </gmd:MD_DigitalTransferOptions>
  </gmd:transferOptions>

Доступны следующие услуги:

  • http://SERVER/geonetwork/opensearch/eng/UUID/OpenSearchDescription.xml

  • http://SERVER/geonetwork/opensearch/eng/UUID/search?queryParams

  • http://SERVER/geonetwork/opensearch/eng/search?queryParams

  • http://SERVER/geonetwork/opensearch/eng/UUID/describe?queryParams

  • http://SERVER/geonetwork/opensearch/eng/describe?queryParams

In above URLs {UUID} is the fileidentifier of the download service metadata.

В ленте службы вашей службы загрузки обязательно добавьте конечную точку GeoNetwork OpenSearch в качестве OpenSearchDescription для службы:

<link rel="search" href="http://SERVER/geonetwork/opensearch/eng/{uuid}/OpenSearchDescription.xml"
type="application/opensearchdescription+xml" title="Open Search document for INSPIRE Download service"/>

Реализация INSPIRE Atom/OpenSearch может быть проверена с помощью тестов Atom в Esdin Test Framework (http://elfproject.eu/documentation/geotool/etf) или средстве проверки метаданных INSPIRE (http://inspire-geoportal.ec.europa.eu). /валидатор2).

Многопоточное индексирование¶

Параметры конфигурации в этой группе определяют, сколько потоков процессора выделяется для задач индексирования в GeoNetwork. Если на вашем компьютере много процессорных ядер, теперь вы можете определить, сколько из них выделить для задач индексирования GeoNetwork. Это может привести к значительному увеличению скорости больших задач индексирования (например, изменение привилегий для 20 000 записей), поскольку GeoNetwork может разделить задачу индексирования на несколько частей и назначить их разным процессорным ядрам.

Количество потоков обработки Максимальное количество потоков обработки, которое может быть выделено для задачи индексирования.

Примечание: этот параметр доступен только для проверенных баз данных. Этими базами данных являются PostGIS и Oracle. Вы также должны тщательно продумать, сколько соединений с базой данных вы выделяете в конфигурации базы данных, поскольку каждый поток может связать одно соединение с базой данных на время длительного сеанса индексирования (например). Дополнительные сведения о том, как настроить количество соединений в пуле соединений с базой данных, см. в расширенной конфигурации.

Многоязычные настройки¶

Параметры в этой группе определяют, как GeoNetwork будет искать метаданные на нескольких языках.

Enable auto-detecting search request language: If this option is selected, GeoNetwork will analyse the search query and attempt to detect the language that is used before defaulting to the GUI language.

Результаты поиска на запрошенном языке сортируются сверху: Если выбран этот параметр, к каждому запросу будет добавлено условие сортировки, чтобы гарантировать, что результаты на текущем языке всегда сортируются сверху. Это отличается от повышения приоритета языка тем, что переопределяет релевантность результата. Например, если результат на немецком языке имеет очень высокую релевантность, но язык поиска французский, то все результаты на французском языке будут предшествовать результатам на немецком языке.

Поиск только на запрошенном языке Параметры в этом разделе определяют порядок сортировки/приоритета документов относительно языка в документе по сравнению с языком поиска.

  • Все документы на всех языках (без настроек) - Язык поиска игнорируется и не влияет на порядок результатов.

  • Предпочитать документы с переводом на запрашиваемый язык — документы с переводом на языке поиска (в любом месте документа) будут иметь приоритет над документами без каких-либо элементов на языке поиска.

  • Предпочитать документы, язык которых соответствует запрошенному языку. Документы на том же языке, что и язык поиска (т. е. документы, указанные как написанные на том же языке, что и язык поиска), имеют приоритет над документами, язык которых не соответствует языку.

  • Переводы на запрошенный язык - Результаты поиска будут содержать только те документы, которые имеют некоторые переводы на язык поиска.

  • Document language is the requested language - The search results will contain documents whose metadata language is specified as being the in search language

Представления метаданных¶

Параметры в этом разделе включают/отключают группы элементов метаданных в редакторе/просмотрщике метаданных.

Включить простой вид: Простой вид в редакторе/средстве просмотра метаданных: - удаляет большую часть иерархии из вложенных записей метаданных (например, ISO19115/19139) - не позволяет пользователю добавлять элементы метаданных, которых еще нет в записи метаданных Он предназначен для обеспечения плоского, простого представления записи метаданных. Недостатком простого представления является то, что часть контекстной информации, предоставляемой вложением в запись метаданных, теряется. Включить представление ISO: Стандарт метаданных ISO19115/19139 определяет три группы элементов: - Минимум: те элементы, которые являются обязательными - Основные: элементы, которые должны присутствовать в любой записи метаданных, описывающей набор географических данных - Все: все элементы * Включить представление INSPIRE*: включает группы элементов метаданных, определенные в директиве EU INSPIRE. Включить представление XML: это представление редактирования необработанного текста записи XML. Вы можете отключить это, если (например) не хотите, чтобы неопытные пользователи были сбиты с толку представлением XML, предоставляемым этим представлением.

Права доступа к метаданным¶

Задавать привилегии только группам пользователей: если этот параметр включен, на странице привилегий метаданных будут отображаться только группы, к которым принадлежит пользователь (если только пользователь не является администратором). На данный момент эту опцию нельзя отключить, и, вероятно, она будет объявлена устаревшей в следующей версии GeoNetwork.

Сбор урожая¶

Разрешить редактирование собранных записей: Включает/отключает редактирование собранных записей в каталоге. По умолчанию собранные записи нельзя редактировать.
