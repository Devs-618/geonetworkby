(Необязательно) Сборка¶

Исходный код GeoNetwork доступен в общедоступном репозитории на Github.<https://github.com/geonetwork/core-geonetwork> `_. Это означает, что вы можете клонировать, разветвлять и предлагать push-уведомления о ваших пользовательских изменениях. Если вы не знакомы с репозиториями кода или git, вы должны проверить это краткое руководство<https://try.github.io/levels/1/challenges/1> `_.

GeoNetwork is built using Maven version 3+. It is written on Java 7+. It works both with OpenJDK or the Oracle version. There are several ways to install this on your local machine; for example if you have a Debian based OS (like Ubuntu), you can install them with just this command:

$ sudo apt-get install maven git

Убедитесь, что вы установили maven версии 3!!

$ mvn --version
  Apache Maven 3.2.1 (ea8b2b07643dbb1b84b6d16e1f08391b666bc1e9; 2014-02-14T18:37:52+01:00)
  Maven home: ....

Remember that this will also install Java on your system. You can check that the version is the right one with the following command:

$ java -version

Итак, самый первый шаг после того, как вы настроили свою среду, — это клонировать репозиторий GeoNetwork на вашем локальном компьютере. Это можно сделать в командной строке, используя следующую команду в пустой папке, куда будет помещен исходный код:

$ cd yourEmptyFolder
$ git clone https://github.com/geonetwork/core-geonetwork.git
$ cd core-geonetwork
$ git submodule init
$ git submodule update

Как видите, весь исходный код, показанный на github, теперь также доступен на вашем локальном компьютере.

The source code of GeoNetwork is split on several smaller Maven projects. To run GeoNetwork, you have to build all of them and run the project named «web».

If you are familiar to Maven, you will probably have guessed that you have to run a package install command on the root folder of GeoNetwork source code. But if you try that, Maven will warn you that for building GeoNetwork you need more memory than the default memory provided to Maven. This means, you will have to export the Maven options to increase the memory like this:

$ export MAVEN_OPTS="-Xmx512M -XX:MaxPermSize=256M"

На данный момент нас не интересует запуск тестов, поэтому вы можете пропустить их, используя параметр -DskipTests:

$ mvn package install -DskipTests

Это создаст военный файл, который вы можете использовать в любом контейнере приложений Java (сервере), таком как Tomcat, в web/target/geonetwork.war.

Congratulations! You are ready to run GeoNetwork. To do this, just go to the web folder and run Jetty in there:

$ cd web; mvn jetty:run

After Jetty starts, you can see your running GeoNetwork by opening a browser and enter to http://localhost:8080/geonetwork
