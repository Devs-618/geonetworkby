Развертывание с использованием файла войны¶

To deploy GeoNetwork you just have to place the war file provided or generated on the last optional step on a proper Java container like Tomcat.

Чтобы установить Tomcat в операционной системе на основе Debian, вы можете использовать инструмент apt-get, например:

$ sudo apt-get install tomcat7

После установки Tomcat в вашей системе найдите папку webapps и поместите туда файл geonetwork.war. Это развернет GeoNetwork в вашей системе.

Примечание

You need to ensure Tomcat is configured with enough memory for GeoNetwork to launch. This can be configured via the setenv script in tomcat with the appropriate memory for the JAVA_OPTS property)

Еще один рекомендуемый шаг при развертывании — изменение каталога данных. В каталоге данных GeoNetwork хранит все файлы данных, такие как эскизы или загруженные документы, прикрепленные к записям метаданных. Вы можете начать с пустого каталога GeoNetwork, он будет автоматически заполнен.

Самый простой способ настроить каталог данных — обновить файл config-spring-geonetwork.xml, доступный внутри основного проекта:

<bean id="GeonetworkDataDirectory" class="org.fao.geonet.kernel.GeonetworkDataDirectory" lazy-init="true">
<property name="systemDataDir" ref="GNSystemDataDir"/>
</bean>
<bean id="GNSystemDataDir" class="java.nio.file.Paths" factory-method="get">
<constructor-arg index="0" value="/path/to/gn/dir"/>
<constructor-arg index="1"><array /></constructor-arg>
</bean>

Подробнее о каталоге-данных-настройки.

Вы можете убедиться, что GeoNetwork развернута по следующему адресу: http://localhost:8080/geonetwork.
