Настроить редактор (необязательно)¶

Каждая схема может настраивать различные представления редактора. Эти конфигурации можно найти в файле layout/config-editor.xml каждой схемы.

Этот файл состоит из четырех основных блоков:

Поля:

<!-- Form field type configuration. Default is text. -->
<fields>
<for name="gmd:abstract" use="textarea"/>
<for name="gco:Distance" use="number"/>
<for name="gmd:onLine" addDirective="data-gn-directory-entry-selector">
<directiveAttributes data-template-add-action="true" data-template-type="onLine" data-filter='{"_root": "gmd:CI_OnlineResource"}'/>
</for>
</fields>

Это определяет для каждого определенного типа XSD, какой тип ввода использовать. Как и в случае с gmd:onLine, можно определить настраиваемый тип ввода angular.

После определения всех простых полей мы определяем список наборов полей, которые редактор распознает как нечто, что нужно собрать вместе:

<fieldsWithFieldset>
<name>gmd:MD_Metadata</name>
<name>gmd:identificationInfo</name>
</fieldsWithFieldset>

Теперь определим, какие поля являются многоязычными. Обычно большинство полей являются многоязычными, поэтому мы можем просто указать, какие из них следует исключить.

<multilingualFields>
<!-- In multilingual mode, define which mode the widget should have. If expanded, then one field per language is displayed. -->
<expanded>
<name>gmd:title</name>
<name>gmd:abstract</name>
</expanded>
<exclude>
<name>gmd:identifier</name>
</exclude>
</multilingualFields>

Наконец, мы определяем представления, которые будут настраивать поля, которые будут отображаться при редактировании метаданных. Мы можем определить более одного представления для каждой схемы, поэтому пользователь редактора может выбрать, какое из них использовать при редактировании метаданных.

Когда в представлении определено не конечное поле, редактор автоматически сгенерирует все поля ввода внутри него. Таким образом, вам не нужно явно определять все поля, которые вы хотите показать. Самое простое представление основано на XSD схемы.

Представление xml также можно легко определить:

<view name="xml">
<tab id="xml" default="true"/>
</view>

Если представление слишком велико, мы также можем определить набор вкладок внутри редактора, чтобы не все поля отображались одновременно. Помните, что когда пользователь меняет вкладку, содержимое будет сохранено. Поэтому мы должны разместить связанные поля внутри одной и той же вкладки.

Помните, что вы должны перезапускать контейнер приложения (Tomcat) каждый раз, когда вы изменяете этот файл.

See more on Редактор настройки.
