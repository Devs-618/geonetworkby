Поля поиска¶

Lucene — поисковая система, используемая GeoNetwork. Вся конфигурация Lucene определяется в файле WEB-INF/config-lucene.xml.

В некоторых случаях уместно изменить или расширить поля поиска индекса метаданных. Например, чтобы добавить поле (которое затем доступно для поиска или может использоваться в представлении по умолчанию) или изменить содержимое поля, создается из метаданных (индексация).

Добавление нового поля¶

Каждая схема содержит файл index-fields/default.xsl, в котором определены поля, хранящиеся в индексе. Это xslt, который работает с метаданными xml и создает различные теги полей, которые Lucene распознает и обработает.

<xsl:for-each select="gmd:pointOfContact[1]/*/gmd:role/*/@codeListValue">
<Field name="responsiblePartyRole" string="{string(.)}" store="true" index="true"/>
</xsl:for-each>

Затем эти поля можно использовать в качестве параметров службы Q для поиска по этому полю.

Помните, что после каждого изменения конфигурации индекса новый индекс должен быть перестроен.

Резюме¶

Служба Q возвращает список результатов поиска, но также может возвращать сводку результатов, возвращенных с параметрами «buildSummary=true» и «summaryType=$summary».

Эта сводка представляет собой список наиболее распространенных значений для определенных полей. Эти сводки определены в файле web/src/main/webapp/WEB-INF/config-summary.xml:

<summaryType name="results">
<item facet="keyword" max="100"/>
</summaryType>

Элементы, используемые в этом резюме, должны быть определены в верхнем теге, называемом «фасеты».

Поля повышения¶

Иногда мы хотим, чтобы какое-то поле было более релевантным для поиска, чем другие. Мы можем повышать эти поля внутри тега fieldBoosting. Повышение представляет собой положительное или отрицательное значение с плавающей запятой.

<fieldBoosting>
<Field name="_op0" boost="0.0F"/>
<Field name="_op1" boost="0.0F"/>
<Field name="_op2" boost="0.0F"/>
<Field name="_dummy" boost="0.0F"/>
<Field name="_isTemplate" boost="2.0F"/>
<Field name="_owner" boost="5.0F"/>
</fieldBoosting>
