Редактор настройки¶

Конфигурация редактора метаданных определяется для конкретного стандарта подключаемого модуля схемы (см. Реализация плагинов схемы).

Конфигурация редактора определяет меню навигации (т. е. список представлений и вкладок) для редактора, список полей и тип используемого элемента управления. Элементы управления могут иметь тип HTML (например, текст, дата) или более сложные элементы управления, созданные с использованием директивы `AngularJS.<https://docs.angularjs.org/guide/directive> `_.

Чтобы создать такую конфигурацию редактора, пользователю необходимо знать XSD стандарта, чтобы правильно создавать представления, вкладки и поля в соответствии с именами элементов (см. source file schemas/config-editor.xsd). Создайте корневой элемент редактора и прикрепите:

  • схема и

  • пространства имен для стандартов

<editor xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:noNamespaceSchemaLocation="../../../../../../config-editor.xsd"
  xmlns:gn="http://www.fao.org/geonetwork"
  xmlns:gco="http://www.isotc211.org/2005/gco"
  xmlns:gmd="http://www.isotc211.org/2005/gmd"
  xmlns:gmx="http://www.isotc211.org/2005/gmx"
  xmlns:srv="http://www.isotc211.org/2005/srv"
  xmlns:gml="http://www.opengis.net/gml"
  xmlns:xlink="http://www.w3.org/1999/xlink">

Конфигурация редактора должна определять сначала некоторое общее описание элемента, а затем набор представлений, по крайней мере, с одним.

Дочерние элементы:

  • поля, необязательный элемент (см. Определение типа поля)

  • fieldsWithFieldset, необязательный элемент (см. Элемент группировки из стандартов)

  • multilingualFields, необязательный элемент (см. Определение многоязычных полей)

  • представления, обязательный элемент (см. Настройка представлений)

Определение типа поля¶

Определите конфигурацию типа полей формы. По умолчанию используется простой текстовый ввод. Этот список содержит список исключений, которые не используют простой ввод текста. Список возможных значений:

  • все типы ввода HTML5 или

  • an AngularJS directive name. MUST starts with „data-“ and could end with „-textarea“ to create a textarea element. It could end with „-div“ if the directive does not apply to the input or textarea but to the div containing it.

У элемента может быть определен только один тип.

<editor>
 <fields>
   <for name="gmd:abstract" use="textarea"/>
   <for name="gco:Real" use="number"/>
   <for name="gco:Boolean" use="checkbox"/>
   <for name="gco:Date" use="data-gn-date-picker"/>

Другой вариант определения более сложного типа поля — перехватить элемент с помощью шаблона XSL. Этот подход используется, например, для ключевых слов в ISO19139 (см. source file schemas/iso19139/src/main/plugin/iso19139/layout/layout-custom-fields-keywords.xsl).

Элемент группировки из стандартов¶

Список элементов, которые будут отображаться в наборе полей (т. е. элемент в штучной упаковке). Эти элементы обычно содержат дочерние элементы и определяют основной раздел стандарта. Например, в ISO19139 идентификация и распространение являются основными разделами и обычно должны отображаться как группа информации.

<editor>
   <fields>...</fields>
   <fieldsWithFieldset>
    <name>gmd:identificationInfo</name>
    <name>gmd:distributionInfo</name>

Определение многоязычных полей¶

Настройте здесь список многоязычных полей для стандарта.

По умолчанию, если стандарт поддерживает многоязычие, например ISO19139, все поля будут отображаться как многоязычные поля. Определите в разделе исключения исключение (например, gmd:identifier, например, в ISO19139).

Затем этот раздел также позволяет определить, как многоязычные поля отображаются с помощью развернутых элементов. Если развернуто, то отображается одно поле для каждого языка без необходимости нажимать на переключатель языка.

<editor>
   <fields>...</fields>
   <fieldsWithFieldset>...</fieldsWithFieldset>
    <multilingualFields>
      <expanded>
        <name>gmd:title</name>
        <name>gmd:abstract</name>
      </expanded>
      <exclude>
        <name>gmd:identifier</name>
        <name>gmd:metadataStandardName</name>

Настройка представлений¶

ДОЛЖЕН быть определен хотя бы один вид, но в зависимости от потребностей можно определить и другие режимы просмотра.

По умолчанию ISO19139 определяет 3 вида (т. е. по умолчанию, расширенный, xml) и один отключенный (т. е. INSPIRE). См. source file schemas/iso19139/src/main/plugin/iso19139/layout/config-editor.xml для конфигурации.

Еще одним базовым примером является представление Dublin Core (см. source file schemas/dublin-core/src/main/plugin/dublin-core/layout/config-editor.xml).

Чтобы создать новое представление, используйте следующее:

<views>
    <view name="viewNameInLocalizationFile">
      ...
    </view>

Дочерние элементы:

  • представление, одно или несколько (см. Определение представления)

Определение представления¶

Представление имеет метку и определяет конкретное отображение записей метаданных. Представление состоит из одной или нескольких вкладок.

<views>
    <view name="custom-view">
        ....
    </view>

Представление может отображаться или нет в зависимости от содержимого записи метаданных или текущего сеанса пользователя с использованием атрибутов displayIfRecord и displayIfServiceInfo.

Атрибуты:

  • имя (Обязательно)

Ключ имени представления, хранящийся в {schema}/loc/{lang}/strings.xml, или имя элемента с префиксом пространства имен.

<strings>
  <default>Simple</default>
  <inspire>INSPIRE</inspire>
  <custom-view>My view</custom-view>

  • отключено (необязательно) Фиксированное значение: true

Скрыть вид из меню, если атрибут определен. Позволяет легко отключить представление.

  • class (Optional)

Define custom CSS class to be set on the form element. This is mainly used to set the type of indent:

  • gn-label-above-input: to put label above form input

  • gn-indent-colored: colored left border on each fieldset

  • gn-indent-bluescale: blue scale colored left border on each fieldset

See catalog/views/default/less/gn_editor_default.less to add your custom editor styles.

  • upAndDownControlHidden (необязательно) Фиксированное значение: true

Определите, должны ли отображаться элементы управления вверх и вниз в этом представлении. Если не определено, отображаются элементы управления. Скройте эти элементы управления в представлении, чтобы упростить работу с меньшим количеством элементов управления для конечного пользователя.

  • displayAttributes (Optional) Fixed value: true

Display attributes by default when loading the view.

  • displayTooltips (Optional) Fixed value: true

Display help documentation for all elements by default when loading the view.

  • displayTooltipsMode (Optional)

Display help documentation onhover elements (default) or by clicking on an icon.

  • hideTimeInCalendar (необязательно) Фиксированное значение: true

Определите, должен ли элемент управления календарем разрешать пользователям устанавливать только дату или дату и время. Если атрибут не установлен, то можно установить дату и время. Это контролируется на уровне представления, переключение на другое представление может обеспечить больший контроль над датами.

  • displayIfRecord (необязательно)

Выражение XPath, возвращающее логическое значение, которое будет сравниваться с записью метаданных. если true, представление будет отображаться. например. Отображать пользовательское представление, если стандартное имя метаданных содержит Medsea:

<view name="custom-view"
      displayIfRecord="contains(gmd:MD_Metadata/
                                  gmd:metadataStandardName/gco:CharacterString,
                                'MedSea')"

  • displayIfServiceInfo (необязательно)

Выражение XPath, возвращающее логическое значение, которое будет оцениваться относительно информационного дерева службы (элемент Jeeves /root/gui). если true, представление будет отображаться.

например. Отображать настраиваемый вид, если пользователь является администратором:

<view name="custom-view"
      displayIfServiceInfo="count(session[profile = 'Administrator']) = 1"

displayIfRecord и displayIfServiceInfo можно комбинировать. Используется оператор И. Оба условия ДОЛЖНЫ возвращать значение true, чтобы представление отображалось.

Дочерние элементы:

  • вкладка, одна или несколько (см. Определение вкладки)

  • flatModeExceptions, необязательный элемент (см. Настройка отображения сложных элементов)

  • thesaurusList, необязательный элемент (см. Настройка тезауруса)

Определение вкладки¶

Представление содержит как минимум одну вкладку. В этом случае он будет отображаться по умолчанию, и верхняя панель инструментов не будет отображаться для переключения с одной вкладки на другую.

Добавьте пользовательский вид, одну вкладку по умолчанию и поле для заголовка:

<views>
  <view name="custom-view">
    <tab id="custom-tab" default="true">
      <section>
        <field xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>
      </section>
    </tab>
  </view>

Атрибуты:

  • id (Обязательно)

Клавиша табуляции, используемая в параметре URL для активации этой вкладки. Ключ также используется для метки вкладки, как определено в {schema}/loc/{lang}/strings.xml.

  • по умолчанию (необязательно) Фиксированное значение: true

Определите, является ли эта вкладка вкладкой по умолчанию для представления. Только одна вкладка должна быть по умолчанию в представлении.

  • toggle (необязательно) Фиксированное значение: true

Определите, должна ли вкладка отображаться в раскрывающемся меню вместо вкладки. Это используется для расширенного раздела, который не часто используется конечным пользователем. В этом раскрывающемся меню вкладок может быть сгруппировано более одной вкладки.

  • formatter-order (Optional)

Define the ordering index of this tab in the XSLT formatter (Note used for editor).

  • mode (необязательно) Фиксированное значение: flat

«Плоский» режим — важная концепция для понимания редактором. Он контролирует путь:

  • отображаются сложные элементы (т.е. элементы, имеющие дочерние элементы) и

  • отображаются несуществующие элементы (т.е. элементы в стандарте, а не в текущем документе).

Когда вкладка находится в плоском режиме, на этой вкладке не будут отображаться элементы, которых нет в текущем документе метаданных, и будет отображаться сложный элемент как группа, только если он определен в списке элементов с набором полей (см. creating-custom -editor-fieldsWithFieldset).

Пример для контакта в не &quot;плоском&quot; режиме:

Пример для контакта в &quot;плоском&quot; режиме:

Этот режим упрощает компоновку, но не предоставляет всех элементов управления для удаления некоторых элементов, обычно заключенных в рамки. Конечный пользователь по-прежнему может перейти в расширенный режим просмотра, чтобы получить доступ к этим скрытым элементам в плоском режиме.

Рекомендуется сохранить по крайней мере одно представление в не &quot;плоском&quot; режиме для рецензента или администратора, чтобы иметь возможность:

  • создавать правильные шаблоны на основе стандартов

  • для исправления любых типов ошибок.

  • mode (Mandatory)

  • displayIfRecord (необязательно)

    XPath expression returning boolean value which will be evaluated against the metadata record. if true the view will be displayed.

    eg. Display custom-view if metadata standard name contains Medsea:

    <view name="custom-view"
      displayIfRecord="contains(gmd:MD_Metadata/
                                  gmd:metadataStandardName/gco:CharacterString,
                                'MedSea')"/>

  • displayIfServiceInfo (необязательно)

    Выражение XPath, возвращающее логическое значение, которое будет оцениваться относительно информационного дерева службы (элемент Jeeves /root/gui). если true, представление будет отображаться.

    например. Отображать настраиваемый вид, если пользователь является администратором:

    <view name="custom-view"
      displayIfServiceInfo="count(session[profile = 'Administrator']) = 1"

To combine displayIfRecord and displayIfServiceInfo use an AND operator. In this case both condition MUST returned true for the view to be displayed.

Настройка отображения сложных элементов¶

Элементы для применения исключений «плоского» режима. По умолчанию в «плоском» режиме не отображаются элементы, содержащие только дочерние элементы и не имеющие значения.

Use or and in attribute to display non existing element. To display gmd:descriptiveKeywords element even if does not exist in the metadata record or if the field should be displayed to add new occurrences:

     <field
           xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:descriptiveKeywords/*/gmd:keyword"
           or="keyword"
           in="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:descriptiveKeywords/*"/>
  </tab>

  <!-- Elements that should not use the "flat" mode -->
  <flatModeExceptions>
    <for name="gmd:descriptiveKeywords" />
  </flatModeExceptions>
</view>

Настройка тезауруса¶

To configure the type of transformations or the number of keyword allowed or if the widget has to be displayed in a fieldset or as simple field for e thesaurus define a specific configuration:

например. только 2 темы INSPIRE.

<thesaurusList>
  <thesaurus key="external.theme.httpinspireeceuropaeutheme-theme"
             maxtags="2"
             fieldset="false"
             transformations=""/>
</thesaurusList>

Добавление раздела на вкладку¶

A section is a group of fields. If a name attribute is provided, then it will create an HTML fieldset which is collapsible. If no name attribute is provided, then it just render the inner elements. For example, if you need a tab without a root fieldset, just create the mandatory section with no name and then the inner elements.

Атрибуты:

  • имя (необязательно)

    Необязательное имя для переопределения имени по умолчанию на основе имени поля для раздела. Имя должно быть определено в {schema}/loc/{lang}/strings.xml.

  • xpath (необязательно)

    Xpath элемента для сопоставления. Если для раздела задан XPath, он не должен содержать никаких полей.

  • mode (необязательно) Фиксированное значение: flat

    «Плоский» режим — важная концепция для понимания редактором. Он контролирует путь:

    • отображаются сложные элементы (т.е. элементы, имеющие дочерние элементы) и

    • отображаются несуществующие элементы (т.е. элементы в стандарте, а не в текущем документе).

    Когда вкладка находится в плоском режиме, на этой вкладке не будут отображаться элементы, которых нет в текущем документе метаданных, и будет отображаться сложный элемент как группа, только если он определен в списке элементов с набором полей (см. creating-custom -editor-fieldsWithFieldset).

    Пример для контакта в не &quot;плоском&quot; режиме:

    Пример для контакта в &quot;плоском&quot; режиме:

    Этот режим упрощает компоновку, но не предоставляет всех элементов управления для удаления некоторых элементов, обычно заключенных в рамки. Конечный пользователь по-прежнему может перейти в расширенный режим просмотра, чтобы получить доступ к этим скрытым элементам в плоском режиме.

    Рекомендуется сохранить по крайней мере одно представление в не &quot;плоском&quot; режиме для рецензента или администратора, чтобы иметь возможность:

    • создавать правильные шаблоны на основе стандартов

    • для исправления любых типов ошибок.

  • mode (Mandatory)

  • или (необязательно)

    Локальное имя для сопоставления, если элемент не существует.

  • или (необязательно)

    Локальное имя дочерней геосети (т.е. несуществующий элемент) для сопоставления.

    <field xpath="/gmd:MD_Metadata/gmd:language"
       or="language"
       in="/gmd:MD_Metadata"/>

  • или (необязательно)

  • в (необязательно)

    XPath элемента geonet:child с именем или для поиска. Обычно указывает на родителя последнего элемента атрибута XPath.

  • в (необязательно)

    Элемент для поиска дочернего элемента геосети.

  • displayIfRecord (необязательно)

    XPath expression returning boolean value which will be evaluated against the metadata record. if true the view will be displayed.

    eg. Display custom-view if metadata standard name contains Medsea:

    <view name="custom-view"
      displayIfRecord="contains(gmd:MD_Metadata/
                                  gmd:metadataStandardName/gco:CharacterString,
                                'MedSea')">

Добавление поля¶

Чтобы отобразить простой элемент, используйте атрибут xpath, чтобы указать на отображаемый элемент:

<field xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>

Чтобы переопределить метку поля, используйте атрибут name и определите новую метку в {schema}/loc/{lang}/strings.xml:

<field name="myTitle"
       xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>

Чтобы отобразить сложный элемент, существующий в документе метаданных:

<field name="pointOfContact"
       xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:pointOfContact"/>

В этом случае также отображаются все дочерние элементы.

Чтобы отобразить поле, если оно существует в документе метаданных, или предоставить кнопку добавления, если оно не существует (укажите атрибуты in и or):

<field name="pointOfContact"
       xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:pointOfContact"
       or="pointOfContact"
       in="/gmd:MD_Metadata/gmd:identificationInfo/*"
       del="."/>

Активируйте «плоский» режим на уровне вкладки, чтобы форма отображала только существующие элементы:

<view name="custom-view">
    <tab id="custom-tab" default="true" mode="flat">
      <section>
        <field
                xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>
        <field name="pointOfContact"
               xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:pointOfContact"
               or="pointOfContact"
               in="/gmd:MD_Metadata/gmd:identificationInfo/*"
               del="."/>
      </section>
    </tab>
  </view>

Атрибуты:

  • xpath (Обязательно)

Xpath элемента для сопоставления.

  • если (необязательно)

Необязательное выражение xpath для оценки, чтобы определить, должен ли элемент отображаться только в некоторой ситуации (например, только для записей метаданных службы). например.

<field
  xpath="/gmd:MD_Metadata/gmd:identificationInfo/srv:SV_ServiceIdentification/
  gmd:resourceConstraints/gmd:MD_LegalConstraints/gmd:otherConstraints"
  if="count(gmd:MD_Metadata/gmd:identificationInfo/srv:SV_ServiceIdentification) > 0"/>

  • имя (необязательно)

Имя поля для переопределения имени по умолчанию.

  • isMissingLabel (необязательно)

Метка, отображаемая, если элемент не существует в записи метаданных. Это указывает на то, что элемент отсутствует в текущей записи. Его можно использовать для раздела соответствия, в котором говорится, что элемент «не оценивается». ЭКСПЕРИМЕНТАЛЬНАЯ

  • или (необязательно)

    Локальное имя дочерней геосети (т.е. несуществующий элемент) для сопоставления.

    <field xpath="/gmd:MD_Metadata/gmd:language"
       or="language"
       in="/gmd:MD_Metadata"/>

  • в (необязательно)

    Элемент для поиска дочернего элемента геосети.

  • удалить (необязательно)

    Относительный XPath удаляемого элемента при нажатии кнопки удаления.

    например. Если поле шаблона соответствует привязке и позволяет редактировать URL-адрес поля, элемент управления удалением должен удалить родительский элемент gmd:onLine.

    <field name="url"
  xpath="/gmd:MD_Metadata/gmd:distributionInfo/gmd:MD_Distribution/gmd:transferOptions
            /gmd:MD_DigitalTransferOptions/gmd:onLine/gmd:CI_OnlineResource/gmd:linkage"
  del="../..">
  <template>

    Del attribute can be used in template mode or not. Example to remove spatialResolution while only editing denominator or distance. Denominator or distance are mandatory but as the del element point to the spatialResolution ancestor, there is no mandatory flag displayed and the remove control remove the spatialResolution element.

    <field xpath="/gmd:MD_Metadata/gmd:identificationInfo/
                */gmd:spatialResolution/*/gmd:distance"
       del="../.."/>
<field xpath="/gmd:MD_Metadata/gmd:identificationInfo/
                */gmd:spatialResolution/
                  */gmd:equivalentScale/*/gmd:denominator"
       del="../../../.."/>

  • templateModeOnly (необязательно) Фиксированное значение: true

    Определите, должен ли режим шаблона быть единственным используемым режимом. В этом случае поле всегда отображается на основе конфигурации поля фрагмента шаблона XML. Значение по умолчанию — ложь.

  • notDisplayedIfMissing (необязательно) Фиксированное значение: true

    Если поле найдено, а также дочерняя геосеть, дочерняя геосеть для добавления новой не отображается.

  • use (Optional)

    The form field type to use (one of the HTML5 type) or an AngularJS directive to use. This list is defined as an open enumeration. For directive, the value will be set in a simple text input by default. If the directive needs to deal with carriage return character, the directive name MUST contains «-textarea» in order to set the value in a textarea instead of the text input.

  • use (Optional)

    Field type. Register here any Angular directive to be used on the client side. Default is simple text field.

Дочерние элементы:

  • template, необязательный элемент (см. Добавление поля на основе шаблона)

Добавление поля на основе шаблона¶

A template configuration for an XML snippet to edit.

Поле шаблона состоит из фрагмента XML, соответствующего редактируемому элементу, где редактируемые значения идентифицируются с помощью нотации {{fields}}. Каждое поле должно быть определено как значение, из которого будет создано одно поле ввода.

Этот режим используется, чтобы скрыть сложность редактируемого элемента XML. например.

<field name="url"
       templateModeOnly="true"
       xpath="/gmd:MD_Metadata/gmd:distributionInfo/g.../gmd:linkage">
   <template>
     <values>
       <key label="url"
            xpath="gmd:URL"
            tooltip="gmd:linkage"/>
     </values>
     <snippet>t
       <gmd:linkage>
         <gmd:URL>{{url}}</gmd:URL>
       </gmd:linkage>
     </snippet>
   </template>

Режим поля шаблона обеспечит редактирование только части элемента сниппета. В некоторых случаях сниппет может содержать больше элементов, чем редактируемый. В таком случае сниппет ДОЛЖЕН указывать список потенциальных элементов, чтобы не потерять информацию при использовании этого режима. Используйте элемент gn:copy, чтобы правильно объединить шаблон с текущим документом.

например. gmd:MD_Identifier может содержать узел gmd:authority, который необходимо сохранить.

<snippet>
  <gmd:identifier>
    <gmd:MD_Identifier>
      <gn:copy select="gmd:authority"/>
      <gmd:code>
        <gco:CharacterString>{{code}}</gco:CharacterString>
      </gmd:code>
    </gmd:MD_Identifier>
  </gmd:identifier>
</snippet>

Warning: Template based field does not support multilingual editing for ISO standards (ie. only the main language is edited - therefore, multilingual elements will be preserved).

Добавление документации или справки¶

Вставьте фрагмент HTML в редактор.

<field name="edmerpName"
       xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/
                gmd:pointOfContact[*/gmd:role/*/@codeListValue='edmerp']"
       del=".">

<text ref="edmerp-help"/>

Фрагмент определен в файле локализации strings.xml:

<edmerp-help>
  <div class="row">
    <div class="col-xs-offset-2 col-xs-8">
      <p class="help-block">The European Directory for Marine Environment
          Research Project (EDMERP) contains descriptions of many projects.
          This catalogue is maintained ...</p>
      </div>
  </div>
</edmerp-help>

Атрибуты:

  • ref (Обязательно)

Имя тега элемента для вставки в файл локализации.

  • если (необязательно)

An XPath expression to evaluate. If true, the text is displayed.
