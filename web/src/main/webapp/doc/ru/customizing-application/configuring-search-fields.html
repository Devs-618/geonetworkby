Настройка полей поиска¶

В некоторых случаях уместно изменить или расширить поля поиска индекса метаданных. Например, чтобы добавить поле (которое затем доступно для поиска или может использоваться в представлении по умолчанию) или изменить содержимое поля, создается из метаданных (индексация).

`Люсен<http://lucene.apache.org/java/docs/index.html> `_ — поисковая система, используемая GeoNetwork. Вся конфигурация Lucene определяется в файле WEB-INF/config-lucene.xml.

Добавить поле поиска¶

Индексированные поля определяются для каждой схемы в папке схемы (например, WEB-INF/data/config/schema_plugins/iso19139/index-fields) в файле default.xsl. Этот файл определяет для каждого критерия поиска соответствующий элемент в записи метаданных. Например, индексация заголовка записи ISO19139:

<xsl:for-each select="gmd:identificationInfo/gmd:MD_DataIdentification/
                       gmd:citation/gmd:CI_Citation/
                       gmd:title/gco:CharacterString">
    <Field name="mytitle" string="{string(.)}" store="true" index="true"/>
</xsl:for-each>

Обычно, если поле предназначено только для поиска и не должно отображаться в результатах поиска, для атрибута store может быть установлено значение false.

Как только поле добавлено в индекс, пользователь может запросить его, используя его в качестве критерия поиска в различных видах поисковых служб. Например, используя:

http://localhost:8080/geonetwork/srv/en/q?mytitle=africa

Если пользователь хочет, чтобы это поле было токенизировано, его следует добавить в раздел токенизации файла config-lucene.xml:

<tokenized>
  <Field name="mytitle"/>

Если пользователь хочет, чтобы это поле возвращалось в результатах поиска для службы поиска, то поле следует добавить в конфигурацию Lucene в разделе dumpFields:

<dumpFields>
  <field name="mytitle" tagName="mytitle"/>

Увеличение документов и полей¶

Повышение уровня документов и полей позволяет администратору каталога настраивать оценку Lucene по умолчанию для продвижения определенных типов записей.

Обычный вариант использования — когда каталог содержит множество рядов для агрегирования наборов данных. Отсутствие продвижения сериала может сделать сериал «бесполезным», даже если эти записи содержат важный контент. Увеличение этого типа документа позволяет продвигать серии и направлять конечного пользователя от серий к связанным записям (через навигацию по отношениям).

В этом случае следующая конфигурация позволяет увеличить серию и уменьшить важность записей, входящих в серию:

<boostDocument name="org.fao.geonet.kernel.search.function.ImportantDocument">
  <Param name="fields" type="java.lang.String" value="type,parentUuid"/>
  <Param name="values" type="java.lang.String" value="series,NOTNULL"/>
  <Param name="boosts" type="java.lang.String" value=".2F,-.3F"/>
</boostDocument>

Повышение является положительным или отрицательным значением с плавающей запятой.

Эта функция должна использоваться опытными пользователями для изменения оценки поведения при поиске по умолчанию в соответствии с содержимым каталога. Он нуждается в настройке и экспериментах, чтобы не слишком продвигать некоторые записи. Во время тестирования, если результаты поиска выглядят по-разному при регистрации или нет, может быть уместно игнорировать некоторые внутренние поля в вычислении повышения, которые могут изменить оценку в зависимости от текущего пользователя. Пример конфигурации:

<fieldBoosting>
  <Field name="_op0" boost="0.0F"/>
  <Field name="_op1" boost="0.0F"/>
  <Field name="_op2" boost="0.0F"/>
  <Field name="_dummy" boost="0.0F"/>
  <Field name="_isTemplate" boost="0.0F"/>
  <Field name="_owner" boost="0.0F"/>
</fieldBoosting>

Улучшение результатов поиска¶

По умолчанию Lucene вычисляет оценку в соответствии с критериями поиска, соответствующим набором результатов и содержимым индекса. В случае поиска без критериев Lucene вернет лучшие документы в порядке индекса (поскольку ни один из них не является более релевантным, чем другие).

Чтобы изменить расчет оценки, можно определить функцию повышения. Ускоряющий запрос должен быть загружен в путь к классам. Доступен пример повышения класса. RecencyBoostingQuery будет продвигать недавно измененные документы:

<boostQuery name="org.fao.geonet.kernel.search.function.RecencyBoostingQuery">
  <Param name="multiplier" type="double" value="2.0"/>
  <Param name="maxDaysAgo" type="int" value="365"/>
  <Param name="dayField" type="java.lang.String" value="_changeDate"/>
</boostQuery>
