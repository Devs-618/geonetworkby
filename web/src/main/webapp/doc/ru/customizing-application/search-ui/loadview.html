Создайте свой собственный вид¶

The search page is served by the catalog.search service.

Эта служба заканчивается файлом base-layout-cssjs-loader.xsl (см. source file web/src/main/webapp/xslt/base-layout-cssjs-loader.xsl), который содержит логику для загрузите правильную html-страницу.

Параметр представления¶

Вы можете передать параметр «представление» URL-адресу «catalog.search», чтобы указать на пользовательское представление.

Если параметр не указан, значение представления будет взято из настроек system/ui/defaultView, а если параметр не установлен, значение будет по умолчанию.

Он будет храниться в XSLT-переменной $searchView.

Загрузка представления¶

Представление загружается из (см. source file web/src/main/webapp/xslt/ui-search/search.xsl):

<div data-ng-include="'{$uiResourcesPath}views/{$searchView}/templates/index.html'"
    class="gn-full">

Основной шаблон представления загружается с помощью директивы AngularJS ng-include и указывает на путь ``/web-ui/src/main/resources/catalog/views/${view}/templates/index.html. ``.

Для представления по умолчанию вы можете найти этот шаблон в source file web-ui/src/main/resources/catalog/views/default/templates/index.html.

Загрузка javascript¶

Чтобы запустить основной модуль AngularJS, вы должны использовать директиву ng-app и указать на свой основной модуль AngularJS.

<html ng-app="{$angularModule}" lang="{$lang}" id="ng-app">

В случае со страницей поиска (catalog.search) переменная $angularApp равна gn_search. А для просмотра по умолчанию $angularModule равен gn_search_default.

Таким образом, по умолчанию основной модуль AngularJS загружается здесь из объявленного здесь модуля gn_search_default (см. source file web-ui/src/main/resources/catalog/views/default/module.js.

Отсюда вы можете построить дерево зависимостей модуля AngularJS и загрузить свое приложение.

CSS-файлы¶

По той же логике CSS, загружаемый на страницу, называется gn_search_default.less.
