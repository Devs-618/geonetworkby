Изменить конфигурацию представления по умолчанию¶

Настройки¶

In order to enable developers to specify some configuration settings, we tried as much as possible to use settings instead of hardcoded value in core code.

Эти настройки в основном хранятся в двух объектах value AngularJS:

  • gnViewerSettings

  • gnSearchSettings

Эти объекты внедряются с помощью концепции AngularJs $inject во все основные службы и директивы.

Настройки просмотра¶

Объект gnViewSettings содержит все настройки, относящиеся к странице просмотра карт.

  • defaultContext : путь к контексту OWS по умолчанию. Если он не установлен, необходимо определить объект ol.View для вашей карты.

  • bgLayers: массив слоев, которыми будет управлять селектор фонового слоя. Это значение используется только в том случае, если defaultContext не определен.

  • servicesUrl : объект конфигурации для предопределенных записей серверов WMS и WMTS.

Настройки поиска¶

Объект gnSearchSettings содержит все настройки, относящиеся к главной странице поиска.

  • olStyles : определение ol.style.Style для некоторых объектов на карте (нарисованный объект, мера, ограничивающая рамка метаданных и т. д.).

  • viewerMap : объект ol.Map, используемый для средства просмотра. Этот объект карты может быть расширен, если вы установили параметр defaultContext.

  • searchMap: объект ol.Map, используемый для страницы поиска.

  • hitsperpageValues : Массив для указания возможных значений количества метаданных на странице результатов.

  • sortbyValues : Массив для указания возможных значений для параметра сортировки на странице списка результатов.

  • resultViewTpls: шаблон, используемый для отображения результатов поиска. По умолчанию это сетка.

  • formatter : Установите параметры для просмотра метаданных.

  • linkTypes : Связь между протоколом ресурсов и действием в списке результатов метаданных (например, открытие WMS в Mapviewer).

Обновите эти настройки¶

Обратите внимание, что некоторые настройки задаются в базе данных (например, объект viewerSettings.mapConfig берется из настроек базы данных), поэтому вы можете редактировать их на странице администратора :ref:`settings<system-configuration> `.

Для других вы можете просто отредактировать файл конфигурации (см. source file web-ui/src/main/resources/catalog/views/default/config.js) и изменить некоторые значения настроек.

Важно

Если вы измените существующие файлы, вы рискуете столкнуться с проблемами слияния при обновлении новой версии. Лучший способ — создать свой собственный вид.
