Рэдактар наладкі¶

Канфігурацыя рэдактара метададзеных вызначана для пэўнага стандарту плагіна схемы (гл. Укараненне ўбудоў схемы).

Канфігурацыя рэдактара вызначыла меню навігацыі (г.зн. спіс відаў і ўкладак) для рэдактара, спіс палёў і тып элемента кіравання для выкарыстання. Элементы кіравання могуць мець тып HTML (напрыклад, тэкст, дата) або больш пашыраныя элементы кіравання з выкарыстаннем дырэктывы AngularJS<https://docs.angularjs.org/guide/directive> `_.

Каб пабудаваць такую канфігурацыю рэдактара, карыстальнік павінен ведаць XSD стандарту, каб належным чынам будаваць віды, укладкі і палі ў адпаведнасці з імёнамі элементаў (гл. source file schemas/config-editor.xsd). Стварыце каранёвы элемент рэдактара і далучыце:

  • схема і

  • прасторы імёнаў для стандартаў

<editor xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:noNamespaceSchemaLocation="../../../../../../config-editor.xsd"
  xmlns:gn="http://www.fao.org/geonetwork"
  xmlns:gco="http://www.isotc211.org/2005/gco"
  xmlns:gmd="http://www.isotc211.org/2005/gmd"
  xmlns:gmx="http://www.isotc211.org/2005/gmx"
  xmlns:srv="http://www.isotc211.org/2005/srv"
  xmlns:gml="http://www.opengis.net/gml"
  xmlns:xlink="http://www.w3.org/1999/xlink">

Канфігурацыя рэдактара павінна вызначаць спачатку нейкае агульнае апісанне элемента, а затым набор праглядаў з хаця б адным.

Даччыныя элементы:

  • поля, дадатковы элемент (гл. Вызначэнне тыпу поля)

  • fieldsWithFieldset, дадатковы элемент (гл. Элемент групоўкі са стандартаў)

  • multilingualFields, дадатковы элемент (гл. Вызначэнне шматмоўных палёў)

  • прагляды, абавязковы элемент (гл. Налада відаў)

Вызначэнне тыпу поля¶

Вызначце канфігурацыю тыпу палёў формы. Па змаўчанні просты ўвод тэксту. Гэты спіс змяшчае спіс выключэнняў, які не выкарыстоўвае просты ўвод тэксту. Спіс магчымых значэнняў:

  • усе тыпы ўводу HTML5 або

  • an AngularJS directive name. MUST starts with 'data-' and could end with '-textarea' to create a textarea element. It could end with '-div' if the directive does not apply to the input or textarea but to the div containing it.

Для элемента можа быць вызначаны толькі адзін тып.

<editor>
 <fields>
   <for name="gmd:abstract" use="textarea"/>
   <for name="gco:Real" use="number"/>
   <for name="gco:Boolean" use="checkbox"/>
   <for name="gco:Date" use="data-gn-date-picker"/>

Іншы варыянт вызначэння больш прасунутага тыпу поля - перахапіць элемент з дапамогай шаблону XSL. Такі падыход выкарыстоўваецца, напрыклад, для ключавых слоў у ISO19139 (гл. source file schemas/iso19139/src/main/plugin/iso19139/layout/layout-custom-fields-keywords.xsl).

Элемент групоўкі са стандартаў¶

Спіс элементаў для адлюстравання ў наборы палёў (г.зн. элемент у скрынцы). Гэтыя элементы звычайна ўтрымліваюць даччыныя элементы і вызначаюць асноўны раздзел у стандарце. Напрыклад, у ISO19139 ідэнтыфікацыя і размеркаванне з&#39;яўляюцца асноўным раздзелам і звычайна павінны адлюстроўвацца як група інфармацыі.

<editor>
   <fields>...</fields>
   <fieldsWithFieldset>
    <name>gmd:identificationInfo</name>
    <name>gmd:distributionInfo</name>

Вызначэнне шматмоўных палёў¶

Наладзьце тут спіс шматмоўных палёў для стандарту.

Па змаўчанні, калі стандарт падтрымлівае шматмоўнасць, напрыклад ISO19139, усе палі будуць адлюстроўвацца як шматмоўныя палі. Вызначце ў раздзеле выключэння выключэнне (напрыклад, gmd:identifier, напрыклад, у ISO19139).

Тады гэты раздзел таксама дазваляе вызначыць, як шматмоўныя палі адлюстроўваюцца з дапамогай разгорнутых элементаў. Пры разгортванні адлюстроўваецца адно поле для кожнай мовы, без неабходнасці націскаць кнопку пераключэння мовы.

<editor>
   <fields>...</fields>
   <fieldsWithFieldset>...</fieldsWithFieldset>
    <multilingualFields>
      <expanded>
        <name>gmd:title</name>
        <name>gmd:abstract</name>
      </expanded>
      <exclude>
        <name>gmd:identifier</name>
        <name>gmd:metadataStandardName</name>

Налада відаў¶

Прынамсі адзін выгляд ПАВІНЕН быць вызначаны, але ў залежнасці ад патрэб можна вызначыць больш рэжымаў прагляду.

Па змаўчанні ISO19139 вызначае 3 віды (г.зн. па змаўчанні, пашыраны, xml) і адзін адключаны (г.зн. INSPIRE). Каб даведацца пра канфігурацыю, глядзіце source file schemas/iso19139/src/main/plugin/iso19139/layout/config-editor.xml.

Іншы базавы прыклад - прагляд Dublin Core (гл. source file schemas/dublin-core/src/main/plugin/dublin-core/layout/config-editor.xml).

Каб стварыць новы выгляд, выкарыстоўвайце наступнае:

<views>
    <view name="viewNameInLocalizationFile">
      ...
    </view>

Даччыныя элементы:

  • прагляд, адзін ці некалькі (гл. Вызначэнне выгляду)

Вызначэнне выгляду¶

Прагляд мае метку і вызначае канкрэтны рэндэрынг запісаў метададзеных. Прагляд складаецца з адной або некалькіх укладак.

<views>
    <view name="custom-view">
        ....
    </view>

Выгляд можа адлюстроўвацца ці не адлюстроўвацца ў залежнасці ад змесціва запісу метададзеных або бягучага сеанса карыстальніка з дапамогай атрыбутаў displayIfRecord і displayIfServiceInfo.

Атрыбуты:

  • імя (абавязкова)

Ключ назвы прагляду, які захоўваецца ў {schema}/loc/{lang}/strings.xml, або імя элемента з прэфіксам прасторы імёнаў.

<strings>
  <default>Simple</default>
  <inspire>INSPIRE</inspire>
  <custom-view>My view</custom-view>

  • адключана (неабавязкова) Фіксаванае значэнне: ісціна

Схаваць выгляд з меню, калі атрыбут вызначаны. Дазваляе лёгка адключыць выгляд.

  • class (Optional)

Define custom CSS class to be set on the form element. This is mainly used to set the type of indent:

  • gn-label-above-input: to put label above form input

  • gn-indent-colored: colored left border on each fieldset

  • gn-indent-bluescale: blue scale colored left border on each fieldset

See catalog/views/default/less/gn_editor_default.less to add your custom editor styles.

  • upAndDownControlHidden (неабавязкова) Фіксаванае значэнне: true

Вызначце, ці павінны паказвацца элементы кіравання ўверх і ўніз у гэтым праглядзе. Калі не вызначана, элементы кіравання адлюстроўваюцца. Схавайце гэтыя элементы кіравання ў праглядзе, каб зрабіць гэта прасцей і менш элементаў кіравання для канчатковага карыстальніка.

  • displayAttributes (Optional) Fixed value: true

Display attributes by default when loading the view.

  • displayTooltips (Optional) Fixed value: true

Display help documentation for all elements by default when loading the view.

  • displayTooltipsMode (Optional)

Display help documentation onhover elements (default) or by clicking on an icon.

  • hideTimeInCalendar (Неабавязкова) Фіксаванае значэнне: true

Вызначце, ці павінен кантроль календара дазваляць карыстальнікам усталёўваць толькі дату або дату і час. Калі атрыбут не ўсталяваны, можна ўсталяваць дату і час. Гэта кантралюецца на ўзроўні прагляду, пераключэнне на іншы прагляд можа дазволіць больш кантраляваць даты.

  • displayIfRecord (неабавязкова)

Выраз XPath вяртае лагічнае значэнне, якое будзе ацэньвацца ў параўнанні з запісам метададзеных. калі ісціна, выгляд будзе адлюстроўвацца. напр. Адлюстроўваць карыстацкі выгляд, калі стандартнае імя метаданых змяшчае Medsea:

<view name="custom-view"
      displayIfRecord="contains(gmd:MD_Metadata/
                                  gmd:metadataStandardName/gco:CharacterString,
                                'MedSea')"

  • displayIfServiceInfo (неабавязкова)

Выраз XPath вяртае лагічнае значэнне, якое будзе ацэньвацца ў адпаведнасці з дрэвам службовай інфармацыі (элемент Jeeves /root/gui). калі ісціна, выгляд будзе адлюстроўвацца.

напр. Паказваць карыстальніцкі выгляд, калі карыстальнік з&#39;яўляецца адміністратарам:

<view name="custom-view"
      displayIfServiceInfo="count(session[profile = 'Administrator']) = 1"

displayIfRecord і displayIfServiceInfo можна аб&#39;яднаць. Выкарыстоўваецца аператар І. Абедзве ўмовы ПАВІННЫ вярнуць ісціну, каб прагляд быў адлюстраваны.

Даччыныя элементы:

  • ўкладка, адна ці некалькі (гл. Вызначэнне ўкладкі)

  • flatModeExceptions, дадатковы элемент (гл. Настройка адлюстравання складаных элементаў)

  • thesaurusList, дадатковы элемент (гл. Настройка тэзаўруса)

Вызначэнне ўкладкі¶

Прагляд змяшчае як мінімум адну ўкладку. У такім выпадку ён будзе адлюстроўвацца па змаўчанні, а верхняя панэль інструментаў для пераключэння з адной укладкі на іншую не будзе.

Дадайце карыстальніцкі выгляд, адну ўкладку па змаўчанні і поле для загалоўка:

<views>
  <view name="custom-view">
    <tab id="custom-tab" default="true">
      <section>
        <field xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>
      </section>
    </tab>
  </view>

Атрыбуты:

  • ідэнтыфікатар (абавязковы)

Клавіша табуляцыі, якая выкарыстоўваецца ў параметрах URL для актывацыі гэтай укладкі. Ключ таксама выкарыстоўваецца для цэтліка ўкладкі, як вызначана ў {schema}/loc/{lang}/strings.xml.

  • па змаўчанні (неабавязкова) Фіксаванае значэнне: праўда

Вызначце, ці з&#39;яўляецца гэтая ўкладка стандартнай для прагляду. Толькі адна ўкладка павінна быць стандартнай у праглядзе.

  • пераключальнік (неабавязкова) Фіксаванае значэнне: ісціна

Вызначце, ці павінна ўкладка адлюстроўвацца ў выпадальным меню замест укладкі. Гэта выкарыстоўваецца для пашыранага раздзела, які не часта выкарыстоўваецца канчатковым карыстальнікам. У гэтым выпадальным меню ўкладак можна згрупаваць больш чым адну ўкладку.

  • formatter-order (Optional)

Define the ordering index of this tab in the XSLT formatter (Note used for editor).

  • рэжым (неабавязковы) Фіксаванае значэнне: плоскі

«Плоскі» рэжым з&#39;яўляецца важным паняццем для разумення для рэдактара. Ён кантралюе спосаб:

  • складаныя элементы адлюстроўваюцца (г.зн. элементы, якія маюць дзяцей) і

  • адлюстроўваюцца неіснуючыя элементы (г. зн. элементы ў стандарце, а не ў бягучым дакуменце).

Калі ўкладка знаходзіцца ў плоскім рэжыме, гэтая ўкладка не будзе адлюстроўваць элементы, якіх няма ў бягучым дакуменце з метададзенымі, і будзе адлюстроўваць складаны элемент як групу, толькі калі ён вызначаны ў спісе элементаў з наборам палёў (гл. creating-custom -editor-fieldsWithFieldset).

Прыклад для кантакту ў не &quot;плоскім&quot; рэжыме:

Прыклад для кантакту ў «плоскім» рэжыме:

Гэты рэжым спрашчае макет, але не забяспечвае ўсе элементы кіравання для выдалення некаторых элементаў, якія звычайна знаходзяцца ў скрынках. Канчатковы карыстальнік усё яшчэ можа перайсці ў пашыраны рэжым прагляду, каб атрымаць доступ да схаваных элементаў у плоскім рэжыме.

Рэкамендуецца захаваць прынамсі адзін выгляд у не &quot;плоскім&quot; рэжыме для аглядальніка або адміністратара, каб мець магчымасць:

  • ствараць правільныя шаблоны на аснове стандартаў

  • для выпраўлення любых тыпаў памылак.

  • mode (Mandatory)

  • displayIfRecord (неабавязкова)

    XPath expression returning boolean value which will be evaluated against the metadata record. if true the view will be displayed.

    eg. Display custom-view if metadata standard name contains Medsea:

    <view name="custom-view"
      displayIfRecord="contains(gmd:MD_Metadata/
                                  gmd:metadataStandardName/gco:CharacterString,
                                'MedSea')"/>

  • displayIfServiceInfo (неабавязкова)

    Выраз XPath вяртае лагічнае значэнне, якое будзе ацэньвацца ў адпаведнасці з дрэвам службовай інфармацыі (элемент Jeeves /root/gui). калі ісціна, выгляд будзе адлюстроўвацца.

    напр. Паказваць карыстальніцкі выгляд, калі карыстальнік з&#39;яўляецца адміністратарам:

    <view name="custom-view"
      displayIfServiceInfo="count(session[profile = 'Administrator']) = 1"

To combine displayIfRecord and displayIfServiceInfo use an AND operator. In this case both condition MUST returned true for the view to be displayed.

Настройка адлюстравання складаных элементаў¶

Элементы для прымянення выключэнняў &quot;плоскага&quot; рэжыму. Па змаўчанні &quot;плоскі&quot; рэжым не адлюстроўвае элементы, якія змяшчаюць толькі даччыныя элементы і не маюць значэнняў.

Use or and in attribute to display non existing element. To display gmd:descriptiveKeywords element even if does not exist in the metadata record or if the field should be displayed to add new occurrences:

     <field
           xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:descriptiveKeywords/*/gmd:keyword"
           or="keyword"
           in="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:descriptiveKeywords/*"/>
  </tab>

  <!-- Elements that should not use the "flat" mode -->
  <flatModeExceptions>
    <for name="gmd:descriptiveKeywords" />
  </flatModeExceptions>
</view>

Настройка тэзаўруса¶

To configure the type of transformations or the number of keyword allowed or if the widget has to be displayed in a fieldset or as simple field for e thesaurus define a specific configuration:

напр. толькі 2 тэмы INSPIRE.

<thesaurusList>
  <thesaurus key="external.theme.httpinspireeceuropaeutheme-theme"
             maxtags="2"
             fieldset="false"
             transformations=""/>
</thesaurusList>

Даданне раздзела ва ўкладку¶

A section is a group of fields. If a name attribute is provided, then it will create an HTML fieldset which is collapsible. If no name attribute is provided, then it just render the inner elements. For example, if you need a tab without a root fieldset, just create the mandatory section with no name and then the inner elements.

Атрыбуты:

  • імя (неабавязкова)

    Неабавязковая назва для перавызначэння стандартнай на аснове назвы поля для раздзела. Імя павінна быць вызначана ў {schema}/loc/{lang}/strings.xml.

  • xpath (неабавязкова)

    Шлях элемента для супадзення. Калі для раздзела ўсталяваны XPath, ён не павінен утрымліваць ніякіх палёў.

    • адлюстроўваюцца неіснуючыя элементы (г. зн. элементы ў стандарце, а не ў бягучым дакуменце).

    Рэкамендуецца захаваць прынамсі адзін выгляд у не &quot;плоскім&quot; рэжыме для аглядальніка або адміністратара, каб мець магчымасць:

    • ствараць правільныя шаблоны на аснове стандартаў

    • для выпраўлення любых тыпаў памылак.

  • mode (Mandatory)

  • або (неабавязкова)

    Лакальнае імя для супадзення, калі элемент не існуе.

  • або (неабавязкова)

    Лакальнае імя даччынай геасеткі (г.зн. неіснуючага элемента), якое павінна адпавядаць.

    <field xpath="/gmd:MD_Metadata/gmd:language"
       or="language"
       in="/gmd:MD_Metadata"/>

  • або (неабавязкова)

  • у (неабавязкова)

    XPath элемента geonet:child з назвай або, якую трэба шукаць. Звычайна паказвае на бацькоўскі элемент апошняга атрыбуту XPath.

  • у (неабавязкова)

    Элемент для пошуку даччынай сеткі geonet.

  • displayIfRecord (неабавязкова)

    XPath expression returning boolean value which will be evaluated against the metadata record. if true the view will be displayed.

Даданне поля¶

Каб адлюстраваць просты элемент, выкарыстоўвайце атрыбут xpath, каб паказаць элемент для адлюстравання:

<field xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>

Каб перавызначыць метку поля, выкарыстоўвайце атрыбут name і вызначце гэтую новую метку ў {schema}/loc/{lang}/strings.xml:

<field name="myTitle"
       xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>

Каб адлюстраваць складаны элемент, які існуе ў дакуменце метададзеных:

<field name="pointOfContact"
       xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:pointOfContact"/>

У гэтым выпадку ўсе даччыныя элементы таксама адлюстроўваюцца.

Каб адлюстраваць поле, калі яно існуе ў дакуменце з метададзенымі, або прадаставіць кнопку дадання, калі яно не існуе (укажыце атрыбут in і or):

<field name="pointOfContact"
       xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:pointOfContact"
       or="pointOfContact"
       in="/gmd:MD_Metadata/gmd:identificationInfo/*"
       del="."/>

Актывуйце «плоскі» рэжым на ўзроўні ўкладак, каб форма адлюстроўвала толькі існуючыя элементы:

<view name="custom-view">
    <tab id="custom-tab" default="true" mode="flat">
      <section>
        <field
                xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:citation/*/gmd:title"/>
        <field name="pointOfContact"
               xpath="/gmd:MD_Metadata/gmd:identificationInfo/*/gmd:pointOfContact"
               or="pointOfContact"
               in="/gmd:MD_Metadata/gmd:identificationInfo/*"
               del="."/>
      </section>
    </tab>
  </view>

  • калі (неабавязкова)

Неабавязковы выраз xpath для ацэнкі, каб вызначыць, ці павінен элемент адлюстроўвацца толькі ў некаторых сітуацыях (напрыклад, толькі для запісаў метаданых службы). напр.

<field

  if="count(gmd:MD_Metadata/gmd:identificationInfo/srv:SV_ServiceIdentification) > 0"/>

Імя поля для перавызначэння імя па змаўчанні.

    <field name="url"
  xpath="/gmd:MD_Metadata/gmd:distributionInfo/gmd:MD_Distribution/gmd:transferOptions

    Калі поле знойдзена, а таксама даччыная геасетка, даччыная геасетка, якую трэба дадаць, не адлюстроўваецца.

  • use (Optional)

    The form field type to use (one of the HTML5 type) or an AngularJS directive to use. This list is defined as an open enumeration. For directive, the value will be set in a simple text input by default. If the directive needs to deal with carriage return character, the directive name MUST contains "-textarea" in order to set the value in a textarea instead of the text input.

    Field type. Register here any Angular directive to be used on the client side. Default is simple text field.

Даданне поля на аснове шаблона¶

A template configuration for an XML snippet to edit.

Поле шаблону складаецца з фрагмента XML, які адпавядае элементу для рэдагавання, дзе значэнні, якія трэба рэдагаваць, ідэнтыфікуюцца з дапамогай натацыі {{fields}}. Кожнае поле неабходна вызначыць як значэнне, з якога будзе створана адно поле ўводу.

Гэты рэжым выкарыстоўваецца, каб схаваць складанасць элемента XML для рэдагавання. напр.

<field name="url"
       templateModeOnly="true"
       xpath="/gmd:MD_Metadata/gmd:distributionInfo/g.../gmd:linkage">
   <template>
     <values>
       <key label="url"
            xpath="gmd:URL"
            tooltip="gmd:linkage"/>
     </values>
     <snippet>t
       <gmd:linkage>
         <gmd:URL>{{url}}</gmd:URL>
       </gmd:linkage>
     </snippet>
   </template>

Рэжым поля шаблону прадугледжвае рэдагаванне толькі часткі элемента фрагмента. У некаторых выпадках фрагмент можа ўтрымліваць больш элементаў, чым адрэдагаваны. У такім выпадку фрагмент ПАВІНЕН ідэнтыфікаваць спіс патэнцыйных элементаў, каб не страціць інфармацыю пры выкарыстанні гэтага рэжыму. Выкарыстоўвайце элемент gn:copy, каб правільна аб&#39;яднаць шаблон з бягучым дакументам.

напр. Gmd:MD_Identifier можа ўтрымліваць вузел gmd:authority, які неабходна захаваць.

<snippet>
  <gmd:identifier>
    <gmd:MD_Identifier>
      <gn:copy select="gmd:authority"/>
      <gmd:code>
        <gco:CharacterString>{{code}}</gco:CharacterString>
      </gmd:code>
    </gmd:MD_Identifier>
  </gmd:identifier>
</snippet>

Warning: Template based field does not support multilingual editing for ISO standards (ie. only the main language is edited - therefore, multilingual elements will be preserved).
