Стварыце свой уласны выгляд¶

The search page is served by the catalog.search service.

Гэты сэрвіс заканчваецца ў base-layout-cssjs-loader.xsl (гл. source file web/src/main/webapp/xslt/base-layout-cssjs-loader.xsl), які змяшчае логіку загрузіць правільную старонку html.

Параметр выгляду¶

Вы можаце перадаць параметр view у URL catalog.search, каб паказаць на карыстальніцкі выгляд.

Калі параметр не ўказаны, значэнне прагляду будзе ўзята з налад system/ui/defaultView, а калі налада не ўстаноўлена, гэта значэнне будзе па змаўчанні.

Ён будзе захоўвацца ў зменнай XSLT $searchView.

Загрузка выгляду¶

Выгляд загружаецца з (гл. source file web/src/main/webapp/xslt/ui-search/search.xsl):

<div data-ng-include="'{$uiResourcesPath}views/{$searchView}/templates/index.html'"
    class="gn-full">

Асноўны шаблон прагляду загружаецца праз дырэктыву AngularJS ng-include і будзе паказваць на шлях ``/web-ui/src/main/resources/catalog/views/${view}/templates/index.html ``.

Для выгляду па змаўчанні вы можаце знайсці гэты шаблон у source file web-ui/src/main/resources/catalog/views/default/templates/index.html.

Загрузка javascript¶

Каб ініцыяваць асноўны модуль AngularJS, вы павінны выкарыстоўваць дырэктыву ng-app і паказаць на вашым галоўным модулі AngularJS.

<html ng-app="{$angularModule}" lang="{$lang}" id="ng-app">

У выпадку старонкі пошуку (catalog.search), зменная $angularApp роўная gn_search. А для прагляду па змаўчанні $angularModule роўны gn_search_default.

Такім чынам, па змаўчанні асноўны модуль AngularJS загружаецца сюды з аб&#39;яўленага тут модуля gn_search_default (гл. source file web-ui/src/main/resources/catalog/views/default/module.js).

Адсюль вы можаце стварыць дрэва залежнасцей модуля AngularJS і загрузіць сваё прыкладанне.

Файлы CSS¶

Згодна з той жа логікай, CSS, які загружаецца на старонку, з&#39;яўляецца gn_search_default.less.
