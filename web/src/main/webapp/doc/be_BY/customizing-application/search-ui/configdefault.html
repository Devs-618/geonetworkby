Змяніць канфігурацыю выгляду па змаўчанні¶

Налады¶

In order to enable developers to specify some configuration settings, we tried as much as possible to use settings instead of hardcoded value in core code.

Гэтыя налады ў асноўным захоўваюцца ў 2 аб&#39;ектах значэнне AngularJS:

  • Настройкі gnViewer

  • gnSearchSettings

Гэтыя аб&#39;екты ўстаўляюцца праз канцэпцыю AngularJs $inject ва ўсе асноўныя службы і дырэктывы.

Налады праграмы прагляду¶

Аб&#39;ект gnViewSettings змяшчае ўсе налады адносна старонкі прагляду карты.

  • defaultContext: шлях да кантэксту OWS па змаўчанні. Калі не ўстаноўлена, аб&#39;ект ol.View павінен быць вызначаны для вашай карты.

  • bgLayers : масіў слаёў, якімі будзе кіраваць селектар фонавых слаёў. Гэта значэнне выкарыстоўваецца, толькі калі defaultContext не вызначаны.

  • servicesUrl: Аб&#39;ект канфігурацыі для загадзя вызначаных запісаў сервераў WMS і WMTS.

Налады пошуку¶

Аб&#39;ект gnSearchSettings змяшчае ўсе налады адносна галоўнай старонкі пошуку.

  • olStyles : ol.style.Style вызначэнне для некаторых аб&#39;ектаў на карце (намаляваны аб&#39;ект, мера, рамка аб&#39;ёму метададзеных і г.д.).

  • viewerMap : Аб&#39;ект ol.Map, які выкарыстоўваецца для прагляду. Гэты аб&#39;ект карты можа быць пашыраны, калі вы задалі параметр defaultContext.

  • searchMap : аб&#39;ект ol.Map, які выкарыстоўваецца для старонкі пошуку.

  • hitsperpageValues : масіў для ўказання магчымых значэнняў для колькасці метададзеных на старонцы вынікаў.

  • sortbyValues : масіў для ўказання магчымых значэнняў для параметру сартавання па на старонцы спісу вынікаў.

  • resultViewTpls : шаблон, які выкарыстоўваецца для адлюстравання вынікаў пошуку. Па змаўчанні гэта сетка.

  • фармататар : усталяваць параметры прагляду метададзеных.

  • linkTypes : прывязка паміж пратаколам рэсурсаў і дзеяннем у спісе вынікаў метаданых (напрыклад, адкрыць WMS у праглядальніку карт).

Абнавіце гэтыя налады¶

Звярніце ўвагу, што некаторыя налады задаюцца ў базе дадзеных (напрыклад, аб&#39;ект viewerSettings.mapConfig паходзіць з налад базы дадзеных), так што вы можаце рэдагаваць іх на старонцы адміністратара налад :ref:`<system-configuration> `.

Для іншых вы можаце проста адрэдагаваць файл канфігурацыі (гл. source file web-ui/src/main/resources/catalog/views/default/config.js) і змяніць некаторыя значэнні налад.

Important

Калі вы змяняеце існуючыя файлы, вы падвяргаеце сябе праблемам аб&#39;яднання пры абнаўленнях новай версіі. Лепшы спосаб - стварыць свой уласны погляд.
