Выкарыстанне падзей¶

Пачынаючы з GeoNetwork 3.0.x, ёсць шэраг падзей, якія вы можаце праслухоўваць у сваім кодзе Java.

Уключэнне слухачоў падзей¶

To enable this on your Maven project, you have to add the event dependencies. Edit the file custom/pom.xml and add the dependencies tag:

<dependencies>
   <dependency>
     <groupId>${project.groupId}</groupId>
     <artifactId>events</artifactId>
     <version>${project.version}</version>
   </dependency>
   <dependency>
     <groupId>${project.groupId}</groupId>
     <artifactId>core</artifactId>
     <version>${project.version}</version>
   </dependency>
</dependencies>

Затым стварыце файл custom/src/main/resources/config-spring-geonetwork.xml, каб сказаць Spring загрузіць вашы карыстальніцкія біны, дадаўшы наступнае змесціва:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:util="http://www.springframework.org/schema/util"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation=
          "http://www.springframework.org/schema/beans
           http://www.springframework.org/schema/beans/spring-beans.xsd
           http://www.springframework.org/schema/util
           http://www.springframework.org/schema/util/spring-util.xsd
           http://www.springframework.org/schema/context
           http://www.springframework.org/schema/context/spring-context-3.2.xsd">
  <bean class="org.fao.geonet.events.listeners.MyCustomListener" ></bean>
</beans>

Гэты файл павінен утрымліваць спіс усіх класаў, якія праслухоўваюць падзеі ў межах GeoNetwork.

Просты прыклад¶

Мы можам дадаць просты прыклад слухача, як гэты, які будзе друкаваць радок кожны раз, калі метаданыя выдаляюцца.

package org.fao.geonet.events.listeners;

import org.fao.geonet.domain.*;

import org.fao.geonet.events.md.MetadataRemove;

import org.springframework.context.ApplicationListener;

import org.springframework.stereotype.Component;

@Component
public class MyCustomListener implements ApplicationListener<MetadataRemove> {
   @Override
   public void onApplicationEvent(MetadataRemove event) {
      System.out.println("REMOVED");
   }
}

Напрыклад, мы можам выклікаць знешні REST API, які запускаецца кожны раз, калі метададзеныя выдаляюцца або абнаўляюцца.
