Даданне новага карыстальніцкага інтэрфейсу¶

Каб цалкам зразумець гэтую частку падручніка, было б добра спачатку азнаёміцца з падручнікам Наладзьце карыстацкі інтэрфейс GeoNetwork.

У GeoNetwork вы можаце мець больш чым адзін стыль, усталяваны ў адным разгортванні. Вы можаце выбраць, які стыль выкарыстоўваць у мэнэджару налад. Акрамя таго, вы можаце выкарыстаць параметр &quot;?view=$name&quot;, каб прымусіць пэўны стыль у запыце старонкі.

Стварыце пусты стыль¶

Now, we are going to create our custom style on the separated Maven project we created before. First we define the minimal user interface customization so we can test that our style is being generated and used.

Нам патрэбны пусты файл па шляху custom/src/main/resources/catalog/views/custom/less/gn_search_custom.less

Затым нам трэба вызначыць асноўны модуль Angular, які будзе выкарыстоўвацца ў гэтым стылі. Стварыце файл на custom/src/main/resources/catalog/views/custom/module.js і дадайце наступнае змесціва:

(  function() {
     goog.provide('gn_search_custom');
     goog.require('gn_search');
     var module = angular.module('gn_search_custom', ['gn_search']);
})();

Далей мы ствараем новы файл на custom/src/main/resources/catalog/views/custom/templates/index.html і дадаем наступны кантэнт:

<div>This is my custom GeoNetwork</div>

Нарэшце, мы павінны сказаць бібліятэцы wro4j, дзе будуць нашы файлы. Адрэдагуйце файл web-ui/src/main/resources/WEB-INF/classes/web-ui-wro-sources.xml і дадайце нашы папкі:

<require pathOnDisk="web-ui/src/main/resources" >
  <jsSource webappPath="/catalog/js/" />
  <jsSource webappPath="/catalog/views/default/" />
  <jsSource webappPath="/catalog/views/custom/" />
  <jsSource webappPath="/catalog/components/" />
  <cssSource webappPath="/catalog/views/default/" />
  <cssSource webappPath="/catalog/views/custom/" />
  <cssSource webappPath="/catalog/style/" />
  <cssSource webappPath="/catalog/lib/bootstrap-table/dist" />
</require>

Цяпер, калі мы пабудуем, разгорнем і ўвядзем GeoNetwork, мы можам выкарыстоўваць гэты новы стыль пад назвай &quot;карыстальніцкі&quot;: http://localhost:8080/geonetwork/srv/eng/catalog.search?view=custom

Імпартаваць стыль па змаўчанні¶

Вы можаце імпартаваць стыль па змаўчанні, каб лёгка пачаць свой новы стыль.

Адрэдагуйце custom/src/main/resources/catalog/views/custom/less/gn_search_custom.less і дадайце наступны радок:

@import "gn_search_default.css";

Затым зрабіце модуль Angular залежным ад стандартнага модуля інтэрфейсу Angular. Адрэдагуйце custom/src/main/resources/catalog/views/custom/module.js і заменіце змесціва на:

(  function() {
      goog.provide('gn_search_custom');
      goog.require('gn_search_default');
      var module = angular.module('gn_search_custom', ['gn_search_default']);
})();

Нарэшце, заменіце файл custom/src/main/resources/catalog/views/custom/templates/index.html зместам файла web-ui/src/main/resources/catalog/views/default/templates/index. html

You have now a duplicate of the default style you can improve with your own customizations.

Адной з галоўных пераваг аддзялення вашага кода з&#39;яўляецца тое, што вы можаце перавызначыць некаторыя файлы, якія пастаўляюцца па змаўчанні ў карыстальніцкім інтэрфейсе GeoNetwork, і гэта перавызначэнне не будзе супярэчыць будучым абнаўленням кода.

Змяніць вынікі пошуку¶

Адной з найбольш распаўсюджаных налад стылю GeoNetwork з&#39;яўляецца змяненне вонкавага выгляду спісу вынікаў пошуку. Мы можам паказаць на іншы шаблон у файле config.js. Адрэдагуйце файл custom/src/main/resources/catalog/views/custom/config.js і змяніце ўласцівасць searchSettings.resultViewTpls.

searchSettings.resultViewTpls = [{
                tplUrl: '../../catalog/views/custom/resultsview/' +
                'partials/viewtemplates/grid.html',
                tooltip: 'Grid',
                icon: 'fa-th'
              }];

І цяпер мы павінны стварыць файл web-ui/src/main/resources/catalog/views/custom/resultsview/partials/viewtemplates/grid.html і выкарыстаць патрэбны шаблон, напрыклад:

<ul class="list-group gn-resultview gn-resultview-sumup">
  <li class="list-group-item gn-grid"
    data-ng-repeat="md in searchResults.records"
    data-gn-fix-mdlinks=""
    data-gn-displayextent-onhover=""
    data-gn-zoomto-onclick="">

   <div title="{{(md.abstract || md.defaultAbstract) | striptags}}"
       data-ng-click="openRecord($index, md, searchResults.records)">
    <!-- Thumbnail -->
    <div class="gn-md-thumbnail">
      <img class="gn-img-thumbnail"
           data-ng-src="{{md.getThumbnails().list[0].url}}"
           data-ng-if="md.getThumbnails().list[0].url"/>

      <!-- Display the first metadata status (apply to ISO19139 record) -->
      <div data-ng-if="md.status_text.length > 0"
           title="{{md.status_text[0]}}"
           class="gn-status gn-status-{{md.status[0]}}">{{md.status_text[0]}}
      </div>
    </div>
   </div>
  </li>
</ul>

Мы можам вызначыць любы клас, які хочам, і спасылацца на яго ў файле gn_search_default.css, каб надаць яму стыль.
