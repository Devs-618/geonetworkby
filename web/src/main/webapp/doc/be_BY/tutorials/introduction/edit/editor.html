Наладзіць рэдактар (неабавязкова)¶

Кожная схема можа канфігураваць розныя віды рэдактара. Гэтыя канфігурацыі можна знайсці ў файле layout/config-editor.xml кожнай схемы.

Гэты файл мае чатыры асноўныя блокі:

Палі:

<!-- Form field type configuration. Default is text. -->
<fields>
<for name="gmd:abstract" use="textarea"/>
<for name="gco:Distance" use="number"/>
<for name="gmd:onLine" addDirective="data-gn-directory-entry-selector">
<directiveAttributes data-template-add-action="true" data-template-type="onLine" data-filter='{"_root": "gmd:CI_OnlineResource"}'/>
</for>
</fields>

Гэта вызначае для кожнага вызначанага тыпу XSD, які тып уводу выкарыстоўваць. Як і ў gmd:onLine, можна вызначыць наладжвальны тып вуглавога ўводу.

Пасля вызначэння ўсіх простых палёў мы вызначаем спіс набораў палёў, якія рэдактар распазнае як нешта для зборкі:

<fieldsWithFieldset>
<name>gmd:MD_Metadata</name>
<name>gmd:identificationInfo</name>
</fieldsWithFieldset>

Цяпер мы вызначым, якія палі з&#39;яўляюцца шматмоўнымі. Звычайна большасць палёў шматмоўныя, таму мы можам проста вызначыць, якія з іх выключыць.

<multilingualFields>
<!-- In multilingual mode, define which mode the widget should have. If expanded, then one field per language is displayed. -->
<expanded>
<name>gmd:title</name>
<name>gmd:abstract</name>
</expanded>
<exclude>
<name>gmd:identifier</name>
</exclude>
</multilingualFields>

Нарэшце, мы вызначаем віды, якія будуць наладжваць палі, якія будуць паказвацца пры рэдагаванні метададзеных. Мы можам вызначыць больш чым адзін від для кожнай схемы, таму карыстальнік рэдактара можа выбраць, які з іх выкарыстоўваць пры рэдагаванні метададзеных.

Калі ў праглядзе вызначана неканчатковае поле, рэдактар аўтаматычна згенеруе ўсе палі ўводу ўнутры яго. Такім чынам, вам не трэба відавочна вызначаць усе палі, якія вы хочаце паказаць. Самы просты выгляд - гэта выгляд, заснаваны на XSD схемы.

Выгляд xml таксама можна лёгка вызначыць:

<view name="xml">
<tab id="xml" default="true"/>
</view>

Калі прагляд занадта вялікі, мы таксама можам вызначыць набор укладак у рэдактары, каб не ўсе палі паказваліся адначасова. Памятайце, што калі карыстальнік змяняе ўкладку, змесціва будзе захавана. Такім чынам, мы павінны размясціць звязаныя палі ўнутры адной укладкі.

Памятайце, што вам трэба перазапускаць кантэйнер прыкладання (Tomcat) кожны раз, калі вы змяняеце гэты файл.

See more on Рэдактар наладкі.
