Падручнік па наладжванні службы загрузкі INSPIRE на аснове Atom/OpenSearch¶

У гэтым падручніку паказана, як можна наладзіць GeoNetwork для прадастаўлення паслуг спампоўкі ў адпаведнасці з тэхнічнымі рэкамендацыямі па паслугах загрузкі<http://inspire.ec.europa.eu/documents/technical-guidance-implementation-inspire-download-services> `_.

Звярніце ўвагу, што тэхнічныя інструкцыі дазваляюць як Atom/OpenSearch, так і `WFS<http://www.opengeospatial.org/standards/wfs> `_. Гэты падручнік апісвае, як наладзіць службу спампоўкі з дапамогай Atom/Opensearch.

Асновы Atom/OpenSearch палягаюць у тым, што для кожнага набору дадзеных даступны шэраг файлаў для загрузкі ў розных варыянтах (мова/праекцыя/фармат). Гэтыя густы рэкламуюцца ў дакуменце &quot;Dataset Atom Feed&quot;. Набор дакументаў &quot;Dataset Atom Feed&quot; згрупаваны ў дакумент &quot;Service Atom Feed&quot;. Для кожнага дакумента &quot;Service Atom Feed&quot; даступны дакумент OpenSearchDescription. Вэб-сайт можа спасылацца на гэты дакумент OpenSearchDescription, які дазваляе ажыццяўляць пошук у службе загрузкі з дапамогай панэлі пошуку браўзера (OpenSearch).

See:

  • Linking data using ATOM feeds

  • OpenSearch and INSPIRE ATOM

External mode implementation¶

GeoNetwork рэалізуе OpenSearch на наборы каналаў Dataset- і Service Atom. Дакумент OpenSearch Description ствараецца для кожнай стужкі Atom службы.

Неўзабаве GeoNetwork будзе падтрымліваць унутраны і знешні рэжым.

  • ва ўнутраным рэжыме GeoNetwork будзе дынамічна ствараць файлы Atom з метададзеных.

  • у вонкавым рэжыме GeoNetwork будзе прымаць файлы Atom, якія звязаны з запісамі метададзеных.

У цяперашні час цалкам падтрымліваецца толькі знешні рэжым. Файлы Atom можна ствараць у любым тэкставым або xml-рэдактары ў адпаведнасці з тэхнічнымі інструкцыямі. Для аптымізацыі падтрымкі Atom GeoNetwork выкарыстоўвае пагадненне, якое ў цяперашні час не прадугледжана тэхнічнымі рэкамендацыямі. Дамоўленасць заключаецца ў тым, што любы gmd:MD_Distribution, які мае спасылку на файл Atom, патрабуе, каб поле пратакола ўтрымлівала вызначанае значэнне, каб паказаць размеркаванне як дакумент Atom. Значэнне па змаўчанні для гэтага пратакола - INSPIRE-ATOM, але вы можаце змяніць яго ў меню Адміністратар &gt; Налады. Перавага гэтага падыходу заключаецца ў тым, што GeoNetwork не трэба адкрываць спасылку на файл, каб вызначыць, ці з&#39;яўляецца гэта файл Atom. Мы бачылі, як розныя дзяржавы-члены абавязваюць гэтую канвенцыю ў лакалізаваных тэхнічных рэкамендацыях INSPIRE.

<gmd:MD_DigitalTransferOptions>
 <gmd:onLine>
  <gmd:CI_OnlineResource>
   <gmd:linkage>
    <gmd:URL>http://www.broinspireservices.nl/atom/awp.atom</gmd:URL>
   </gmd:linkage>
   <gmd:protocol>
    <gco:CharacterString>INSPIRE-ATOM</gco:CharacterString>
   </gmd:protocol>
   <gmd:name>
    <gco:CharacterString>gdn.Aardwarmtepotentie</gco:CharacterString>
   </gmd:name>
  </gmd:CI_OnlineResource>
 </gmd:onLine>
</gmd:MD_DigitalTransferOptions>

Пры разгортванні Geonetwork пераканайцеся, што тэзаўрусы GEMET загружаны, і актывуйце рэдактар INSPIRE, як апісана ў раздзеле &quot;Канфігурацыя inspire&quot; <http://geonetwork-opensource.org/manuals/trunk/eng/users/administrator-guide/configuring-the-catalog/inspire-configuration.html> `_. У гэты файл &quot;schemas/iso19139/src/main/plugin/iso19139/loc/eng/labels.xml &quot; (і той самы файл на іншых мовах, якія выкарыстоўваюцца ў каталогу) у памочніку gmd:protocol (каля радка 2000) дадаць пратакол &quot;INSPIRE-ATOM&quot;, які выкарыстоўваецца для ўказання спасылак, якія спасылаюцца на файл Atom (будуць перададзены толькі спасылкі, якія маюць гэтае значэнне пратаколу).

У раздзеле «Адміністратар &gt; Налады» актывуйце пашырэнне INSPIRE і актывуйце збор ураджаю Atom (праз пэўны інтэрвал). На старонцы налад можна ўсталяваць пратакол, які выкарыстоўваецца ў метададзеных для ўказання спасылкі Atom.

Для кожнага набору дадзеных, які вы збіраецеся апублікаваць, стварыце запіс iso19115 з дапамогай шаблону INSPIRE. Звязаць кожны запіс з адпаведным файлам atom (або загрузіць файл atom)

Пераканайцеся, што метаданыя ўтрымліваюць той самы код gmd:, які выкарыстоўваецца ў файле Atom.

Цяпер стварыце метаданыя службы з дапамогай шаблону INSPIRE і звяжыце іх са службай Atom і асобнымі наборамі даных (з дапамогай srv:operatesOn).

Before you validate the implementation, run the Atom harvester in admin > settings manually. A panel will display the number of processed Atom feeds and any errors that may have occurred while processing (more details in log file).

Праверце рэалізацыю¶

If you are running the above setup online, you can use the INSPIRE validator. If the above setup is running locally, you can use Esdin Test Framework to validate the INSPIRE setup.

Частае выкананне тэсту падчас распрацоўкі дапамагае выявіць праблемы на ранняй стадыі.
