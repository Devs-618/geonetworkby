Managing templates¶

Templates are metadata records that you can use for describing new resources. This topic describes how to load and manage templates.

Стварэнне і кіраванне шаблонамі¶

You can:

  • Manage templates in the catalog the same way as metadata records, with a special 'template'-tag.

  • Create, update and remove templates using the contribute section.

  • Convert metadata records to templates and vice versa at any time from the metadata editor using the save as template button.

You can assign templates to limited groups, so only these groups can use the template in their work process (See Прывілеі кіравання).

Загрузка стандартных шаблонаў¶

The Metadata and Templates page in the Administration page displays all available standards.

If you have provided these in the schema plugin, using this page, users can:

  • загрузіць узоры па змаўчанні

  • загрузіць стандартныя шаблоны

Note: You have to be logged in as an administrator to access this page and function.

Імпарт шаблонаў¶

An alternate way of loading templates is to use the metadata import page where you can import XML files and selecting the type of record: template.

Стварэнне ўласных шаблонаў¶

Each standard provides default samples, but you can build your own template to make editing task as easy as possible depending on:

  • тып рэсурсаў для апісання (напрыклад, шаблон для папяровых карт)

  • структура вашай арганізацыі (напрыклад, вызначэнне шаблонаў для паслуг)

  • тып выкарыстання метададзеных (напрыклад, агульнадаступнае выкарыстанне, унутранае выкарыстанне, якасць даных)

  • тып карыстальнікаў

In a template, you should:

  • усталяваць як мага больш значэнняў па змаўчанні (напрыклад, вызначыць кантакт па змаўчанні)

  • стварайце элементы, якія рэкамендуе ваша кіраўніцтва па кадаванні (каб не марнаваць час на пошук элементаў у пашыраным праглядзе)

  • даць інструкцыі

Галоўная мэта складаецца ў тым, каб кіраваць працай рэдактара, не патрабуючы занадта шмат ведаў аб стандартным выкарыстанні.

Для далейшай налады вы можаце палепшыць убудову схемы, вызначыўшы карыстальніцкую дакументацыю, рэкамендаваныя значэнні, ... (гл. Укараненне ўбудоў схемы) або стварыўшы карыстальніцкі выгляд (гл. creating-custom- рэдактар).
