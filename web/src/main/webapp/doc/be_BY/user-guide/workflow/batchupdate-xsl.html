Абнаўленне набору запісаў з дапамогай XSLT¶

Прымяненне змяненняў¶

Карыстальнік-адміністратар можа выкарыстоўваць API каталога для абнаўлення набору запісаў з дапамогай трансфармацыі XSLT. Крокі наступныя:

  • Увайдзіце як карыстальнік адміністратара

  • Пошук запісаў для абнаўлення з дапамогай службы q, напрыклад. выбраць усе шаблоны ў пэўным стандарце: http://localhost:8080/geonetwork/srv/eng/q?_schema=iso19115-3&_isTemplate=y

  • Выберыце ўсе запісы, якія адпавядаюць пошуку з http://localhost:8080/geonetwork/srv/eng/metadata.select?selected=add-all

  • Запусціце працэс з http://localhost:8080/geonetwork/srv/eng/md.processing.batch?process=my-custom-process

  • Праверце справаздачу аб выкананні з http://localhost:8080/geonetwork/srv/eng/md.processing.batch.report

У канцы працэсу рэзюмэ апісвае:

  • Колькасць запісаў для апрацоўкі

  • Колькасць запісаў для апрацоўкі

  • Колькасць запісаў, якія не знойдзены (у выпадку, калі адзін запіс будзе выдалены пасля выбару)

  • Колькасць запісаў з памылкамі

  • Колькасць запісаў з працэсам, не вызначаным у яго стандарце

  • Колькасць запісаў, якія бягучы карыстальнік не можа рэдагаваць

Працэс таксама можа быць ужыты да аднаго запісу метададзеных з дапамогай службы md.processing напр.http://localhost:8080/geonetwork/srv/eng/md.processing?uuid=46eac9e4-33cb-45b7-a104-7bcc8e654c98&process=keywords-mapper&search=water&replace=Water

Параметры:

  • uuid або id: метаданыя UUID або ўнутраны ідэнтыфікатар метададзеных

  • працэс: Ідэнтыфікатар працэсу

  • дадаць іншыя дадатковыя параметры ў залежнасці ад працэсу (гл. ніжэй).

Даданне пакетнага працэсу¶

Стварэнне файла апрацоўкі¶

Пакетны працэс вызначаецца на аснове кожнай схемы. Каб праверыць спіс даступных працэсаў для стандартнага, праверце <datadirectory> /config/schemaPlugins/<schemaId> тэчка /process.

Тэчка process змяшчае набор XSLT. Імя файла XSLT без пашырэння выкарыстоўваецца для запуску працэсу. Напрыклад, пры выкарыстанні md.processing.batch?process=my-custom-process працэс XSLT ПАВІНЕН мець назву my-custom-process.xsl.

Апрацоўка XML запісу¶

Працэс XSLT будзе прымяняцца да кожнага запісу метададзеных у выбары. Кожны дакумент будзе мець у якасці каранёвага элемента дакумент метаданых XML з элементам geonet:info. Элемент geonet:info змяшчае метаданыя аб метададзеных. Гэты элемент ПАВІНЕН быць выдалены працэсам, каб не змяніць запіс пры захаванні ў базе дадзеных.

<gmd:MD_Metadata>
    ...
    <geonet:info xmlns:geonet="http://www.fao.org/geonetwork">
        <id>73481</id>
        <uuid>bb151890-2da5-4cfb-8659-7839e7138be7</uuid>
        <schema>iso19139</schema>
        <createDate>2015-12-23T17:05:36</createDate>
        <changeDate>2015-12-23T18:07:40</changeDate>
        <source>2cc603e1-981c-41a2-a183-39429c7dcc49</source>
        <ownerId>1</ownerId>
        <edit>true</edit>
        <owner>true</owner>
        <isPublishedToAll>false</isPublishedToAll>
        <view>true</view>
        <notify>true</notify>
        <download>true</download>
        <dynamic>true</dynamic>
        <featured>true</featured>
        <selected>true</selected>
    </geonet:info>
</gmd:MD_Metadata>

Даданне параметраў¶

Працэс XSLT можа атрымліваць параметры, якія могуць быць прадстаўлены ў URL-адрасе з дапамогай xsl:param. Напрыклад, пры выкарыстанні md.processing.batch?process=my-custom-process&amp;myParameter=test.

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform" version="2.0"
                exclude-result-prefixes="#all">

    <xsl:param name="myParameter" select="''"/>

У гэтым прыкладзе xsl:param будзе даступны як зменная з дапамогай $myParameter і будзе мець пустое значэнне, калі не ўстаноўлена параметрам URL. Каб праверыць, што параметр усталяваны належным чынам, выкарыстоўвайце xsl:message для вываду інфармацыі ў файл часопіса.

<xsl:param name="myParameter" select="''"/>
<xsl:message>myParameter: <xsl:value-of select="$myParameter"/></xsl:message>

Працэс XSLT таксама мае доступ да параметраў каталога:

  • guiLang: Бягучая мова інтэрфейсу

  • baseUrl: Базавы URL службы (напрыклад, http://localhost:8080/geonetwork)

  • catalogurl: URL каталога (напрыклад, http://localhost:8080/geonetwork/srv/eng)

  • nodeId: ідэнтыфікатар вузла (па змаўчанні srv)

Каб выкарыстоўваць адзін з гэтых параметраў у працэсе, выкарыстоўвайце xsl:param:

<xsl:param name="guiLang" select="''"/>

Зрабіць копію - мінімум¶

Працэс ПАВІНЕН як мінімум:

  • копія ўсяго

  • выдаліць метаданыя geonet:info

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
                xmlns:geonet="http://www.fao.org/geonetwork" version="2.0"
                exclude-result-prefixes="#all">

    <!-- Do a copy of every nodes and attributes recursively -->
    <xsl:template match="@*|node()">
        <xsl:copy>
            <xsl:apply-templates select="@*|node()"/>
        </xsl:copy>
    </xsl:template>

    <!-- Remove geonet:* elements. -->
    <xsl:template match="geonet:*" priority="2"/>
</xsl:stylesheet>

Затым наладзьце працэс абнаўлення запісу метададзеных.

Апрацоўка прыкладаў¶

Глядзіце прыклады source file schemas/iso19139/src/main/plugin/iso19139/process.

У дадатак да мінімуму, які працэс ПАВІНЕН рабіць, працэс можа вызначаць дадатковыя дзеянні з дапамогай новых шаблонаў:

  • Выдаленне элемента. напр. выдаленне ўсіх справаздач аб DQ_TopologicalConsistency:

<xsl:template match="gmd:report[gmd:DQ_TopologicalConsistency]"
              priority="2"/>

Усталюйце прыярытэт 2, каб ваш шаблон меў прыярытэт над асноўным шаблонам, ствараючы копію ўсяго.

Рэгістрацыя працэсу ў якасці дзеяння рэдактара¶

Працэс XSLT можна выкарыстоўваць у рэдактары для запуску пэўных дзеянняў. Напрыклад, прагляд INSPIRE адлюстроўвае кнопку для дадання ідэнтыфікатара рэсурсу, калі ні адзін не вызначае канчатак ідэнтыфікатарам метададзеных.

<action type="batch"
        process="add-resource-id"
        if="count(gmd:MD_Metadata/gmd:identificationInfo/*/
                      gmd:citation/gmd:CI_Citation/
                          gmd:identifier[
                          ends-with(
                              gmd:MD_Identifier/gmd:code/gco:CharacterString,
                              //gmd:MD_Metadata/gmd:fileIdentifier/gco:CharacterString
                          )]) = 0"/>

Глядзіце спасылку:creating-custom-editor.

Даданне пераўтварэння XSLT для імпарту¶

Дадайце пераўтварэнні XSL у тэчку web/geonetwork/xsl/conversion/import, каб даць карыстальніку новыя параметры імпарту. Файлы можна дадаваць у гэту тэчку без перазапуску праграмы.
