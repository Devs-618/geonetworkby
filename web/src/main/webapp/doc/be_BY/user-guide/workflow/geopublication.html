Публікацыя ГІС-даных на картаграфічным серверы¶

This topic describes how to publish GIS data in a map server.

Before you begin¶

  • The catalog administrator needs to configure connection to map servers in the administration interface (see Канфігурацыя картографічных сервераў для геапублікацыі).

  • The editor must reference GIS files or spatial database tables in the metadata record (see Звязванне табліцы базы дадзеных або файла ГІС у сетцы).

To publish data from the metadata editor in a remote map server:

Пры выяўленні прасторавага рэсурсу і наладцы аднаго або некалькіх карт-сервераў у рэдактары метададзеных становіцца даступны майстар геапублікацыі.

Выберыце рэсурс са спісу для адлюстравання майстра. Майстар адлюструе спіс даступных сервераў карт:

Choose the server you want to publish the data in. When selected, the catalog will check if the dataset is already published.

If found, it is added to the map. .. figure:: img/geopublication-wizard-open.png

If not, a status message indicates that the dataset is not available.

З меню кіруйце працэсам публікацыі:

  • Націсніце кнопку &quot;Праверыць&quot;, каб пераканацца, што набор даных ужо апублікаваны на выбраным серверы карт.

  • Націсніце кнопку апублікаваць, каб зарэгістраваць дадзеныя на выбраным серверы карт.

  • Націсніце кнопку адмяніць публікацыю, каб адмяніць рэгістрацыю дадзеных на выбраным серверы карт.

  • Каб дадаць спасылкі на службу OGC у бягучым запісе метададзеных для апублікаванага ўзроўню, націсніце кнопку «Звязаць службу з метададзенымі».

Next steps¶

Once you have registered a WMS layer in a metadata record, you can generate an overview using the service (see Стварэнне мініяцюры з дапамогай слаёў WMS).
