ТРАЦІ Уборка ўраджаю¶

Каталогі THREDDS апісваюць спісы набораў даных. Яны арганізаваны ў іерархічнай форме з пералікам апісальнай інфармацыі і метадаў доступу для кожнага набору даных. Звычайна яны каталогізуюць наборы даных netCDF, але не абмяжоўваюцца гэтымі тыпамі файлаў. Гэты тып збору скануе метаданыя збору каталога THREDDS для набораў даных і сэрвісаў, апісаных у ім або ў спасылачных наборах даных netCDF. Гэты тып збору можа здабываць фрагменты метададзеных з каталога THREDDS, што дазваляе карыстальніку звязваць або капіяваць гэтыя фрагменты ў шаблон для стварэння запісаў метададзеных.

Даданне каталожнага камбайна THREDDS¶

Даступныя варыянты:

  • Сайт

    • Назва - гэта кароткае апісанне каталога THREDDS. Ён будзе паказаны на галоўнай старонцы збору ўраджаю як назва гэтага асобніка камбайна THREDDS.

    • URL-адрас каталога - аддалены URL-адрас каталога THREDDS, з якога будуць збірацца метаданыя. Гэта павінна быць xml-версія каталога (г.зн. заканчвацца на .xml). Камбайн будзе праглядаць усе наборы даных і сэрвісы, вызначаныя ў гэтым каталогу, ствараючы для іх метаданыя ў адпаведнасці з параметрамі, апісанымі ніжэй.

    • Мова метададзеных - выкарыстоўвайце гэты параметр, каб вызначыць мову метаданых, якія будуць сабраны.

    • Катэгорыя тэмы ISO - выкарыстоўвайце гэты параметр, каб вызначыць катэгорыю тэмы ISO метаданых службы.

    • Стварыць метададзеныя ISO19119 для ўсіх сэрвісаў у каталогу - Выберыце гэты параметр, каб стварыць метададзеныя ISO19119 для сэрвісаў, вызначаных у каталогу THREDDS (напрыклад, OpenDAP, OGC WCS, ftp), а таксама для самога каталога THREDDS.

    • Стварыць метаданыя для калекцыйных набораў даных - Выберыце гэты параметр, каб стварыць метададзеныя для кожнага калекцыйнага набору даных (набор даных THREDDS, які змяшчае іншыя наборы даных). Стварэнне метададзеных можна наладзіць з дапамогай параметраў, якія адлюстроўваюцца пры выбары гэтага параметра, як апісана ніжэй.

    • Стварыць метададзеныя для атамарных набораў даных - Выберыце гэты параметр, каб стварыць метададзеныя для кожнага атамарнага набору даных (набор даных THREDDS, які не змяшчае іншых набораў даных – напрыклад, каталагізаваць набор даных netCDF). Стварэнне метададзеных можна наладзіць з дапамогай параметраў, якія адлюстроўваюцца пры выбары гэтага параметра, як апісана ніжэй.

      • Ігнараваць атрыбут збору - Выберыце гэты параметр для збору метададзеных для выбраных набораў даных незалежна ад атрыбуту збору для набору даных у каталогу THREDDS. Калі гэты параметр не выбраны, метаданыя будуць стварацца толькі для набораў даных, у якіх для атрыбута збору ўстаноўлена значэнне true.

      • Выманне элементаў метададзеных DIF і стварэнне метаданых ISO - Выберыце гэты параметр, каб стварыць метаданыя ISO для набораў даных у каталогу THREDDS, якія маюць элементы метададзеных DIF. Калі выбрана гэтая опцыя, паказваецца спіс схем, якія маюць табліцу стыляў DIFToISO.xsl (гл., напрыклад, GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/DIFToISO.xsl). Метададзеныя ствараюцца шляхам счытвання элементаў метададзеных DIF у THREDDS у запіс метададзеных фармату DIF і затым пераўтварэння гэтага запісу DIF у ISO з дапамогай табліцы стыляў DIFToISO.

      • Выманне метаданых выяўлення набору даных Unidata з выкарыстаннем фрагментаў - выберыце гэты параметр, калі метаданыя ў вашых наборах даных THREDDS або netCDF/ncml адпавядаюць канвенцыям выяўлення набору даных Unidata (гл. http://www.unidata.ucar.edu/software/netcdf-java/formats /DataDiscoveryAttConvention.html). Вам трэба будзе напісаць свае ўласныя табліцы стыляў, каб атрымаць гэтыя метададзеныя ў выглядзе фрагментаў і вызначыць шаблон для аб&#39;яднання з фрагментамі. Калі гэты параметр выбраны, будуць паказаны наступныя дадатковыя параметры:

        • Выберыце схему для выходных запісаў метададзеных - выберыце схему метададзеных ISO або профіль для сабраных запісаў метададзеных. Заўвага: у спісе будуць паказаны толькі схемы, якія маюць табліцы стыляў фрагментаў THREDDS (размяшчэнне гэтых табліц стыляў глядзіце ў наступнай опцыі).

        • Табліца стыляў для стварэння фрагментаў метададзеных - Выберыце табліцу стыляў для выкарыстання для пераўтварэння метададзеных для набору даных (метададзеных THREDDS і netCDF ncml, калі дастасавальна) у фрагменты метададзеных. Гэтыя табліцы стыляў можна знайсці ў каталогу convert/ThreddsToFragments у каталогу схем, напрыклад. для iso19139 гэта будзе GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/ThreddsToFragments.

        • Стварыць падшаблоны для фрагментаў і XLink іх у шаблон - Выберыце гэты параметр, каб стварыць падшаблон (=фрагмент метададзеных, які захоўваецца ў каталогу GeoNetwork) для кожнага створанага фрагмента метададзеных.

        • Шаблон для спалучэння з фрагментамі - Выберыце шаблон, які будзе запоўнены фрагментамі метададзеных, створанымі для кожнага набору даных. Згенераваныя фрагменты метададзеных выкарыстоўваюцца для замены элементаў у шаблонах, на якія спасылаюцца, на спасылку x на падшаблон, калі пазначана опцыя Стварыць падшаблоны. Калі Стварыць падшаблоны не адзначана, то фрагменты проста капіююцца ў запіс метададзеных шаблону.

        • Для Atomic Datasets прадастаўляецца адна дадатковая опцыя Збірайце толькі новыя або змененыя наборы даных. Калі гэты параметр адзначаны, будуць збірацца толькі наборы даных, якія былі зменены або не існавалі на момант апошняга запуску камбайна.

    • Стварыць мініяцюры - Выберыце гэты параметр, каб стварыць мініяцюры для слаёў WMS у спасылачных службах WMS

    • Значок - значок для прызначэння сабраных метададзеных. Значок будзе выкарыстоўвацца пры паказе вынікаў пошуку.

  • Параметры - Параметры раскладу.

  • Прывілеі

  • Катэгорыя паслугі - Выберыце катэгорыю для прызначэння запісаў паслуг ISO19119 для паслуг THREDDS.

  • Катэгорыя для набораў даных - Выберыце катэгорыю для прызначэння згенераваных запісаў метададзеных (і любых падшаблонаў).

Унізе старонкі ёсць наступныя кнопкі:

  • Назад - Вярнуцца да галоўнай старонкі збору ўраджаю. Вызначэнне нарыхтоўкі не дадаецца.

  • Захаваць - Захоўвае гэтае вызначэнне камбайна, ствараючы новы асобнік збору ўраджаю. Пасля завяршэння аперацыі захавання будзе адлюстравана галоўная старонка збору ўраджаю.

Больш падрабязна аб зборы элементаў метададзеных THREDDS DIF з дапамогай THREDDS Harvester¶

Каталогі THREDDS могуць уключаць элементы са стандарту метададзеных DIF. Бібліятэка Unidata netcdf-java забяспечвае працэс DIFWriter, які можа ствараць запіс метададзеных DIF з гэтых элементаў. GeoNetwork мае табліцу стыляў DIFToISO для пераўтварэння гэтых запісаў DIF у ISO.

Дадатковая інфармацыя аб зборы метаданых выяўлення набору даных Unidata з дапамогай THREDDS Harvester¶

Параметры, апісаныя вышэй для Выняць метаданыя выяўлення набору даных Unidata з выкарыстаннем фрагментаў (гл. http://www.unidata.ucar.edu/software/netcdf-java/formats/DataDiscoveryAttConvention.html для больш падрабязнай інфармацыі аб гэтых пагадненнях) выклікаюць наступнае працэс для кожнага набору даных збору або атамнага набору даных у каталогу THREDDS:

  1. Камбайн аб&#39;ядноўвае URI каталога, згенераваны uuid, метададзеныя THREDDS для набору даных (створаныя з выкарыстаннем вэб-службы паднабору каталогаў) і ncml для набораў даных netCDF у адзін дакумент xml. Прыклад паказаны ніжэй.

  2. Затым гэты дакумент трансфармуецца з выкарыстаннем названай табліцы стыляў (гл. параметр Табліца стыляў вышэй), каб атрымаць дакумент з фрагментамі метададзеных.

  3. Затым выклікаецца зборнік фрагментаў метададзеных для стварэння падшаблонаў і/або метададзеных для кожнага набору даных па запыце

Прыклад¶

Элементы метададзеных DIF у наборах даных у каталогах THREDDS не так шырока выкарыстоўваюцца, як элементы метададзеных, якія прытрымліваюцца пагадненняў аб метададзеных адкрыцця набору даных Unidata. У гэтым прыкладзе будзе паказана, як сабраць элементы метададзеных, якія адпавядаюць правілам выяўлення дадзеных Unidata (гл. http://www.unidata.ucar.edu/software/netcdf-java/formats/DataDiscoveryAttConvention.html).

Дзве эталонныя табліцы стыляў прадстаўлены ў якасці прыкладаў таго, як сабраць фрагменты метададзеных з каталога THREDDS. Адна з гэтых табліц стыляў, thredds-metadata.xsl, прызначана для генерацыі фрагментаў метададзеных iso19139 з метададзеных THREDDS у адпаведнасці з правіламі выяўлення набору даных Unidata. Іншая табліца стыляў, netcdf-attributes.xsl, прызначана для генерацыі фрагментаў iso19139 з набораў даных netCDF у адпаведнасці з пагадненнямі аб выяўленні набораў даных Unidata. Гэтыя табліцы стыляў прызначаныя для выкарыстання з шаблонам «ШАБЛОН ЗБОРУ — ТРАТЫ — ВЫЯЎЛЕННЕ ДАННЫХ» і могуць быць знойдзены ў каталогу «пераўтварэнне» схемы, напрыклад. для ISO19139 гэта GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/ThreddsToFragments.

Узор шаблона «ШАБЛОН НАБОРКІ – ТРЭДЫ – ВЫЯЎЛЕННЕ ДАДЗЕНЫХ» быў прадстаўлены для выкарыстання з табліцамі стыляў, апісанымі вышэй для схемы метададзеных iso19139. Гэты шаблон знаходзіцца ў каталогу «шаблоны» схемы, напрыклад. для ISO19139 гэта GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/thredds-harvester-unidata-data-discovery.xml. Перад спробай запусціць гэты прыклад, вы павінны пераканацца, што гэты шаблон і іншыя са схемы iso19139 былі загружаны ў GeoNetwork з дапамогай функцыі «Дадаць шаблоны» ў меню адміністравання.

Зараз мы прывядзем прыклад таго, як наладзіць камбайн і збіраць метаданыя THREDDS з аднаго з агульнадаступных каталогаў unidata motherlode на http://motherlode.ucar.edu:8080/thredds/catalog/satellite/3.9/WEST-CONUS_4km /каталог.xml. Калі вы ўставіце гэты URL у свой браўзер, вы ўбачыце XML-прадстаўленне гэтага каталога THREDDS. Гэта дакумент, які чытаецца і пераўтвараецца ў метададзеныя камбайнам THREDDS.

У GeoNetwork перайдзіце ў меню «Адміністраванне», абярыце «Кіраванне ўборкай», як апісана раней. Дадайце камбайн у каталог THREDDS. Запоўніце форму кіравання ўборкай ураджаю.

Першае, на што варта звярнуць увагу, гэта тое, што URL-адрас службы павінен быць http://motherlode.ucar.edu:8080/thredds/catalog/satellite/3.9/WEST-CONUS_4km/catalog.xml. Пераканайцеся, што вы выкарыстоўваеце xml-версію каталога. Калі вы выкарыстоўваеце HTML-версію, вы не зможаце сабраць метададзеныя.

Цяпер, паколькі гэты каталог THREDDS з унідатай мае шмат набораў даных на ўзроўні файлаў (на самай справе шмат тысяч), мы будзем збіраць толькі метаданыя калекцыі. Каб зрабіць гэта, вы павінны адзначыць Стварыць метададзеныя для збору набораў даных і ігнараваць атамарныя наборы даных.

Далей, таму што метаданыя ў гэтым каталогу адпавядаюць канвенцыям выяўлення даных Unidata, таму мы абярэм Выняць метаданыя выяўлення набору даных Unidata з дапамогай фрагментаў.

Далей мы праверым Ігнараваць атрыбут збору ўраджаю. Мы робім гэта таму, што наборы даных у каталогу THREDDS могуць мець атрыбут, які паказвае, трэба збіраць набор даных ці не. Паколькі ні адзін з набораў даных у гэтым каталогу не мае атрыбута збору ўраджаю, мы будзем яго ігнараваць. Калі б мы не адзначылі гэты сцяжок, усе наборы даных былі б прапушчаны.

Далей мы абярэм схему метададзеных, у якую будуць запісаны сабраныя метададзеныя. Тут мы абярэм iso19139, таму што гэта схема, для якой у нас ёсць табліцы стыляў, якія будуць пераўтвараць метададзеныя THREDDS у фрагменты метададзеных iso19139 і шаблон, у які гэтыя фрагменты метададзеных можна скапіяваць або звязаць. Пасля выбару iso19139 з&#39;явяцца варыянты, якія паказваюць гэтыя табліцы стыляў і шаблоны.

Першы выбар - гэта табліца стыляў, якая будзе ствараць фрагменты метададзеных iso19139. Паколькі нас цікавяць элементы метададзеных thredds у каталогу THREDDS, мы абярэм (iso19139) thredds-metadata (размешчаны ў GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/ThreddsToFragments), каб пераўтварыць гэтыя элементы ў фрагменты метададзеных iso19139.

Для мэт гэтай дэманстрацыі мы не будзем правяраць Ствараць падшаблоны для фрагментаў (xlinks...). Гэта азначае, што фрагменты метададзеных, створаных табліцай стыляў, будуць скапіяваны непасрэдна ў шаблон метададзеных. Яны не змогуць быць выкарыстаны паўторна (напрыклад, сумесна паміж рознымі запісамі метададзеных). Глядзіце папярэдні раздзел аб фрагментах метададзеных, калі вы не ўпэўненыя, што гэта значыць.

Нарэшце, мы абярэм ШАБЛОН НАБОРКІ - ТРЭДЫ - ВЫЯЎЛЕННЕ UNIDATA у якасці запісу метаданых шаблону, які будзе аб&#39;яднаны з фрагментамі метададзеных для стварэння выходных запісаў. Гэты шаблон будзе загружаны ў GeoNetwork з GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/thredds-harvester-unidata-data-discovery.xml праз функцыю «Дадаць шаблоны» ў інтэрфейсе адміністравання. Гэты шаблон можа быць запоўнены метададзенымі, агульнымі для ўсіх запісаў, перш чым запусціць камбайн. Працэс, з дапамогай якога шаблон выкарыстоўваецца для стварэння запісаў метададзеных, выглядае наступным чынам:

  1. Для кожнага набору даных у каталогу THREDDS шаблон будзе скапіраваны для стварэння новага запісу метададзеных iso19139

  2. Кожны фрагмент метададзеных, сабраны з набору даных THREDDS, будзе скапіяваны ў новы запіс метададзеных iso19139 шляхам супастаўлення ідэнтыфікатара ў шаблоне з ідэнтыфікатарам у фрагменце (гэта адпаведнасць ствараецца распрацоўшчыкам шаблону і табліцы стыляў).

  3. Затым новы запіс устаўляецца ў каталог метададзеных GeoNetwork, а змест запісу індэксуецца ў Lucene для пошуку.

Затым вы можаце запоўніць астатнюю частку формы ў адпаведнасці з тым, як часта вы хочаце абнаўляць сабраныя метаданыя, якія катэгорыі будуць прысвоены створанаму запісу метададзеных, які значок будзе адлюстроўвацца разам з запісамі метададзеных у выніках пошуку і што будуць прывілеі на створаныя запісы метададзеных.

Захавайце экран камбайна. Затым на экране кіравання зборам ураджаю пастаўце галачку побач з толькі што створаным камбайнам, Актывуйце яго, а затым Запусціце. Праз некалькі імгненняў (у залежнасці ад падключэння да Інтэрнэту і прылады) вы павінны націснуць Абнавіць. Калі ваш ураджай быў паспяховым, вы павінны ўбачыць панэль вынікаў прыкладна накшталт той, што паказана на наступным скрыншоце.

Звярніце ўвагу, што ў гэтым каталогу THREDDS было створана 48 запісаў метададзеных для 48 набораў даных на ўзроўні збору. Кожны запіс метададзеных быў сфарміраваны шляхам дублявання шаблону метададзеных і наступнага капіравання ў яго 13 фрагментаў метададзеных - такім чынам, у агульнай складанасці сабрана 624 фрагменты.

Прыклад аднаго з запісаў метададзеных узроўню калекцыі, створаных камбайнам у гэтым прыкладзе і візуалізаваных GeoNetwork, паказаны ніжэй.
