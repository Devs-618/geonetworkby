Збор лакальнай файлавай сістэмы¶

Гэты камбайн будзе збіраць метаданыя ў выглядзе файлаў XML з файлавай сістэмы, даступнай на машыне, на якой працуе сервер GeoNetwork.

Даданне харвестара лакальнай файлавай сістэмы¶

На малюнку вышэй паказаны даступныя варыянты:

  • Сайт - Параметры аддаленага сайта.

    • Назва - гэта кароткае апісанне харвестара файлавай сістэмы. Ён будзе паказаны на галоўнай старонцы збору ўраджаю як назва для гэтага асобніка харвестара лакальнай файлавай сістэмы.

    • Каталог - шлях да каталога, які змяшчае метаданыя (у выглядзе файлаў XML), якія трэба сабраць.

    • Recurse - Калі пазначана і шлях Directory змяшчае іншыя каталогі, то камбайн будзе праглядаць усё дрэва файлавай сістэмы ў гэтым каталогу і дадаваць усе знойдзеныя файлы метададзеных.

    • Захоўваць лакальна пры выдаленні ў крыніцы - калі пазначана, то запісы метададзеных, якія ўжо былі сабраны, будуць захоўвацца, нават калі яны былі выдалены з указанага Каталога.

    • Значок - значок для прызначэння сабраных метададзеных. Значок будзе выкарыстоўвацца пры паказе сабраных запісаў метададзеных у выніках пошуку.

  • Параметры - параметры раскладу.

  • Сабранае змесціва - параметры, якія прымяняюцца да сабранага змесціва.

    • Ужыць гэты XSLT да сабраных запісаў - Выберыце тут XSLT, які пераўтворыць сабраныя запісы ў іншы фармат.

    • Праверыць - калі пазначана, метаданыя будуць правераны пасля атрымання. Калі праверка не праходзіць, метаданыя будуць прапушчаны.

  • Прывілеі - прызначыць прывілеі для сабраных метададзеных.

Заўвагі¶

  • каб быць паспяхова сабраны, запісы метададзеных, атрыманыя з файлавай сістэмы, павінны адпавядаць схеме метададзеных у лакальным асобніку GeoNetwork
