Паслугі OGC па ўборцы ўраджаю¶

Сэрвіс OGC рэалізуе аперацыю GetCapabilities, якую GeoNetwork, выступаючы ў якасці кліента, можа выкарыстоўваць для атрымання метададзеных для сэрвісу (ISO19119) і рэсурсаў, якія прадстаўляюцца сэрвісам (ISO19115/19139). Гэты камбайн падтрымлівае наступныя службы і версіі OGC:

  • Web Map Service (WMS) - версіі 1.0.0, 1.1.1, 1.3.0

  • Web Feature Service (WFS) - версіі 1.0.0 і 1.1.0

  • Служба вэб-пакрыцця (WCS) - версія 1.0.0

  • Сэрвіс вэб-апрацоўкі (WPS) - версіі 0.4.0 і 1.0.0

  • Сэрвісы каталогаў для Інтэрнэту (CSW) - версія 2.0.2

  • Служба датчыкаў назірання (SOS) - версія 1.0.0

Даданне OGC Service Harvester¶

Параметры канфігурацыі:

  • Сайт

    • Назва - назва каталога і будзе адным з крытэрыяў пошуку.

    • Тып - Тып службы OGC паказвае, ці павінен камбайн запытваць пэўны від паслугі. Падтрымліваюцца тыпы WMS (1.0.0, 1.1.1, 1.3.0), WFS (1.0.0 і 1.1.0), WCS (1.0.0), WPS (0.4.0 і 1.0.0), CSW (2.0) .2) і SOS (1.0.0).

    • URL-адрас службы - URL-адрас службы - гэта URL-адрас службы, з якой трэба звязацца (без такіх параметраў, як &quot;REQUEST=GetCapabilities&quot;, &quot;VERSION=&quot;, ...). Гэта павінен быць сапраўдны URL, напрыклад http://your.preferred.ogcservice/type_wms.

    • Мова метададзеных - абавязковае поле, якое будзе вызначаць мову метададзеных. Гэта павінна быць мова, якую выкарыстоўвае адміністратар вэб-службы OGC.

    • Катэгорыя тэмы ISO - выкарыстоўваецца для запаўнення элемента катэгорыі тэмы ў метададзеных. Рэкамендуецца выбраць адну, паколькі катэгорыя тэмы з&#39;яўляецца абавязковай для стандарту ISO19115/19139, калі іерархічны ўзровень - &quot;наборы даных&quot;.

    • Тып імпарту - па змаўчанні камбайн стварае адзін запіс метаданых службы. Сцяжкі ў гэтай групе вызначаюць іншыя метададзеныя, якія будуць створаны.

      • Стварэнне метададзеных для элементаў пласта з выкарыстаннем інфармацыі GetCapabilities: выбар гэтай опцыі азначае, што камбайн будзе перабіраць наборы даных, якія абслугоўвае служба, як апісана ў дакуменце GetCapabilities.

      • Create metadata for layer elements using MetadataURL attributes: Checkthis option means that the harvester will generate metadata from an XML document referenced in the MetadataUrl attribute of the dataset in the GetCapabilities document. If the document referred to by this attribute is not valid (eg. unknown schema, bad XML format), the GetCapabilities document is used as per the previous option.

      • Стварыць мініяцюры для слаёў WMS: Калі вы збіраеце з OGC WMS, галачка гэтай опцыі азначае, што эскізы будуць стварацца падчас збору.

    • Мэтавая схема - схема метаданых запісаў метаданых набору даных, якія будуць створаны гэтым камбайнам.

    • Значок - значок па змаўчанні, які адлюстроўваецца ў якасці лагатыпа атрыбуцыі для метададзеных, створаных гэтым камбайнам.

  • Параметры - Параметры раскладу.

  • Прывілеі

  • Катэгорыя для службы - Метададзеныя для сабранай службы прысвойваюцца катэгорыі, выбранай у гэтай опцыі (напрыклад, &quot;інтэрактыўныя рэсурсы&quot;).

  • Катэгорыя для набораў даных - Метададзеныя для сабраных набораў даных прысвойваюцца катэгорыі, выбранай у гэтай опцыі (напрыклад, «наборы даных»).

Заўвагі¶

  • кожны раз, калі камбайн працуе, ён будзе выдаляць раней сабраныя запісы і ствараць новыя запісы. GeoNetwork будзе генераваць uuid для ўсіх метададзеных (як сэрвісаў, так і набораў даных). Выключэннем з гэтага правіла з&#39;яўляюцца метаданыя набору даных, створаныя з выкарыстаннем тэга MetadataUrl, у дакуменце GetCapabilities, у гэтым выпадку замест гэтага выкарыстоўваецца uuid аддаленага дакумента XML

  • мініяцюры могуць стварацца толькі пры зборы OGC Web Map Service (WMS). WMS павінен падтрымліваць праекцыю WGS84

  • абраная схема Target павінна мець падтрымку XSLT, якія выкарыстоўваюцца камбайнам для пераўтварэння аператара GetCapabilities у запісы метададзеных з гэтай схемы. Калі вы сумняваецеся, выкарыстоўвайце iso19139.
