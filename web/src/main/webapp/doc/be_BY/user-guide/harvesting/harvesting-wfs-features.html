Збор WFS GetFeature¶

Метададзеныя могуць прысутнічаць у табліцах рэляцыйных баз даных, якія звычайна выкарыстоўваюцца многімі арганізацыямі. Размяшчэнне OGC Web Feature Service (WFS) над рэляцыйнай базай дадзеных дазволіць здабываць метададзеныя з дапамогай стандартных механізмаў запытаў. Гэты тып збору дазваляе карыстальніку ўказваць запыт GetFeature і адлюстроўваць інфармацыю з аб&#39;ектаў на фрагменты метададзеных, якія можна звязаць або скапіяваць у шаблон для стварэння запісаў метададзеных.

Даданне OGC WFS GetFeature Harvester¶

Служба вэб-функцый OGC (WFS) рэалізуе аперацыю запыту GetFeature, якая вяртае даныя ў выглядзе функцый (звычайна радкі з звязаных табліц у рэляцыйнай базе даных). GeoNetwork, дзейнічаючы як кліент, можа прачытаць адказ GetFeature і прымяніць прадастаўленую карыстальнікам табліцу стыляў XSLT для стварэння фрагментаў метададзеных, якія можна звязаць або скапіяваць у прадастаўлены карыстальнікам шаблон для стварэння запісаў метададзеных.

Даступныя варыянты:

  • Сайт

  • Назва - гэта кароткае апісанне камбайна. Ён будзе паказаны на галоўнай старонцы збору ўраджаю як назва гэтага камбайна WFS GetFeature.

  • URL службы - голы URL службы WFS (параметры OGC не патрабуюцца)

  • Мова метададзеных - мова, якая будзе выкарыстоўвацца ў запісах метададзеных, створаных камбайнам

  • OGC WFS GetFeature Query - запыт OGC WFS GetFeature, які выкарыстоўваецца для здабывання функцый з WFS.

  • Схема для выходных запісаў метададзеных - выберыце схему метададзеных або профіль для сабраных запісаў метададзеных. Заўвага: у спісе будуць паказаны толькі схемы, якія маюць табліцы стыляў фрагментаў WFS (размяшчэнне гэтых табліц стыляў глядзіце ў наступнай опцыі).

    • Табліца стыляў для стварэння фрагментаў - прадастаўленая карыстальнікам табліца стыляў, якая трансфармуе адказ GetFeature у дакумент з фрагментамі метададзеных (фармат гэтага дакумента глядзіце ніжэй). Табліцы стыляў існуюць у каталогу WFSToFragments, які знаходзіцца ў каталогу канвертавання абранай схемы вываду. напр. для схемы iso19139 гэты каталог - GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/WFSToFragments.

    • Захаваць вялікі адказ на дыск - пастаўце галачку, калі вы чакаеце, што адказ WFS GetFeature будзе вялікім (напрыклад, большым за 10 МБ). Калі пазначана, адказ GetFeature будзе захаваны на дыск у часовым файле. Затым кожная функцыя будзе вынята з часовага файла і выкарыстана для стварэння фрагментаў і запісаў метададзеных. Калі не адзначыць, адказ будзе захоўвацца ў аператыўнай памяці.

    • Стварыць падшаблоны - пастаўце галачку, калі вы жадаеце, каб сабраныя фрагменты метададзеных захоўваліся як падшаблоны ў каталогу метададзеных і хлінкаваліся ў шаблон метададзеных (гл. наступны варыянт). Калі не пазначана, фрагменты будуць скапіяваны ў шаблон метададзеных.

    • Шаблон для стварэння метададзеных з выкарыстаннем фрагментаў - Выберыце шаблон метададзеных, які будзе спалучацца з сабранымі фрагментамі метададзеных для стварэння запісаў метададзеных. Гэта стандартны запіс шаблону метададзеных GeoNetwork.

  • Катэгорыя для запісаў, створаных са звязанымі фрагментамі - Выберыце шаблон метададзеных, які будзе аб&#39;яднаны з сабранымі фрагментамі метададзеных для стварэння запісаў метададзеных. Гэта стандартны запіс шаблону метададзеных GeoNetwork.

  • Параметры

  • Прывілеі

  • Катэгорыя для падшаблонаў - калі фрагменты захоўваюцца ў GeoNetwork як падшаблоны, яны будуць аднесены да выбранай тут катэгорыі.

Больш падрабязна пра пераўтварэнне адказу GetFeature Response у фрагменты метададзеных¶

У корані<record> элемент можа быць ад нуля да многіх<record> элементы. Пры генерацыі метададзеных кожны элемент запісу прывядзе да генерацыі аднаго дакумента метададзеных, у адваротным выпадку<record> элемент выкарыстоўваецца для групоўкі фрагментаў метададзеных толькі па меры неабходнасці (напрыклад, фрагменты, створаныя для набору даных або аб&#39;екта).

У межах а<record> элемент можа быць ад нуля да многіх<fragment> элементаў і ад нуля да многіх<replacementGroup> элементы. А<replacementGroup> сам элемент можа ўтрымліваць ад нуля да многіх<fragment> элементы. Упарадкаванне<fragment> элементы і<replacementGroup> элементы ў межах a<record> або<replacementGroup> элемент не важны.

<fragment>элементы ўтрымліваюць асобныя фрагменты xml. Змест ст<fragment> можа быць любым xml-элементам з падтрымоўванай схемы GeoNetwork з умовай, што элемент павінен утрымліваць дастатковую колькасць рэлевантных метададзеных, каб можна было ідэнтыфікаваць мэтавую схему (г.зн. адрозніваць прасторы імёнаў).

<replacementGroup>элементы маюць значэнне толькі падчас генерацыі метададзеных. Яны выкарыстоўваюцца для групоўкі нуля ці больш фрагментаў для ўстаўкі або стварэння спасылак у копіі шаблону метададзеных, які выкарыстоўваецца для стварэння метададзеных. Дзе<replacementGroup> элемент змяшчае няма<fragment> элементаў, спасылачны элемент у копіі шаблона будзе выдалены, інакш ён будзе заменены змесцівам фрагмента.

Дапушчальныя атрыбуты гэтых элементаў і іх функцыі наступныя:

элемент

Атрыбут

Апісанне

Запіс

Uuid

Uuid згенераванага запісу метаданых (неабавязкова - у адваротным выпадку адзін будзе прызначаны камбайнам)

Фрагмент

ідэнтыфікатар

Ідэнтыфікатар элемента ў шаблоне метададзеных для замены/спасылкі. Ігнаруецца, калі фрагмент знаходзіцца ў групе замены.

Uuid

Uuid для згенераванага падшаблона (выкарыстоўваецца для спасылкі на гэты падшаблон з метададзеных)

Назва

Назва фрагмента – выкарыстоўваецца ў якасці загалоўка xlink

Група замены

ідэнтыфікатар

Ідэнтыфікатар элемента ў шаблоне метададзеных для выдалення, замены або спасылкі на ўтрымоўваныя фрагменты

Нарэшце, можна прывесці два прыклады таго, як сабраць метададзеныя з асаблівасцей зборшчыка OGC WFS, выкарыстоўваючы табліцы стыляў і шаблоны, якія пастаўляюцца разам з GeoNetwork.

У камплекце GeoServer Boundaries Harvest прыклад¶

У гэтым прыкладзе мяркуецца, што вы ўсталявалі ў камплекце GeoServer, які пастаўляецца з GeoNetwork. Канчатковым вынікам гэтага прыкладу будзе 251 запіс метададзеных ISO19139, якія спасылаюцца ў 1506 фрагментах (6 на запіс), створаных з адказу GetFeature у шейп-файле межаў у асобніку GeoServer, які пастаўляецца разам з GeoNetwork. Створаныя запісы ўтрымліваюць метададзеныя аб краінах свету.

Працэдура выканання:

  • У меню Адміністраванне-&gt;Канфігурацыя сістэмы ўключыце XLink Resolver і Захавайце канфігурацыю ў базе дадзеных.

  • Дадайце камбайн OGC WFS GetFeature response з меню «Адміністраванне-&gt;Збор».

  • Дайце яму Імя (напрыклад, gboundaries) і ўвядзіце URL-адрас службы wfs з уключанага геасервера (напрыклад, http://localhost:8080/geoserver/wfs) у полі URL-адрас службы.

  • Мы будзем выкарыстоўваць просты запыт GetFeature, каб выбраць усе краіны з шейп-файла межаў за WFS. XML для такога запыту (які трэба ўвесці ў тэкставае поле GetFeature Query):

<wfs:GetFeature service="WFS" version="1.1.0"
        xmlns:wfs="http://www.opengis.net/wfs">

 <wfs:Query typeName="gboundaries"/>

</wfs:GetFeature>

  • Выберыце схему вываду - мы абярэм iso19139, бо гэтая схема мае прыклады табліц стыляў і шаблонаў, якія нам патрэбны для гэтага прыкладу. Звярніце ўвагу, што пасля выбару гэтага параметра наступныя параметры становяцца бачнымі, і мы прымем наступныя дзеянні:

    • Выберыце табліцу стыляў &#39;geoserver_boundary_fragments&#39;, якая пастаўляецца, каб атрымаць фрагменты з адказу GetFeature у выпадальным спісе Табліца стыляў для стварэння фрагментаў. Гэтую табліцу стыляў можна знайсці ў GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/WFSToFragments.

    • Абярыце прыкладаемы шаблон «Шаблон тэсціравання межаў краін Geoserver WFS Fragments» з выпадальнага спісу Шаблон для стварэння метаданых з выкарыстаннем фрагментаў. Гэты шаблон можна знайсці ў GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/geoserver_fragment_tester.xml.

  • Выберыце катэгорыю для запісаў, створаных камбайнам, адзначце поле Толькі адзін запуск, дадайце некаторыя прывілеі (прасцей за ўсё дазволіць усім карыстальнікам мець правы прагляду). На гэтым этапе форма запісу камбайна павінна выглядаць так, як паказана на наступным скрыншоце.

  • Захаваць форму запісу камбайна.

  • Вы вернецеся ў меню аперацый камбайна, дзе вы можаце Актываваць камбайн, а затым Запусціць яго.

На старонцы вынікаў паказана, што з адказу WFS GetFeature было сабрана 1506 фрагментаў метададзеных. Яны былі захаваны ў базе дадзеных GeoNetwork як падшаблоны і звязаны ў шаблон метададзеных, каб сфармаваць 251 новы запіс метададзеных.

Прыклад базы дадзеных філосафа Degree Version 2.x¶

У гэтым прыкладзе мяркуецца, што вы спампавалі Deegree версіі 2.x і загрузілі прыклад базы дадзеных Philosopher. Канчатковым вынікам гэтага прыкладу будуць 7 запісаў метададзеных ISO19139, якія спасылаюцца ў 42 фрагменты (6 на запіс), створаныя з адказу GetFeature з вашай устаноўкі дыплома. Запісы ўтрымліваюць метададзеныя пра 7 вядомых філосафаў.

Працэдура выканання:

  • У меню Адміністраванне-&gt;Канфігурацыя сістэмы ўключыце XLink Resolver і Захавайце канфігурацыю ў базе дадзеных.

  • Дадайце камбайн OGC WFS GetFeature response з меню «Адміністраванне-&gt;Збор».

  • Дайце яму Імя (напрыклад, deegree22-philosopher-test) і ўвядзіце URL вашай устаноўкі deegree 2.2 у поле Service URL.

  • Мы будзем выкарыстоўваць просты запыт GetFeature, каб выбраць усіх філосафаў з базы дадзеных WFS. XML для такога запыту (які трэба ўвесці ў тэкставае поле GetFeature Query):

<wfs:GetFeature version="1.1.0" xmlns:app="http://www.deegree.org/app"
           xmlns:wfs="http://www.opengis.net/wfs">

 <!-- request all Philosopher instances -->
 <wfs:Query typeName="app:Philosopher"/>

</wfs:GetFeature>

  • Выберыце схему вываду - мы абярэм iso19139, бо гэтая схема мае прыклады табліц стыляў і шаблонаў, якія нам патрэбны для гэтага прыкладу. Звярніце ўвагу, што пасля выбару гэтага параметра наступныя параметры становяцца бачнымі, і мы прымем наступныя дзеянні:

    • Выберыце табліцу стыляў &#39;deegree2_philosopher_fragments&#39;, якая пастаўляецца, каб атрымаць фрагменты з адказу GetFeature у выпадальным спісе Табліца стыляў для стварэння фрагментаў. Гэтую табліцу стыляў можна знайсці ў GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/convert/WFSToFragments.

    • Выберыце прыкладаны шаблон «Шаблон тэсціравання філасофскай базы дадзеных Deegree 22 WFS Fragments» з выпадальнага спісу Шаблон для стварэння метаданых з выкарыстаннем фрагментаў. Гэты шаблон можна знайсці ў GEONETWORK_DATA_DIR/config/schema_plugins/iso19139/templates/deegree_fragment_tester.xml.

  • Выберыце катэгорыю для запісаў, створаных камбайнам, адзначце поле Толькі адзін запуск, дадайце некаторыя прывілеі (прасцей за ўсё дазволіць усім карыстальнікам мець правы прагляду). На гэтым этапе форма запісу камбайна павінна выглядаць так, як паказана на наступным скрыншоце.

  • Захаваць форму запісу камбайна.

  • Вы вернецеся ў меню аперацый камбайна, дзе вы можаце Актываваць камбайн, а затым Запусціць яго.

Пасля таго, як камбайн быў запушчаны, вы павінны ўбачыць экран вынікаў, які выглядае прыкладна так, як паказана ніжэй.

На старонцы вынікаў відаць, што з адказу WFS GetFeature было сабрана 42 фрагменты метададзеных. Яны былі захаваны ў базе дадзеных GeoNetwork як падшаблоны і звязаны ў шаблон метададзеных, каб сфармаваць 7 новых запісаў метададзеных.
