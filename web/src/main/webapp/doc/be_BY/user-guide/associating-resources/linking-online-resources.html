Звязванне вэб-сайтаў, вэб-сэрвісаў, ... з выкарыстаннем URL¶

Гэты раздзел адносіцца ў асноўным да запісаў ISO19139 і часткова да стандартаў Dublin Core (г. зн. толькі дакументы могуць быць звязаны ў Dublin Core).

Звязванне дакумента¶

Для звязвання дакументаў можна выкарыстоўваць 2 спосабу:

  • шляхам прадастаўлення URL

  • загрузіўшы дакумент

To add a new one, click on the + caret button then Add an online resource button or, if already one exists, just click on the title of the Online resources.

Каб звязаць URL, задайце наступныя ўласцівасці:

  • Пратакол для апісання тыпу далучанага дакумента (Вэб-адрас (URL) з&#39;яўляецца стандартным)

  • Спасылка, каб паказаць на мэтавы дакумент. Гэта могуць быць любыя спасылкі, такія як http://, ftp://, file:///, ...

  • Імя не з&#39;яўляецца абавязковым і дае метку для стварэння гіперспасылкі

  • Апісанне не з&#39;яўляецца абавязковым і змяшчае больш падрабязную інфармацыю аб спасылцы

Каб загрузіць дакумент, пераключыцеся на ўкладку Загрузіць і абярыце дакумент або перацягніце яго ва ўсплываючым акне. У такім выпадку пратакол схаваны і ўсталяваны ў WWW:DOWNLOAD.

У залежнасці ад вашых патрэб можна дадаць больш канкрэтныя спасылкі, якія будуць звязаны з рознымі дзеяннямі і будуць адлюстроўвацца ў праграмах.

Звязванне ўзроўню WMS¶

Каб мець магчымасць праглядаць запіс у праграме прагляду карт, можа спатрэбіцца дадаць спасылку на адзін або некалькі сэрвісаў WMS, якія публікуюць набор даных. Інтэрнет-рэсурс закадаваны з дапамогай наступнага ў ISO19139:

<gmd:onLine xmlns:gmd="http://www.isotc211.org/2005/gmd"
            xmlns:gco="http://www.isotc211.org/2005/gco">
   <gmd:CI_OnlineResource>
      <gmd:linkage>
         <gmd:URL>https://download.data.grandlyon.com/wms/grandlyon</gmd:URL>
      </gmd:linkage>
      <gmd:protocol>
         <gco:CharacterString>OGC:WMS</gco:CharacterString>
      </gmd:protocol>
      <gmd:name>
         <gco:CharacterString>cad_cadastre.cadsubdivisionsection</gco:CharacterString>
      </gmd:name>
      <gmd:description>
         <gco:CharacterString>Subdivision de section cadastrale (Plan cadastral informatisé du Grand Lyon)(OGC:WMS)</gco:CharacterString>
      </gmd:description>
   </gmd:CI_OnlineResource>
</gmd:onLine>

Каб дадаць пласт WMS:

  • абярыце пратакол OGC:WMS Web Map Service,

  • ўсталяваць URL службы,

  • then the wizard query the service to retrieve the list of layers

  • абярыце адзін або некалькі слаёў у спісе або задайце іх уручную.

Звязванне табліцы базы дадзеных або файла ГІС у сетцы¶

Каб спасылацца на файл ГІС або табліцу базы дадзеных, карыстальнік можа загрузіць гэты рэсурс або зрабіць спасылку на гэты рэсурс (гл. Звязванне дакумента). Тып пратаколу залежыць ад тыпу звязанага рэсурсу:

Тып рэсурсу

Запампаваны вектарны файл (напрыклад, сціснуты Shapefile)

URL

URL файла, створаны пасля загрузкі ў каталог. напр. http://localhost:8080/geonetwork/srv/eng/resources.get?id=1631&amp;fname=CCM.zip&amp;access=private

Пратакол

WWW: СПАМПАВАЦЬ

Імя

Імя файла (толькі для чытання)

Тып рэсурсу

Вектарны файл у сетцы

URL

Шлях да файла. напр. file:///shared/geodata/world/hydrology/rivers.shp

Пратакол

ФАЙЛ:GEO або ФАЙЛ:РАСТРАВЫ

Імя

Апісанне файла

Тып рэсурсу

Вектар (табліца PostGIS)

URL

jdbc:postgresql://лакальны хост:5432/лагін:пароль@db

Пратакол

БД: POSTGIS

Імя

Назва табліцы

Калі ёсць інфармацыя пра базу дадзеных або файл у лакальнай сетцы, можа спатрэбіцца схаваць гэтую інфармацыю для агульнадаступных карыстальнікаў (гл. Абмежаванне інфармацыі раздзеламі метададзеных).

Рабіць

Дадайце дакумент і спасылку на geopublisher

Linking data using ATOM feeds¶

If your organisation provides ATOM feeds to facilitate access to the data, metadata records can link to these feeds. Users can link to a service feed in a service record and to a dataset feed in a dataset record.

<gmd:MD_DigitalTransferOptions>
 <gmd:onLine>
  <gmd:CI_OnlineResource>
   <gmd:linkage>
    <gmd:URL>http://www.broinspireservices.nl/atom/awp.atom</gmd:URL>
   </gmd:linkage>
   <gmd:protocol>
    <gco:CharacterString>INSPIRE Atom</gco:CharacterString>
   </gmd:protocol>
   <gmd:name>
    <gco:CharacterString>gdn.Aardwarmtepotentie</gco:CharacterString>
   </gmd:name>
  </gmd:CI_OnlineResource>
 </gmd:onLine>
</gmd:MD_DigitalTransferOptions>

Once registered in the metadata, the ATOM feed will be displayed on the record view:

Users can choose a service, the list of datasets available are retrieved and the download links displayed to the user. A dataset feed can have one or more downloads:

Examples:

  • NGR National Georegister, Statistics Netherlands Land Use 2015 ATOM

The catalog also provides the capability to produce ATOM feeds for services and datasets from metadata records. To enable this, check the ATOM API (see OpenSearch and INSPIRE ATOM). A GML dataset can be exposed using the following encoding in order to be published in a dataset ATOM feed:

<gmd:transferOptions>
   <gmd:MD_DigitalTransferOptions>
      <gmd:unitsOfDistribution>
         <gco:CharacterString>B</gco:CharacterString>
      </gmd:unitsOfDistribution>
      <gmd:transferSize>
         <gco:Real>428973180</gco:Real>
      </gmd:transferSize>
      <gmd:onLine>
         <gmd:CI_OnlineResource>
            <gmd:linkage>
               <gmd:URL>https://download.data.public.lu/resources/inspire-annex-i-theme-addresses-addresses/20191118-115245/ad.address.gml</gmd:URL>
            </gmd:linkage>
            <gmd:protocol>
               <gco:CharacterString>WWW:DOWNLOAD-1.0-http--download</gco:CharacterString>
            </gmd:protocol>
            <gmd:applicationProfile>
               <gco:CharacterString>INSPIRE-Download-Atom</gco:CharacterString>
            </gmd:applicationProfile>
            <gmd:name>
               <gmx:MimeFileType type="application/octet-stream">AD.Address.gml</gmx:MimeFileType>
            </gmd:name>
            <gmd:description>
               <gco:CharacterString></gco:CharacterString>
            </gmd:description>
            <gmd:function>
               <gmd:CI_OnLineFunctionCode codeList="http://standards.iso.org/ittf/PubliclyAvailableStandards/ISO_19139_Schemas/resources/codelist/ML_gmxCodelists.xml#CI_OnLineFunctionCode"
                                          codeListValue="download">download</gmd:CI_OnLineFunctionCode>
            </gmd:function>
         </gmd:CI_OnlineResource>
      </gmd:onLine>
   </gmd:MD_DigitalTransferOptions>
</gmd:transferOptions>

Examples:

  • INSPIRE Portal of the Grand-Duchy of Luxembourg / INSPIRE - Annex I Theme Addresses - Addresses

Reference documents:

  • INSPIRE technical guidelines for download services.
