Прывілеі кіравання¶

To manage privileges to your metadata record and any attached data, you will need to identify User Groups and the privileges you want to assign to users in these groups. eg. Viewing the metadata, downloading the data attached to the record, etc.

For example, you can specify that the metadata and related services are visible to all (Internet users) or just to internal users only (Intranet). Privileges are assigned on a per group basis. Depending on the user profile (Guest, Registered User, Editor, Admin etc.) access to these functions may differ on a per user basis.

Assigning privileges¶

Каб прызначыць прывілеі, выканайце наступныя дзеянні:

  1. Find your metadata record using the search option. Whether you have multiple or single results from the search, on top of the individual record or next to the record you will always see a row of buttons including a Privileges button.

  2. Click the Privileges button. A drop down menu appears. You can assign certain privileges to specific groups using checkboxes.

  3. Click the small box next to the privilege to place or remove a checkmark. The Set All and Clear All buttons allow you to place and remove the checkmarks all at once.

Ніжэй прыводзіцца кароткае апісанне кожнай прывілеі, каб дапамагчы вам вызначыць, якія з іх вы павінны прызначыць якой групе(ям).

Апублікаваць: карыстальнікі ў названай групе/ах могуць праглядаць метададзеныя, напрыклад. калі ён адпавядае крытэрам пошуку, уведзеным такім карыстальнікам.

Спампаваць: карыстальнікі ў названай групе/ах могуць спампоўваць дадзеныя.

Інтэрактыўная карта: Карыстальнікі з названай групы могуць атрымаць інтэрактыўную карту. Інтэрактыўную карту трэба ствараць асобна з дапамогай вэб-сервера карт, напрыклад GeoServer, які распаўсюджваецца разам з GeoNetwork.

Выбранае: пры выпадковым выбары GeoNetwork запіс метададзеных можа з&#39;явіцца ў раздзеле Выбранае на галоўнай старонцы GeoNetwork.

Апавяшчаць: Карыстальнікі ў пазначанай групе атрымліваюць апавяшчэнне аб спампоўцы даных, далучаных да запісу метададзеных.

Review and make sure that you understand Карыстальнікі, групы і ролі in the User and Group Administration section of this manual.

Note

Публічны запіс метададзеных - гэта запіс метададзеных, які мае прывілей прагляду для групы пад назвай &quot;Усе&quot;.

Наступныя правілы прымяняюцца да дазволаў на прагляд і рэдагаванне запісу метададзеных:

Прагляд¶

Адміністратар можа праглядаць любыя метададзеныя.

Аглядальнік кантэнту можа праглядаць метададзеныя, калі:

  • Метададзеныя з&#39;яўляюцца публічнымі або

  • Метададзеныя з&#39;яўляюцца часткай групы, членам якой з&#39;яўляецца карыстальнік.

Адміністратар карыстальніка або рэдактар можа праглядаць:

  • All metadata that has the view privilege selected for one of the groups they are a member of.

  • All metadata created by them.

Зарэгістраваны карыстальнік можа праглядаць:

  • All metadata that has the view privilege selected for one of the groups they are a member of.

Any user (logged in or not) can view the public metadata.

Рэдагаванне¶

Адміністратар можа рэдагаваць любыя метададзеныя.

Аглядальнік можа рэдагаваць метаданыя, калі:

  • Уладальнік метададзеных з&#39;яўляецца членам адной з груп, прызначаных рэцэнзенту.

  • They are the metadata owner.

A User Administrator or an Editor can only edit metadata they created.

Setting Privileges¶

Налада прывілеяў для запісу метададзеных¶

A button to access the Privileges page for a metadata record displays in the search results or when you are viewing the record for:

  • Усе адміністратары

  • Усе аглядальнікі, якія ўваходзяць у адну з груп, прызначаных уладальніку метададзеных.

  • Уладальнік метададзеных

Only Administrators and Reviewers can edit privileges for the All and Intranet groups.

Ўстаноўка прывілеяў для абранага набору запісаў метададзеных¶

You can set privileges on a selected set of records in the search results using the "actions on selected set" menu.

Прымяняюцца наступныя правілы:

  • групы - гэта тыя, да якіх належыць карыстальнік

  • указаныя прывілеі будуць прымяняцца толькі да запісаў, на якія карыстальнік мае права ўласнасці або адміністравання - любыя іншыя запісы будуць прапушчаны

  • the current records privileges will be reset and replaced by the selected privilege
