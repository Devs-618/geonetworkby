Карыстальнік па змаўчанні¶

Upon installation a default user with name admin and password admin is created. It is recommended to log in with these credentials directly after the installation has completed and to change the default password.

Сесія карыстальніка¶

After the authentication process, a user session is created. This session will be closed automatically at some point by the server for security reasons. The default session timeout is set to 35 min (see Канфігурацыя часу чакання сеанса карыстальніка for details).

Карыстальнікі, групы і ролі¶

У каталогу выкарыстоўваецца канцэпцыя карыстальнікаў, груп і профіляў карыстальнікаў.

  • Карыстальнік можа быць часткай адной або некалькіх груп.

  • Карыстальнік мае ролю ў групе.

  • Роля адміністратара не звязана з групай.

Спалучэнне ролі і групы вызначае, якія задачы карыстальнік можа выконваць у сістэме або ў пэўных запісах метададзеных.

Карыстальнікі могуць мець розныя ролі ў розных групах. Роля вызначае, якія задачы можа выконваць карыстальнік.

Ролі іерархічныя і заснаваныя на спадчыне. Гэта азначае, што карыстальнік з профілем рэдактара можа ствараць і змяняць новыя запісы метададзеных, але таксама можа выкарыстоўваць усе функцыі, якія можа выкарыстоўваць зарэгістраваны карыстальнік.

Правы, звязаныя з ролямі, падрабязна праілюстраваны ў спісе ніжэй:

  1. Профіль адміністратара

    Адміністратар мае спецыяльныя прывілеі, якія даюць доступ да ўсіх даступных функцый.

    Да іх адносяцца:

    • Full rights for creating new groups and new users.

    • Rights to change users/groups profiles.

    • Full rights for creating/editing/deleting new/old metadata.

    • Выконвайце задачы па адміністраванні і канфігурацыі сістэмы.

  2. Профіль адміністратара

    The user administrator is the administrator of his/her own group(s) with the following privileges:

    • Full rights on creating new users within their own groups.

    • Rights to change users profiles within their own groups.

  3. Профіль аглядальніка кантэнту

    The content reviewer is the only person allowed to give final clearance on the metadata publication on the Intranet and/or on the Internet:

    • Rights on reviewing metadata content within their own groups and authorising its approval and publication.

  4. Профіль рэдактара

    Рэдактар працуе з метададзенымі з наступнымі прывілеямі:

    • Full rights on creating/editing/ deleting new/old data within their own groups.

  5. Профіль зарэгістраванага карыстальніка

    Зарэгістраваны карыстальнік мае больш прывілеяў доступу, чым госці, якія не прайшлі аўтэнтыфікацыю:

    • Right to download protected data.
