Настройка віртуальнага CSW¶

Warning: This feature will be deprecated in future version (4.x). Use Portal configuration instead.

This feature adds the capability to create custom CSW entry points that apply extra criteria to the CSW requests, allowing to implement several useful cases like, for example:

  • Вызначце кропку ўваходу ў INSPIRE CSW, каб дастаўляць толькі звязаныя з INSPIRE метададзеныя, якія захоўваюцца ў каталогу.

  • Вызначце пункты ўваходу ў CSW, каб дастаўляць толькі метаданыя, якія адносяцца да канкрэтнай тэмы: клімат, межы і г.д.

The filter parameter value should use the Lucene query parser syntax, which is applied to these CSW operations:

  • GetRecords: фільтр прымяняецца з запытам CSW у якасці дадатковага крытэрыю запыту.

  • GetRecordById: фільтр прымяняецца з метаданымі id, якія запытваюцца ў якасці дадатковага крытэрыю запыту.

  • GetDomain: фільтр прымяняецца ў якасці крытэрыю запыту для атрымання запытаных уласцівасцей метададзеных.

  • GetCapabilities: фільтр прымяняецца ў якасці крытэрыю запыту для запаўнення спісу ключавых слоў метададзеных у дакуменце GetCapabilities.

Спіс даступных палёў індэксаў Lucene для выкарыстання ў параметрах filter можна атрымаць з файлаў index-fields.xsl у папках схем, размешчаных у WEB-INF/xml/schemas.

Паколькі параметр фільтр не ўплывае на аперацыі Harvest і Transaction, каб пазбегнуць блытаніны лепш выкарыстоўваць гэту функцыю ў якасці канцавых кропак CSW толькі для чытання.
