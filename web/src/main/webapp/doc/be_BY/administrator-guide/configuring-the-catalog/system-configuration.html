Канфігурацыя сістэмы¶

Большасць параметраў канфігурацыі сістэмы могуць быць зменены карыстальнікамі-адміністратарамі з дапамогай вэб-інтэрфейсу ў Кансоль адміністратара > Налады.

Important

Канфігурацыя гэтых параметраў вельмі важная для каталога ў аператыўным кантэксце. Няправільнае разуменне некаторых налад можа прывесці да таго, што сістэма не будзе працаваць належным чынам. Напрыклад, спампоўкі могуць не апрацоўвацца карэктна, або збор метададзеных з іншых сервераў можа не працаваць.

Паколькі форма налад - гэта доўгая форма, кнопка 'захаваць' паўтараецца паміж раздзеламі і захавае ўсе налады.

Апісанне каталога¶

  • Імя каталога Імя вузла. Інфармацыя, якая дапамагае ідэнтыфікаваць каталог чалавеку. Імя адлюстроўваецца на банеры ў CSW GetCapabilities.

  • Ідэнтыфікатар каталога Універсальны ўнікальны ідэнтыфікатар (uuid), які адрознівае ваш каталог ад любога іншага каталога. Гэта ўнікальны ідэнтыфікатар для вашага каталога, і лепш пакінуць яго як uuid. Ён будзе выкарыстоўвацца камбайнам з выкарыстаннем пратакола GeoNetwork для ідэнтыфікацыі зыходнага каталога.

  • Арганізацыя Арганізацыя, да якой належыць вузел. Зноў жа, гэта інфармацыя, якая дапамагае ідэнтыфікаваць каталог чалавеку-карыстальніку.

  • SVN UUID Subversion repository attached to the node. This identifier is created and/or checked on startup to verify that the database match the SVN repository. The repository is used for metadata versioning.

Каталог¶

  • Версія Версія каталога (толькі для чытання, версія базы дадзеных)

  • Мінорная версія Мінорная версія каталога (толькі для чытання, версія базы дадзеных)

Сервер каталогаў¶

  • Хост Імя або IP-нумар вузла (без http://). Напрыклад, яны выкарыстоўваюцца падчас рэдагавання метададзеных для стварэння спасылак на рэсурсы і пры вяртанні магчымасцей сервера падчас запыту CSW.

  • Калі ваш вузел агульнадаступны з Інтэрнэту, вы павінны выкарыстоўваць даменнае імя.

  • Калі ваш вузел схаваны ў вашай прыватнай сетцы і ў вас ёсць брандмаўэр або вэб-сервер, які перанакіроўвае ўваходныя запыты на вузел, вы павінны ўвесці агульнадаступны адрас брандмаўэра або вэб-сервера. Тыповая канфігурацыя заключаецца ў наяўнасці вэб-сервера Apache на адрасе A, які з'яўляецца агульнадаступным і перанакіроўвае запыты на сервер Tomcat на прыватным адрасе B. У гэтым выпадку вы павінны ўвесці A у параметр хаста.

  • Порт Нумар порта сервера (звычайна 80 або 8080). Калі выкарыстоўваецца HTTP, усталюйце яго на 80.

  • Пераважны пратакол Вызначыў пратакол доступу да каталога. Пратакол HTTP, які выкарыстоўваецца для доступу да сервера. Выбар http азначае, што ўся сувязь з каталогам будзе бачная ўсім, хто слухае пратакол. Паколькі гэта ўключае імёны карыстальнікаў і паролі, гэта небяспечна. Выбар https азначае, што ўся сувязь з каталогам будзе зашыфравана і, такім чынам, слухачу будзе значна цяжэй дэкадаваць.

  • Бяспечны порт Нумар бяспечнага порта.

  • Узровень журнала Вызначце ўзровень вядзення часопіса прыкладання. Пасля мадыфікацыі журнал можна праверыць у раздзеле Статыстыка і статус у раздзеле Актыўнасць.

Параметры інтрасеткі¶

Агульнай неабходнасцю для арганізацыі з'яўляецца аўтаматычная дыскрымінацыя паміж ананімнымі ўнутранымі карыстальнікамі, якія атрымліваюць доступ да вузла ўнутры арганізацыі (Інтранэт), і ананімнымі знешнімі карыстальнікамі з Інтэрнэту. Каталог вызначае ананімных карыстальнікаў знутры арганізацыі як якія належаць да групы Унутраная сетка, у той час як ананімныя карыстальнікі з-за межаў арганізацыі вызначаюцца групай Усе. Каб аўтаматычна адрозніваць карыстальнікаў, якія належаць да групы Intranet, вам трэба паведаміць каталогу IP-адрас і маску сеткі.

  • Netmask Маска інтрасеткі (напрыклад, 255.255.255.0).

  • Сетка* Адрас унутранай сеткі ў форме IP (напрыклад, 147.109.100.0).

Z39.50¶

  • Уключыць сервер Z39.50 Уключыць сервер Z39.50. Перазапусціце каталог, каб зрабіць гэта змяненне актыўным.

  • Порт Змяніце порт праслухоўвання Z39.50. Перазапусціце каталог, каб зрабіць гэта змяненне актыўным.

Проксі сервер¶

The settings page offers to set the configuration of a proxy server. This configuration is used by the application to access the internet to get online resources, for example as part of a harvest process.

  • Выкарыстоўваць проксі Уключыце проксі, калі каталог знаходзіцца за проксі і трэба выкарыстоўваць яго для доступу да аддаленых рэсурсаў.

  • Проксі-хост IP-адрас або імя проксі

  • Порт Порт проксі

  • Імя карыстальніка проксі Імя карыстальніка

  • Пароль карыстальніка проксі Пароль імя карыстальніка

  • *Ігнараваць спіс хастоў Каб абыйсці пэўныя хасты, увядзіце пэўны IP-адрас або імя хаста, напрыклад www.mydomain.com, або дыяпазон адрасоў з выкарыстаннем падстаноўных знакаў, напрыклад 192.168.2.*. Выкарыстоўвайце | каб падзяліць розныя значэнні хаста.

Параметры проксі-сервера JVM таксама могуць спатрэбіцца для правільнай налады проксі для ўсяго аддаленага доступу.

Зваротная сувязь¶

Электронная пошта можа быць адпраўлена каталогам.

  • вы карыстаецеся сістэмай самарэгістрацыі карыстальніка

  • вы выкарыстоўваеце працоўны працэс стану метададзеных (гл. жыццёвы цыкл)

  • файл, загружаны з запісам метададзеных, спампоўваецца і выбіраецца прывілей паведамлення

У гэтым раздзеле наладжваецца паштовы сервер для выкарыстання.

  • Электронная пошта Гэта адрас электроннай пошты адміністратара, які выкарыстоўваецца для адпраўкі водгукаў.

  • SMTP host Імя паштовага сервера або IP-адрас для адпраўкі электронных лістоў.

  • Порт SMTP Порт SMTP.

  • Выкарыстоўваць SSL Уключыць рэжым SSL

  • Імя карыстальніка Імя карыстальніка, калі патрабуецца злучэнне на серверы SMTP

  • Пароль Пароль карыстальніка, калі патрабуецца падключэнне да сервера SMTP

Вынікі пошуку метададзеных¶

Налады канфігурацыі ў гэтай групе вызначаюць абмежаванні на ўзаемадзеянне карыстальніка з вынікамі пошуку.

  • Максімальная колькасць выбраных запісаў Максімальная колькасць вынікаў пошуку, якія карыстальнік можа выбраць і апрацаваць з дапамогай пакетных аперацый, напрыклад. Усталюйце прывілеі, катэгорыі і г.д. Гэты параметр дазваляе пазбегнуць доўгага дзеяння, якое можа выклікаць памылку недахопу памяці.

Рэйтынг метаданых¶

Калі ўключана, каталог будзе вылічваць рэйтынгі карыстальнікаў для метададзеных толькі з гэтага вузла (не распаўсюджвацца сярод іншых вузлоў GeoNetwork). Гэта адносіцца толькі да запісаў, сабраных з выкарыстаннем пратаколу GeoNetwork.

Сэрвіс загрузкі¶

Рэдактар метададзеных падтрымлівае загрузку аднаго або некалькіх файлаў, якія можна захоўваць разам з запісам метададзеных. Калі такі запіс адлюстроўваецца ў выніках пошуку, з'яўляецца кнопка «Спампаваць», якая дазваляе карыстальніку выбраць, які файл ён хоча спампаваць. Гэтая група параметраў вызначае, як будзе адбывацца загрузка.

  • Карыстайцеся простым сэрвісам загрузкі файлаў GeoNetwork Націск на любы файл, які захоўваецца з запісам метададзеных, даставіць гэты файл непасрэдна карыстальніку праз браўзер.

  • Выкарыстоўвайце службу адмовы ад адказнасці і абмежаванняў GeoNetwork Націск на любы файл, які захоўваецца з запісам метададзеных, даставіць карыстальніку (праз браўзер) архіў zip, які змяшчае файл дадзеных, сам запіс метададзеных і зводку метададзеных абмежавання рэсурсу як html дакумент. Акрамя таго, карыстальнік павінен будзе падаць некаторыя звесткі (імя, арганізацыя, электронная пошта і неабавязковы каментар) і прагледзець абмежаванні рэсурсаў, перш чым яны змогуць загрузіць zip-архіў.

Метададзеныя XLink¶

Рэзолвер XLink замяняе змесціва элементаў з атрыбутам @xlink:href (за выключэннем некаторых элементаў, такіх як srv:operatesOn) на змесціва, атрыманае з змесціва URL @xlink:href. Развязвальнік XLink павінен быць уключаны, калі вы хочаце сабраць фрагменты метададзеных або паўторна выкарыстоўваць фрагменты метададзеных у вашых запісах метададзеных (напрыклад, пры выкарыстанні каталога кантактаў).

  • Уключыць дазвол XLink: уключае/выключае дазвол XLink.

  • Уключыць лакальны XLink Лакальныя XLink выкарыстоўваюць local://<lang> /<service> URL для спасылак на звязаныя раздзелы замест HTTP URL. Лакальныя XLinks звычайна хутчэй, чым HTTP XLinks.

See also

Для павышэння прадукцыйнасці каталог будзе кэшаваць кантэнт, якога няма ў лакальным каталогу. Ачысціце кэш XLink з Кансолі адміністратара > Інструменты, калі фрагменты былі абноўлены.

Абнаўленне метаданых¶

Для кожнай схемы метададзеных у каталогу ёсць трансфармацыя XSL (update-fixed-info.xsl), якую ён можа прымяніць да запісу метададзеных, які належыць да гэтай схемы. Мэта гэтага пераўтварэння - дазволіць прымяненне фіксаванай схемы, сайта і інфармацыі каталога да запісу метададзеных кожны раз, калі запіс метададзеных захоўваецца ў рэдактары. У якасці прыкладу гэта пераўтварэнне выкарыстоўваецца для стварэння і захавання URL любых файлаў, загружаных і захоўваемых разам з запісам метададзеных у рэдактары.

  • Аўтаматычныя выпраўленні: уключана па змаўчанні. Рэкамендуецца не выкарыстоўваць рэдактар метададзеных, калі аўтавыпраўленне адключана. Глядзіце http://trac.osgeo.org/geonetwork/ticket/368 для больш падрабязнай інфармацыі.

Статыстыка пошуку¶

Уключае/выключае збор статыстыкі пошуку. Статыстыка пошуку захоўваецца ў базе дадзеных і можа быць запытана з дапамогай старонкі Статыстыка пошуку.

Захоўванне статыстыкі пошуку патрабуе вельмі невялікіх вылічальных выдаткаў, бо яна запісваецца ў базу дадзеных у фонавым патоку. Аднак захоўванне базы дадзеных для вельмі загружанага сайта павінна быць старанна спланавана.

Аптымізацыя індэкса¶

Параметры канфігурацыі ў гэтай групе вызначаюць, калі запускаецца аптымізатар індэкса Lucene. Па змаўчанні гэта адбываецца кожны дзень апоўначы. З нядаўнімі абнаўленнямі Lucene, асабліва Lucene 3.6.1, аптымізатар становіцца менш карысным, таму гэтая група канфігурацыі, хутчэй за ўсё, будзе выдалена ў наступных версіях.

Пастаўшчык Open Archive Initiative (OAI-PMH).¶

Параметры ў гэтай групе кантралююць спосаб, якім сервер OAI адказвае на запыты збору OAIPMH з аддаленых сайтаў.

  • Пошук па датах: OAI Harvesters можа запытваць запісы з GeoNetwork у дыяпазоне дат. GeoNetwork можа выкарыстоўваць адно з двух палёў даты з метададзеных, каб праверыць супадзенне з гэтым дыяпазонам дат. Выбар па змаўчанні - Часавы экстэнт, які ўяўляе сабой часовы экстэнт з запісу метададзеных. Іншы варыянт, Дата змены, выкарыстоўвае дату змены запісу метададзеных у базе дадзеных GeoNetwork. Дата мадыфікацыі - гэта апошні раз, калі запіс метададзеных быў абноўлены або сабраны ў GeoNetwork.

  • Тайм-аўт маркера аднаўлення: запісы метаданых, якія адпавядаюць запыту пошуку ўраджаю OAI, звычайна вяртаюцца ў камбайн групамі з фіксаваным памерам (напрыклад, групамі па 10 запісаў). З кожнай групай уключаны маркер аднаўлення, каб камбайн мог запытаць наступную групу запісаў. Тайм-аўт маркера аднаўлення - гэта час (у секундах), які сервер GeoNetwork OAI будзе чакаць выкарыстання маркера аднаўлення. Калі час чакання перавышаны, сервер OAI GeoNetwork выдаляе вынікі пошуку і адмаўляецца распазнаваць маркер аднаўлення. Мэта гэтай функцыі - забяспечыць вызваленне рэсурсаў на серверы GeoNetwork OAI.

  • Памер кэша: максімальная колькасць адначасовых збораў OAI, якую можа падтрымліваць сервер GeoNetwork OAI.

Перазапусціце каталог, каб улічыць усе налады OAI.

Канфігурацыя дырэктывы INSPIRE¶

Глядзіце Налада для дырэктывы INSPIRE.

INSPIRE Atom Feed¶

Дазваляе вызначаць канфігурацыю каналаў Atom, на якія спасылаюцца метададзеныя, каб прадастаўляць паслугі, звязаныя з тэхнічным кіраўніцтвам INSPIRE для паслуг загрузкі <http://inspire.ec.europa.eu/documents/Network_Services/Technical_Guidance_Download_Services_3.0.pdf> `_:

  • Выберыце тып падачы атама:

    • Аддалена: атрымлівайце атамныя каналы, на якія спасылаюцца метаданыя ў інтэрнэт-рэсурсах.

    • Лакальны (для рэалізацыі ў будучых версіях): стварыць атамны канал з выкарыстаннем змесціва метададзеных.

  • Расклад здабывання кармоў: перыядычнае выманне атамных кармоў можа быць запланавана.

  • Значэнне пратаколу Atom: значэнне пратаколу ў інтэрнэт-рэсурсах метаданых для ідэнтыфікацыі рэсурсаў канала atom (значэнне па змаўчанні INSPIRE Atom). GeoNetwork ідэнтыфікуе файл Atom з іншых рэсурсаў, гледзячы на значэнне пратаколу інтэрнэт-рэсурсу. Паколькі для гэтага пратаколу няма агульнапрынятага значэння, GeoNetwork дазваляе адміністратару ўсталёўваць значэнне, якое будзе выкарыстоўвацца ў якасці пратаколу, які вызначае рэсурсы Atom:

    <gmd:transferOptions>
    <gmd:MD_DigitalTransferOptions>
      <gmd:onLine>
        <gmd:CI_OnlineResource>
          <gmd:linkage>
            <gmd:URL>http://geodata.nationaalgeoregister.nl/atom/index.xml</gmd:URL>
          </gmd:linkage>
          <gmd:protocol>
            <gco:CharacterString>INSPIRE Atom</gco:CharacterString>
          </gmd:protocol>
        </gmd:CI_OnlineResource>
      </gmd:onLine>
    </gmd:MD_DigitalTransferOptions>
  </gmd:transferOptions>

Даступныя наступныя паслугі:

  • http://SERVER/geonetwork/opensearch/eng/UUID/OpenSearchDescription.xml

  • http://SERVER/geonetwork/opensearch/eng/UUID/search?queryParams

  • http://SERVER/geonetwork/opensearch/eng/search?queryParams

  • http://SERVER/geonetwork/opensearch/eng/UUID/describe?queryParams

  • http://SERVER/geonetwork/opensearch/eng/describe?queryParams

In above URLs {UUID} is the fileidentifier of the download service metadata.

Не забудзьцеся дадаць канчатковую кропку GeoNetwork OpenSearch у якасці апісання OpenSearchDescription для службы:

<link rel="search" href="http://SERVER/geonetwork/opensearch/eng/{uuid}/OpenSearchDescription.xml"
type="application/opensearchdescription+xml" title="Open Search document for INSPIRE Download service"/>

Рэалізацыя INSPIRE Atom/OpenSearch можа быць праверана з дапамогай тэстаў Atom у Esdin Test Framework (http://elfproject.eu/documentation/geotool/etf) або валідатара метададзеных INSPIRE (http://inspire-geoportal.ec.europa.eu/validator2).

Шматструменнае індэксаванне¶

Параметры канфігурацыі ў гэтай групе вызначаюць, колькі працэсарных патокаў выдзелена задачам індэксацыі ў GeoNetwork. Калі ваша машына мае шмат працэсарных ядраў, цяпер вы можаце вызначыць, колькі вылучыць для задач індэксацыі GeoNetwork. Гэта можа значна павялічыць хуткасць вялікіх задач індэксацыі (напрыклад, змяненне прывілеяў на 20 000 запісаў), таму што GeoNetwork можа падзяліць задачу індэксацыі на некалькі частак і прызначыць іх розным ядрам працэсара.

Колькасць патокаў апрацоўкі Максімальная колькасць патокаў апрацоўкі, якія могуць быць прызначаны для задачы індэксацыі.

Заўвага: гэтая опцыя даступная толькі для правераных баз дадзеных. Гэта базы дадзеных PostGIS і Oracle. Вы таксама павінны ўважліва разгледзець, колькі падключэнняў да базы дадзеных вы выдзяляеце ў канфігурацыі базы дадзеных, паколькі кожны паток можа звязаць адно падключэнне да базы дадзеных на працягу доўгага сеансу індэксацыі (напрыклад). Глядзіце пашыраную канфігурацыю для больш падрабязнай інфармацыі аб тым, як наладзіць колькасць злучэнняў у пуле злучэнняў базы дадзеных.

Шматмоўныя налады¶

Параметры ў гэтай групе вызначаюць, як GeoNetwork будзе шукаць метададзеныя на некалькіх мовах.

Уключыць аўтаматычнае вызначэнне мовы пошукавага запыту: Калі абраны гэты параметр, GeoNetwork прааналізуе пошукавы запыт і паспрабуе вызначыць мову, якая выкарыстоўваецца, перш чым па змаўчанні задаць мову GUI.

Вынікі пошуку на запытанай мове адсартаваныя зверху: Калі выбрана гэтая опцыя, пункт сартавання будзе дададзены да кожнага запыту, каб пераканацца, што вынікі на бягучай мове заўсёды адсартаваны зверху. Гэта адрозніваецца ад павелічэння прыярытэту мовы тым, што яно адмяняе рэлевантнасць выніку. Напрыклад, калі вынік на нямецкай мове мае вельмі высокую рэлевантнасць, але мова пошуку французская, то ўсе вынікі на французскай мове будуць перад нямецкімі.

Пошук толькі на запытанай мове Параметры ў гэтым раздзеле вызначаюць, як дакументы сартуюцца/прыярытызацыі адносна мовы ў дакуменце ў параўнанні з мовай пошуку.

  • Усе дакументы на ўсіх мовах (Без пераваг) - Мова пошуку ігнаруецца і не паўплывае на парадак вынікаў

  • Аддаваць перавагу дакументам з перакладам на запытаную мову - Дакументы з перакладам на мову пошуку (у любым месцы дакумента) будуць мець прыярытэт перад дакументамі без элементаў на мове пошуку

  • Аддаваць перавагу дакументам, мова якіх з'яўляецца запытанай мовай - Дакументы, якія з'яўляюцца той самай мовай, што і мова пошуку (г.зн. дакументы, якія пазначаны як тыя ж мовы, што і мова пошуку), маюць прыярытэт перад дакументамі, якія не з'яўляюцца такімі.

  • Пераклад на запытаную мову - Вынікі пошуку будуць утрымліваць толькі дакументы, якія маюць некаторыя пераклады на мову пошуку.

  • Запытанай мовай з'яўляецца мова дакумента - вынікі пошуку будуць утрымліваць дакументы, мова метаданых якіх вызначана як мова пошуку

Прагляды метададзеных¶

Параметры ў гэтым раздзеле ўключаюць/выключаюць групы элементаў метададзеных у рэдактары/праглядзе метададзеных.

Уключыць просты выгляд: просты выгляд у рэдактары/праглядзе метададзеных: - выдаляе вялікую частку іерархіі з укладзеных запісаў метададзеных (напрыклад, ISO19115/19139) - не дазваляе карыстальніку дадаваць элементы метададзеных, якіх яшчэ няма ў запісе метададзеных Ён прызначаны для забеспячэння плоскага, простага выгляду запісу метададзеных. Недахопам простага прагляду з'яўляецца тое, што частка кантэкстнай інфармацыі, атрыманай укладаннем у запісе метададзеных, губляецца. Уключыць прагляд ISO: Стандарт метададзеных ISO19115/19139 вызначае тры групы элементаў: - Мінімум: тыя элементы, якія з'яўляюцца абавязковымі - Асноўныя: элементы, якія павінны прысутнічаць у любым запісе метададзеных, які апісвае набор геаграфічных даных - Усе: усе элементы * Уключыць прагляд INSPIRE*: уключае групы элементаў метададзеных, вызначаныя ў дырэктыве ЕС INSPIRE. Уключыць прагляд XML: гэта выгляд рэдагавання неапрацаванага тэксту запісу XML. Вы можаце адключыць гэта, калі (напрыклад) вы не жадаеце, каб неспрактыкаваныя карыстальнікі былі збітыя з панталыку XML-прэзентацыяй, якую забяспечвае гэты выгляд.

Прывілеі метаданых¶

Усталёўваць прывілеі толькі для груп карыстальнікаў: калі гэта ўключана, на старонцы прывілеяў метададзеных будуць адлюстроўвацца толькі групы, да якіх належыць карыстальнік (калі карыстальнік не з&#39;яўляецца адміністратарам). На дадзены момант гэты параметр не можа быць адключаны і, верагодна, будзе састарэлым у наступнай версіі GeoNetwork.

Уборка ўраджаю¶

Дазволіць рэдагаванне сабраных запісаў: уключае/выключае рэдагаванне сабраных запісаў у каталогу. Па змаўчанні сабраныя запісы нельга рэдагаваць.
