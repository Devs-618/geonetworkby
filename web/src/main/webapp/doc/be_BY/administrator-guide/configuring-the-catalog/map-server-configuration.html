Канфігурацыя картографічных сервераў для геапублікацыі¶

Каб апублікаваць інфармацыю з каталога ў якасці службаў OGC (WMS, WFS, WCS), адміністратар каталога павінен зарэгістраваць адзін або некалькі карт-сервераў для публікацыі. Каб працаваць з каталогам, картаграфічныя серверы ПАВІННЫ падтрымліваць GeoServer REST API. Былі пратэставаны 2 наступныя рэалізацыі:

  • `Геасервер<http://geoserver.org> `_

  • Mapserver and Mapserver REST API

Наладзьце сервер карты, а затым зарэгіструйце яго з інтэрфейсу адміністравання:

Неабходныя наступныя параметры:

  • Імя: Метка сервера карты, якая будзе адлюстроўвацца пры геапублікацыі інфармацыі.

  • Апісанне: апісанне картаграфічнага сервера.

  • Канфігурацыя REST API: URL службы, якая забяспечвае доступ да аддаленай канфігурацыі з дапамогай REST API.

  • Імя карыстальніка і пароль, якія будуць выкарыстоўвацца для падлучэння да REST API.

  • Прэфікс працоўнай вобласці і URL: Інфармацыя працоўнай вобласці, у якую будуць перададзены даныя.

  • URL сэрвісу WMS/WFS/WCS: URL-адрасы сэрвісаў, якія будуць выкарыстоўвацца пры даданні спасылак на сэрвісы ў запіс метададзеных пасля публікацыі.
